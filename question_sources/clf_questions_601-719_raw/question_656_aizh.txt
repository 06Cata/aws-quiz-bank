题目分析与解答
1. 考察的知识点
本题考察的是AWS Identity and Access Management (IAM)的基本概念，特别是IAM用户和IAM角色的使用场景。
IAM用户通常用于为个人创建访问凭证，而IAM角色通常用于应用程序或服务之间的访问授权。
2. 选项分析
A. When an application that runs on Amazon EC2 instances requires access to other AWS services
错误：对于运行在EC2实例上的应用程序，最佳实践是使用IAM角色而不是IAM用户。IAM角色可以附加到EC2实例，允许应用程序

安全地访问其他AWS服务，而无需管理长期凭证。
B. When the company creates AWS access credentials for individuals
正确：IAM用户是为个人创建访问凭证的正确选择。每个IAM用户都有唯一的访问密钥和密码，可以直接登录AWS管理控制台或通
过API访问AWS服务。
C. When the company creates an application that runs on a mobile phone that makes requests to AWS
错误：对于移动应用程序，通常使用Amazon Cognito或IAM角色来管理访问权限，而不是IAM用户。IAM用户不适合直接用于应用
程序的访问授权。
D. When the company needs to add users to IAM groups
错误：IAM组是用于管理多个IAM用户的权限的集合。虽然IAM用户可以被添加到IAM组，但这并不是创建IAM用户的主要原因。
E. When users are authenticated in the corporate network and want to be able to use AWS without having to sign in a
second time
错误：这种场景通常使用AWS Single Sign-On (SSO)或联合身份验证，而不是创建IAM用户。
3. 我的答案
B. When the company creates AWS access credentials for individuals
