What is a benefit of using an Elastic Load Balancing (ELB) load balancer with applications running in the AWS Cloud?
使用 Elastic Load Balancing (ELB) 负载均衡器与运行在 AWS 云中的应用程序有什么好处？
A. An ELB will automatically scale resources to meet capacity needs.
ELB 会自动扩展资源以满足容量需求。
B. An ELB can balance traffic across multiple compute resources.
ELB 可以在多个计算资源之间平衡流量。
C. An ELB can span multiple AWS Regions.
ELB 可以跨多个 AWS 区域。
D. An ELB can balance traffic between multiple internet gateways.
ELB 可以在多个互联网网关之间平衡流量。
题目分析与解答
1. 考察的知识点
本题考察的是 AWS Elastic Load Balancing (ELB) 的核心功能和特性。
重点在于理解 ELB 的作用，包括流量分配、扩展能力以及区域范围等。
2. 选项分析
A. An ELB will automatically scale resources to meet capacity needs.
错误：ELB 本身不会自动扩展计算资源，它的功能是分配流量到现有的计算资源（如 EC2 实例）。
扩展计算资源通常需要结合 Auto Scaling 组来实现，而不是 ELB 的直接功能。
B. An ELB can balance traffic across multiple compute resources.
正确：这是 ELB 的核心功能之一。ELB 可以将流量分配到多个计算资源（如 EC2 实例），以确保应用的高可用性和负载均衡。
Correct Answer: B
正确答案：B

C. An ELB can span multiple AWS Regions.
错误：ELB 的范围通常限制在单个 AWS 区域内。虽然可以在多个区域部署 ELB，但单个 ELB 不能跨区域工作。
D. An ELB can balance traffic between multiple internet gateways.
错误：ELB 的功能是分配流量到计算资源（如 EC2 实例），而不是在多个互联网网关之间分配流量。
