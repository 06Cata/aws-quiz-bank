Smile03 1 year, 5 months ago
Selected Answer: D
A: Security group acts as a virtual firewall for your EC2 instances to control incoming and outgoing traffic. Inbound rules control the incoming traffic
to your instance, and outbound rules control the outgoing traffic from your instance. When you launch an instance, you can specify one or more
security groups. If you don't specify a security group, Amazon EC2 uses the default security group for the VPC. After you launch an instance, you can
change its security groups.
C: Your VPC automatically comes with a modifiable default network ACL. By default, it allows all inbound and outbound IPv4 traffic and, if applicable,
IPv6 traffic.
You can create a custom network ACL and associate it with a subnet to allow or deny specific inbound or outbound traffic at the subnet level.

Selected Answer: D
D. Network ACL (Access Control List)
Network ACLs act as a firewall for controlling traffic at the subnet level. They are stateless and operate at the subnet level, allowing or denying traffic
based on rules defined for inbound and outbound traffic. Network ACLs provide an added layer of security by allowing you to specify rules that
govern traffic at the network level, complementing the security groups that operate at the instance level.
