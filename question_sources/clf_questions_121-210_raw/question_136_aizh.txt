Which pricing model will interrupt a running Amazon EC2 instance if capacity becomes temporarily unavailable?
哪种定价模式会在容量暂时不可用时中断正在运行的 Amazon EC2 实例？
A. On-Demand Instances
按需实例
B. Standard Reserved Instances
标准预留实例
C. Spot Instances Most Voted
竞价实例

可转换预留实例
题目分析与解答
1. 考察的知识点
本题考察的是AWS EC2实例的定价模型及其行为特点，特别是竞价实例（Spot Instances）的特性。
2. 选项分析
A. On-Demand Instances 按需实例: 按需实例不会因为容量问题被中断。用户按小时或秒计费，随时启动或停止实例，适合短期或
不可预测的工作负载。因此此选项错误。
B. Standard Reserved Instances 标准预留实例: 标准预留实例是用户提前预付费用以换取长期使用的折扣，容量是预留的，不会因
为容量问题被中断。因此此选项错误。
C. Spot Instances 竞价实例: 竞价实例以低价运行，但可能会因为容量不足或价格超过用户设置的上限而被中断。这是竞价实例的
核心特性。因此此选项正确。
Correct Answer: C
正确答案：C
