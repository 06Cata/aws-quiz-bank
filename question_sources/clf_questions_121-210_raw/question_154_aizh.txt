Question #154
Which of the following is a recommended design principle for AWS Cloud architecture?
以下哪项是 AWS 云架构的推荐设计原则？
A. Design tightly coupled components.
设计紧密耦合的组件。
B. Build a single application component that can handle all the application functionality.
构建一个单一的应用程序组件来处理所有的应用程序功能。
C. Make large changes on fewer iterations to reduce chances of failure.
通过减少迭代次数进行大规模更改以降低失败的可能性。
D. Avoid monolithic architecture by segmenting workloads. Most Voted
通过分割工作负载来避免单体架构。
题目分析与解答
1. 考察的知识点
本题考察的是AWS云架构设计原则，特别是与分布式系统设计相关的最佳实践。
重点在于理解如何设计可扩展、灵活且高可用的架构。
2. 选项分析
A. Design tightly coupled components. 设计紧密耦合的组件。
错误：AWS推荐设计松耦合的组件，以提高系统的灵活性和容错能力。紧密耦合的组件会导致系统难以扩展和维护。
B. Build a single application component that can handle all the application functionality. 构建一个单一的应用程序组件来处
理所有的应用程序功能。
Correct Answer: D
正确答案：D

错误：单一组件的设计属于单体架构（Monolithic Architecture），这种架构难以扩展且故障影响范围较大，不符合AWS推荐的分布
式架构设计原则。
C. Make large changes on fewer iterations to reduce chances of failure. 通过减少迭代次数进行大规模更改以降低失败的可能
性。
错误：AWS推荐通过小规模、频繁的迭代来降低风险，而不是进行大规模更改。持续集成和持续交付（CI/CD）是AWS的最佳实践
之一。
D. Avoid monolithic architecture by segmenting workloads. 通过分割工作负载来避免单体架构。
正确：AWS推荐通过分布式架构设计来避免单体架构。分割工作负载可以提高系统的可扩展性、容错能力和灵活性。
