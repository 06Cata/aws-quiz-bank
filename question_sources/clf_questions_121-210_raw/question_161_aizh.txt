Question #161
A company is running a critical workload on an Amazon RDS DB instance. The company needs the DB instance to be highly
available with a recovery time of less than 5 minutes.
Which solution will meet these requirements?
一家公司在一台Amazon RDS DB实例上运行关键工作负载。该公司需要该DB实例具有高可用性，并且恢复时间少于5分钟。
哪种解决方案可以满足这些需求？
A. Create a read replica of the DB instance.
创建该DB实例的只读副本。
B. Create a template of the DB instance by using AWS CloudFormation.
使用AWS CloudFormation创建该DB实例的模板。
C. Take frequent snapshots of the DB instance. Store the snapshots in Amazon S3.
频繁对该DB实例进行快照。将快照存储在Amazon S3中。
D. Modify the DB instance to be a Multi-AZ deployment. Most Voted
将该DB实例修改为Multi-AZ部署。
题目分析与解答
1. 考察的知识点
本题考察的是Amazon RDS的高可用性（High Availability）和灾难恢复（Disaster Recovery）功能，特别是Multi-AZ部署的作用。
2. 选项分析

错误：Read Replica主要用于读操作的性能优化，而不是高可用性。它不能自动接管主实例的写操作，也无法满足5分钟的恢复时间
要求。
B. Create a template of the DB instance by using AWS CloudFormation.
错误：CloudFormation模板用于资源的自动化部署，但它不能提供实时的高可用性或快速恢复功能。
C. Take frequent snapshots of the DB instance. Store the snapshots in Amazon S3.
错误：快照是备份机制，恢复时间取决于快照的大小和恢复过程，通常无法满足5分钟的恢复时间要求。
D. Modify the DB instance to be a Multi-AZ deployment.
正确：Multi-AZ部署提供高可用性，自动故障转移到备用实例，能够满足5分钟的恢复时间要求。
3. 我的答案
D. Modify the DB instance to be a Multi-AZ deployment.
4. 官方答案
D. Modify the DB instance to be a Multi-AZ deployment.
