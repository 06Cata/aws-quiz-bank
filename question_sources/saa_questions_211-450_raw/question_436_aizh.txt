Question #436
A company moved its on-premises PostgreSQL database to an Amazon RDS for PostgreSQL DB instance. The company
successfully launched a new product. The workload on the database has increased. The company wants to accommodate the
larger workload without adding infrastructure.
Which solution will meet these requirements MOST cost-effectively?

作负载有所增加。公司希望在不增加基础设施的情况下容纳更大的工作负载。

题目分析与解答
1. 考察的知识点
Amazon RDS的扩展能力和成本优化。
Correct Answer: A
正确答案：A

如何选择合适的实例类型（预留实例 vs 按需实例）。
如何处理数据库性能问题（扩容 vs 多可用区 vs 添加实例）。
2. 选项分析
A. Buy reserved DB instances for the total workload. Make the Amazon RDS for PostgreSQL DB instance larger.
正确：预留实例可以显著降低成本，适合长期稳定的工作负载。
正确：扩容现有实例是直接提升性能的有效方法，尤其是在单实例的情况下。
此选项既解决了性能问题，又优化了成本，是最符合题目要求的解决方案。
B. Make the Amazon RDS for PostgreSQL DB instance a Multi-AZ DB instance.
错误：Multi-AZ主要用于提高可用性和容错能力，而不是直接提升性能。
此选项无法解决性能问题，也没有提到成本优化。
C. Buy reserved DB instances for the total workload. Add another Amazon RDS for PostgreSQL DB instance.
部分正确：预留实例可以降低成本。
错误：添加另一个实例需要额外的基础设施和配置（如读写分离），并不符合题目要求的“无需添加基础设施”。
D. Make the Amazon RDS for PostgreSQL DB instance an on-demand DB instance.
错误：按需实例成本较高，不适合长期稳定的工作负载。
此选项既没有解决性能问题，也没有优化成本。
