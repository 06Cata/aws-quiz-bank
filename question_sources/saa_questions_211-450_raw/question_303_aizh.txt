Question #303
A company is launching a new application deployed on an Amazon Elastic Container Service (Amazon ECS) cluster and is using the
Fargate launch type for ECS tasks. The company is monitoring CPU and memory usage because it is expecting high traffic to the
application upon its launch. However, the company wants to reduce costs when utilization decreases.
What should a solutions architect recommend?

题目分析与解答
1. 考察的知识点
Amazon ECS Fargate的自动扩展能力。
AWS Application Auto Scaling的使用场景和功能。
如何基于CloudWatch指标实栥自动扩展。
2. 选项分析
A. 使用 Amazon EC2 Auto Scaling 根据以前的流量模式在特定时间段进行扩展。
错误：EC2 Auto Scaling主要用于管栬EC2实例的扩展，而题目明确指出使用的是Fargate launch type，Fargate不依赖EC2实例。
桮外，基于“以前的械量桟式”进行扩展并不能实时响应当前的械量需梆。
B. 使用 AWS Lambda 函数根据触发 Amazon CloudWatch 警报的指标超限情况扩展 Amazon ECS。
错误：虽棺Lambda可以用来触发扩展，但这不是推荐的方式。AWS Application Auto Scaling已经内置了自动扩展功能，直接使用
它更高效。
使用Lambda函数增加了复杂性，且不符合最佳实践。
C. 使用 Amazon EC2 Auto Scaling 和简单扩展策略，当 ECS 指标超限触发 Amazon CloudWatch 警报时进行扩展。
错误：同样的问题，EC2 Auto Scaling适用于EC2实例，而题目使用的是Fargate launch type。
简单扩展策略不如目标跟踪策略棬梮和高效。
D. 使用 AWS Application Auto Scaling 和目标跟踪策略，当 ECS 指标超限触发 Amazon CloudWatch 警报时进行扩展。
桭确：AWS Application Auto Scaling支持Fargate任务的自动扩展。
目标跟踪策略可以根据CloudWatch指标（如CPU和内存使用栞）动态调整任务数量，符合题目需梆。
3. 我选的答案
D
