Question #222
A company has hired an external vendor to perform work in the company’s AWS account. The vendor uses an automated tool that
is hosted in an AWS account that the vendor owns. The vendor does not have IAM access to the company’s AWS account.
How should a solutions architect grant this access to the vendor?
一家公司聘请了一个外部供应商在该公司的AWS账户中执行工作。该供应商使用一个自动化工具，该工具托管在供应商拥有的AWS
账户中。供应商没有IAM访问权限来操作公司的AWS账户。
解决方案架构师应该如何授予供应商访问权限？
A. Create an IAM role in the company’s account to delegate access to the vendor’s IAM role. Attach the appropriate IAM
policies to the role for the permissions that the vendor requires. Most Voted
在公司的账户中创建一个IAM角色，以委派访问权限给供应商的IAM角色。将供应商所需权限的适当IAM策略附加到该角色上。
B. Create an IAM user in the company’s account with a password that meets the password complexity requirements. Attach the
appropriate IAM policies to the user for the permissions that the vendor requires.
在公司的账户中创建一个IAM用户，并设置符合密码复杂性要求的密码。将供应商所需权限的适当IAM策略附加到该用户上。
C. Create an IAM group in the company’s account. Add the tool’s IAM user from the vendor account to the group. Attach the

上。
D. Create a new identity provider by choosing “AWS account” as the provider type in the IAM console. Supply the vendor’s
AWS account ID and user name. Attach the appropriate IAM policies to the new provider for the permissions that the vendor
requires.

权限的适当IAM策略附加到新的身份提供者上。

题目分析与解答
1. 考察的知识点
IAM角色的跨账户访问
权限委派与安全最佳实践
如何为外部供应商提供访问权限
2. 选项分析
A. Create an IAM role in the company’s account to delegate access to the vendor’s IAM role. Attach the appropriate IAM
policies to the role for the permissions that the vendor requires.
正确：这是AWS推荐的最佳实践。通过创建IAM角色并允许外部账户的IAM角色进行假设（assume role），可以安全地实现跨账户
访问。
这种方法避免了直接创建用户或共享密码，符合安全性和权限最小化原则。
B. Create an IAM user in the company’s account with a password that meets the password complexity requirements.
Attach the appropriate IAM policies to the user for the permissions that the vendor requires.
错误：创建IAM用户并共享密码不符合安全最佳实践。密码可能会被泄露或滥用，且无法实现自动化工具的跨账户访问。
此方法不适合供应商的自动化工具使用场景。
C. Create an IAM group in the company’s account. Add the tool’s IAM user from the vendor account to the group. Attach
the appropriate IAM policies to the group for the permissions that the vendor requires.
错误：IAM组仅适用于同一账户内的用户管理，无法跨账户使用。
供应商的IAM用户无法直接加入公司账户的IAM组。
D. Create a new identity provider by choosing “AWS account” as the provider type in the IAM console. Supply the
vendor’s AWS account ID and user name. Attach the appropriate IAM policies to the new provider for the permissions that
the vendor requires.
错误：AWS身份提供者主要用于联合身份认证（Federation），例如与SAML或OpenID Connect集成，而不是用于跨账户访问。
此选项与题目场景不匹配。

理由：选项A符合AWS最佳实践，能够安全地实现跨账户访问，同时满足供应商自动化工具的需求。
