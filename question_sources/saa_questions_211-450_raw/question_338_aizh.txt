Question #338
A solutions architect must create a disaster recovery (DR) plan for a high-volume software as a service (SaaS) platform. All data for
the platform is stored in an Amazon Aurora MySQL DB cluster.
The DR plan must replicate data to a secondary AWS Region.
Which solution will meet these requirements MOST cost-effectively?
解决方案架构师必须为一个高流量的软件即服务 (SaaS) 平台创建灾难恢复 (DR) 计划。该平台的所有数据都存储在 Amazon Aurora
MySQL DB 集群中。
DR 计划必须将数据复制到次级 AWS 区域。
哪种解决方案可以以最具成本效益的方式满足这些需求？
A. Use MySQL binary log replication to an Aurora cluster in the secondary Region. Provision one DB instance for the Aurora
cluster in the secondary Region.
使用 MySQL 二进制日志复制到次级区域的 Aurora 集群。为次级区域的 Aurora 集群预置一个 DB 实例。

使用 AWS Database Migration Service (AWS DMS) 将数据连续复制到次级区域的 Aurora 集群。从次级区域移除 DB 实例。

题目分析与解答
1. 考察的知识点
Amazon Aurora全球数据库的功能与配置。
灾难恢复（Disaster Recovery）解决方案的设计。
成本优化的最佳实践。
2. 选项分析
A. 使用 MySQL 二进制日志复制到次级区域的 Aurora 集群。为次级区域的 Aurora 集群预置一个 DB 实例。
正确性分析：MySQL 二进制日志复制是一种可行的解决方案，但它需要手动设置和管理，且不如 Aurora 全球数据库的自动化和高
效性。
成本分析：需要持续运行一个 DB 实例，成本较高。
结论：此选项正确性较低，且成本不够优化。
B. 为 DB 集群设置 Aurora 全球数据库。设置完成后，从次级区域移除 DB 实例。
正确性分析：Aurora 全球数据库支持跨区域复制，且复制速度快，延迟低。移除次级区域的 DB 实例后，仍然可以保留存储层，降
低成本。
成本分析：此选项通过移除次级区域的 DB 实例，显著降低了运行成本。
结论：此选项正确且成本优化，是最佳选择。
C. 使用 AWS Database Migration Service (AWS DMS) 将数据连续复制到次级区域的 Aurora 集群。从次级区域移除 DB 实例。
正确性分析：AWS DMS 可以用于数据迁移，但它主要用于一次性迁移或轻量级的持续复制，不适合高性能、高容量的 SaaS 平台。
成本分析：DMS 的持续运行成本较高，且管理复杂度增加。
结论：此选项正确性较低，且成本不够优化。
D. 为 DB 集群设置 Aurora 全球数据库。在次级区域指定至少一个 DB 实例。
正确性分析：Aurora 全球数据库是正确的解决方案，但指定至少一个 DB 实例会增加运行成本。
成本分析：次级区域的 DB 实例会持续运行，成本较高。
结论：此选项正确性较高，但成本不够优化。
3. 我的答案
B. 为 DB 集群设置 Aurora 全球数据库。设置完成后，从次级区域移除 DB 实例。
4. 官方答案
B. 为 DB 集群设置 Aurora 全球数据库。设置完成后，从次级区域移除 DB 实例。
5. 与官方答案比较
我的答案与官方答案一致。
原因分析：Aurora 全球数据库是专为跨区域灾难恢复设计的解决方案，支持高效的跨区域复制，同时通过移除次级区域的 DB 实例
优化了成本。
