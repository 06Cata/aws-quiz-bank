Question #237
An application running on an Amazon EC2 instance in VPC-A needs to access files in another EC2 instance in VPC-B. Both VPCs are
in separate AWS accounts. The network administrator needs to design a solution to configure secure access to EC2 instance in
VPC-B from VPC-A. The connectivity should not have a single point of failure or bandwidth concerns.
Which solution will meet these requirements?
一个运行在 VPC-A 中的 Amazon EC2 实例的应用程序需要访问另一个运行在 VPC-B 中的 EC2 实例中的文件。两个 VPC 分别位于不
同的 AWS 账户中。网络管理员需要设计一个解决方案，以配置从 VPC-A 到 VPC-B 中的 EC2 实例的安全访问。连接性不应有单点
故障或带宽问题。
哪个解决方案可以满足这些要求？

2. 选项分析
A. Set up a VPC peering connection between VPC-A and VPC-B.
正确：VPC Peering允许两个VPC之间直接通信，即使它们属于不同的AWS账户。它是一个高带宽、低延迟的解决方案，没有单点故
障。
符合题目要求：安全、可靠的连接，没有单点故障或带宽问题。
B. Set up VPC gateway endpoints for the EC2 instance running in VPC-B.
错误：VPC Gateway Endpoints用于访问AWS服务（如S3或DynamoDB），而不是用于VPC之间的通信。
不符合题目要求：无法实现VPC-A与VPC-B之间的直接通信。
C. Attach a virtual private gateway to VPC-B and set up routing from VPC-A.
错误：Virtual Private Gateway通常用于连接VPC与本地数据中心（通过VPN或Direct Connect），而不是用于VPC之间的通信。
不符合题目要求：无法直接实现VPC-A与VPC-B之间的通信。
D. Create a private virtual interface (VIF) for the EC2 instance running in VPC-B and add appropriate routes from VPC-A.
错误：Private Virtual Interface (VIF)是Direct Connect的一部分，通常用于连接本地数据中心与AWS，而不是用于VPC之间的通信。
不符合题目要求：无法实现VPC-A与VPC-B之间的直接通信。

故障或带宽问题。
