Question #371
A company needs to create an Amazon Elastic Kubernetes Service (Amazon EKS) cluster to host a digital media streaming
application. The EKS cluster will use a managed node group that is backed by Amazon Elastic Block Store (Amazon EBS) volumes
for storage. The company must encrypt all data at rest by using a customer managed key that is stored in AWS Key Management
Service (AWS KMS).
Which combination of actions will meet this requirement with the LEAST operational overhead? (Choose two.)
一家公司需要创建一个 Amazon Elastic Kubernetes Service (Amazon EKS) 集群来托管数字媒体流应用程序。该 EKS 集群将使用由
Amazon Elastic Block Store (Amazon EBS) 卷支持的托管节点组进行存储。公司必须使用存储在 AWS Key Management Service
(AWS KMS) 中的客户管理密钥对所有静态数据进行加密。
以下哪种操作组合能够以最少的运营开销满足此要求？（选择两个）
A. Use a Kubernetes plugin that uses the customer managed key to perform data encryption.
使用一个 Kubernetes 插件，该插件使用客户管理密钥执行数据加密。
B. After creation of the EKS cluster, locate the EBS volumes. Enable encryption by using the customer managed key.
在创建 EKS 集群后，定位 EBS 卷。使用客户管理密钥启用加密。
C. Enable EBS encryption by default in the AWS Region where the EKS cluster will be created. Select the customer managed key
as the default key. Most Voted
在创建 EKS 集群的 AWS 区域中启用 EBS 默认加密。选择客户管理密钥作为默认密钥。
D. Create the EKS cluster. Create an IAM role that has a policy that grants permission to the customer managed key. Associate
the role with the EKS cluster. Most Voted
创建 EKS 集群。创建一个具有授予客户管理密钥权限策略的 IAM 角色。将该角色与 EKS 集群关联。
E. Store the customer managed key as a Kubernetes secret in the EKS cluster. Use the customer managed key to encrypt the
EBS volumes.
将客户管理密钥存储为 Kubernetes secret 在 EKS 集群中。使用客户管理密钥加密 EBS 卷。
Correct Answer: CD
正确答案：CD

题目分析与解答
1. 考察的知识点
Amazon EKS集群的创建与管理。
Amazon EBS卷的加密机制。
AWS Key Management Service (KMS)的使用与集成。
如何使用客户管理密钥 (Customer Managed Key) 实现数据加密。
最小化操作开销的最佳实践。
2. 选项分析
A. 使用一个 Kubernetes 插件，该插件使用客户管理密钥执行数据加密。
错误：虽然可以使用Kubernetes插件来实现加密，但这会增加额外的操作开销，并且不符合“最小化操作开销”的要求。
B. 在创建 EKS 集群后，定位 EBS 卷。使用客户管理密钥启用加密。
错误：手动定位EBS卷并启用加密需要额外的操作步骤，增加了操作开销，不符合题目要求。
C. 在创建 EKS 集群的 AWS 区域中启用 EBS 默认加密。选择客户管理密钥作为默认密钥。
正确：启用EBS默认加密并选择客户管理密钥作为默认密钥是一个自动化的解决方案，减少了操作开销。
D. 创建 EKS 集群。创建一个具有授予客户管理密钥权限策略的 IAM 角色。将该角色与 EKS 集群关联。
正确：为EKS集群创建IAM角色并授予访问客户管理密钥的权限是实现加密的必要步骤，同时不会增加额外的操作开销。
E. 将客户管理密钥存储为 Kubernetes secret 在 EKS 集群中。使用客户管理密钥加密 EBS 卷。
错误：将密钥存储为Kubernetes secret并手动加密EBS卷增加了操作复杂性，不符合“最小化操作开销”的要求。
