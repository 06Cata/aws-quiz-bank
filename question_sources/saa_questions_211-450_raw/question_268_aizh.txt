Question #268
A gaming company has a web application that displays scores. The application runs on Amazon EC2 instances behind an
Application Load Balancer. The application stores data in an Amazon RDS for MySQL database. Users are starting to experience
long delays and interruptions that are caused by database read performance. The company wants to improve the user experience
while minimizing changes to the application’s architecture.
What should a solutions architect do to meet these requirements?
一家游戏公司有一个显示分数的网页应用程序。 该应用程序运行在 Amazon EC2 实例上，并位于 Application Load Balancer 后面。

司希望在尽量减少对应用程序架构更改的情况下改善用户体验。

2. 选项分析
A. Use Amazon ElastiCache in front of the database.
正确：Amazon ElastiCache是一种内存缓存服务，可以显著提高数据库的读取性能，减少用户延迟。
符合题目要求：通过缓存减少数据库的读取压力，同时对现有架构改动最小。
B. Use RDS Proxy between the application and the database.
错误：RDS Proxy主要用于优化数据库连接管理，减少连接开销，但不会直接提高数据库的读取性能。
不符合题目要求：用户体验问题是由于读取性能不足，而不是连接管理问题。
C. Migrate the application from EC2 instances to AWS Lambda.
错误：迁移到AWS Lambda不会直接解决数据库读取性能问题。
不符合题目要求：题目要求对现有架构改动最小，而迁移到Lambda涉及较大的架构变更。
D. Migrate the database from Amazon RDS for MySQL to Amazon DynamoDB.
错误：DynamoDB是NoSQL数据库，与RDS的关系型数据库架构不同，迁移会导致架构和应用程序的重大改动。
不符合题目要求：题目要求对现有架构改动最小，而迁移到DynamoDB涉及较大的架构变更。
3. 我的答案
A. Use Amazon ElastiCache in front of the database.
