题目分析与解答
1. 考察的知识点
事件驱动架构的设计与实现。
AWS服务的选择与最佳实践，包括AWS Step Functions、AWS Lambda、Amazon EventBridge、AWS Glue等。
如何构建分布式、无服务器架构以减少运营开销。
2. 选项分析
A. Build out the workflow in AWS Glue. Use AWS Glue to invoke AWS Lambda functions to process the workflow steps.
正确性分析：虽然AWS Glue可以用于数据处理和ETL任务，但它并不是设计事件驱动架构的最佳选择。AWS Glue主要用于数据转换
和管理，而不是构建分布式工作流。
结论：错误。
B. Build out the workflow in AWS Step Functions. Deploy the application on Amazon EC2 instances. Use Step Functions to
invoke the workflow steps on the EC2 instances.
正确性分析：虽然AWS Step Functions可以用于构建工作流，但使用EC2实例会增加运营开销，不符合题目要求的“最小化运营开
销”和“无服务器架构”的要求。
结论：错误。
C. Build out the workflow in Amazon EventBridge. Use EventBridge to invoke AWS Lambda functions on a schedule to
process the workflow steps.
正确性分析：Amazon EventBridge适合事件驱动架构，但它主要用于事件路由和调度，而不是构建复杂的工作流。题目要求的是一
个分布式工作流，而EventBridge无法直接满足这一需求。
结论：错误。
D. Build out the workflow in AWS Step Functions. Use Step Functions to create a state machine. Use the state machine to
invoke AWS Lambda functions to process the workflow steps.
正确性分析：AWS Step Functions是构建分布式工作流的最佳选择。它支持状态机，可以轻松集成AWS Lambda以实现无服务器架
构，同时减少运营开销。
结论：正确。
3. 我的答案
D
4. 官方答案
D
5. 与官方答案的比较
我的答案与官方答案一致。
原因分析：AWS Step Functions结合AWS Lambda是构建事件驱动、分布式、无服务器架构的最佳实践，完全符合题目要求。
