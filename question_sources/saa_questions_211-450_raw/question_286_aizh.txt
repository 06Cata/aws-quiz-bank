Question #286
A company has a static website that is hosted on Amazon CloudFront in front of Amazon S3. The static website uses a database
backend. The company notices that the website does not reflect updates that have been made in the website’s Git repository. The
company checks the continuous integration and continuous delivery (CI/CD) pipeline between the Git repository and Amazon S3.
The company verifies that the webhooks are configured properly and that the CI/CD pipeline is sending messages that indicate
successful deployments.
A solutions architect needs to implement a solution that displays the updates on the website.
Which solution will meet these requirements?
一家公司有一个静态网站，该网站托管在 Amazon S3 上，并通过 Amazon CloudFront 提供服务。 该静态网站使用一个数据库后
端。 公司注意到网站没有反映出在网站的 Git 仓库中所做的更新。 公司检查了 Git 仓库与 Amazon S3 之间的持续集成和持续交付
(CI/CD) 管道。 公司验证了 webhook 配置正确，并且 CI/CD 管道发送了表示部署成功的消息。
解决方案架构师需要实施一个解决方案，以便在网站上显示更新。
满足这些需求的解决方案是什么？
A. Add an Application Load Balancer.
添加一个 Application Load Balancer。
B. Add Amazon ElastiCache for Redis or Memcached to the database layer of the web application.
向 web 应用程序的数据库层添加 Amazon ElastiCache for Redis 或 Memcached。
C. Invalidate the CloudFront cache. Most Voted
使 CloudFront 缓存失效。
D. Use AWS Certificate Manager (ACM) to validate the website’s SSL certificate.
使用 AWS Certificate Manager (ACM) 验证网站的 SSL 证书。
Correct Answer: C
正确答案：C
题目分析与解答
1. 考察的知识点
Amazon CloudFront缓存机制及如何使缓存失效。
静态网站托管在Amazon S3上的最佳实践。
CI/CD管道与部署更新的流程。
2. 选项分析
A. Add an Application Load Balancer.
错误：Application Load Balancer主要用于分发流量到多个后端服务或实例，与解决CloudFront缓存问题无关。
B. Add Amazon ElastiCache for Redis or Memcached to the database layer of the web application.
错误：ElastiCache用于加速数据库查询或缓存动态内容，与静态网站的更新问题无关。
C. Invalidate the CloudFront cache.
正确：CloudFront会缓存内容以提高性能，但可能导致更新的内容无法及时显示。使缓存失效可以强制CloudFront从S3获取最新内
容。
D. Use AWS Certificate Manager (ACM) to validate the website’s SSL certificate.
错误：ACM用于管理SSL证书，与网站内容更新无关。
3. 我选的答案
C. Invalidate the CloudFront cache.
4. 官方答案
C. Invalidate the CloudFront cache.
5. 与官方答案比较
我的答案与官方答案一致。
原因：使CloudFront缓存失效是解决静态网站更新问题的正确方法。
