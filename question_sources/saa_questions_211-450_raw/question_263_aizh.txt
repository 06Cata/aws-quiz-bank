题目分析与解答
1. 考察的知识点
本题主要考察容器技术在AWS上的部署方式，包括Amazon ECS和Fargate的使用。
重点关注如何减少运维工作量、自动扩展能力以及避免额外的基础设施管理。
2. 选项分析
A. Deploy an Amazon Elastic Container Service (Amazon ECS) cluster.
分析：正确。Amazon ECS是AWS的托管容器服务，支持自动化管理和扩展。部署ECS集群是实现容器化应用的基础步骤。
B. Deploy the Kubernetes control plane on Amazon EC2 instances that span multiple Availability Zones.
分析：错误。部署Kubernetes控制平面需要额外管理基础设施（EC2实例），这与题目要求的“不能管理额外基础设施”相违背。
C. Deploy an Amazon Elastic Container Service (Amazon ECS) service with an Amazon EC2 launch type. Specify a desired
task number level of greater than or equal to 2.
分析：错误。使用EC2启动类型需要管理底层EC2实例，这增加了运维工作量，与题目要求的“减少维护工作量”不符。
D. Deploy an Amazon Elastic Container Service (Amazon ECS) service with a Fargate launch type. Specify a desired task
number level of greater than or equal to 2.
分析：正确。Fargate是无服务器的容器运行时，完全由AWS管理底层基础设施，符合题目要求的“减少维护工作量”和“不能管理额外
基础设施”。
E. Deploy Kubernetes worker nodes on Amazon EC2 instances that span multiple Availability Zones. Create a deployment
that specifies two or more replicas for each microservice.
分析：错误。部署Kubernetes工作节点需要管理EC2实例，与题目要求的“不能管理额外基础设施”相违背。
