Question #448
A company has two VPCs named Management and Production. The Management VPC uses VPNs through a customer gateway to
connect to a single device in the data center. The Production VPC uses a virtual private gateway with two attached AWS Direct
Connect connections. The Management and Production VPCs both use a single VPC peering connection to allow communication
between the applications.
What should a solutions architect do to mitigate any single point of failure in this architecture?
一家公司有两个名为 Management 和 Production 的 VPC。Management VPC 使用通过客户网关的 VPN 来连接数据中心中的单一
设备。Production VPC 使用一个虚拟私有网关，并附加了两个 AWS Direct Connect 连接。Management 和 Production VPC 都使用
单一 VPC 对等连接来允许应用程序之间的通信。
解决方案架构师应该怎么做以减轻此架构中的任何单点故障？
A. Add a set of VPNs between the Management and Production VPCs.
在 Management 和 Production VPC 之间添加一组 VPN。
B. Add a second virtual private gateway and attach it to the Management VPC.
为 Management VPC 添加第二个虚拟私有网关并将其附加到 Management VPC。
C. Add a second set of VPNs to the Management VPC from a second customer gateway device. Most Voted
从第二个客户网关设备向 Management VPC 添加第二组 VPN。
D. Add a second VPC peering connection between the Management VPC and the Production VPC.
在 Management VPC 和 Production VPC 之间添加第二个 VPC 对等连接。

2. 选项分析
A. Add a set of VPNs between the Management and Production VPCs.
错误：VPC Peering已经存在于Management和Production VPC之间，添加VPN连接不会解决单点故障问题。
VPN连接的主要作用是跨网络连接，而不是解决单点故障。
B. Add a second virtual private gateway and attach it to the Management VPC.
错误：Management VPC已经通过VPN连接到数据中心，添加第二个虚拟私有网关不会解决单点故障问题。
虚拟私有网关主要用于Direct Connect，而不是VPN连接。
C. Add a second set of VPNs to the Management VPC from a second customer gateway device.
正确：当前Management VPC通过单个客户网关设备连接到数据中心，这存在单点故障风险。
添加第二组VPN连接并使用第二个客户网关设备可以提高冗余性，解决单点故障问题。
D. Add a second VPC peering connection between the Management VPC and the Production VPC.
错误：VPC Peering本质上是高可用的，添加第二个VPC Peering连接没有意义。
单点故障问题主要在Management VPC与数据中心的连接，而不是VPC Peering。
3. 我选的答案
C. Add a second set of VPNs to the Management VPC from a second customer gateway device.
4. 官方答案
C. Add a second set of VPNs to the Management VPC from a second customer gateway device.
