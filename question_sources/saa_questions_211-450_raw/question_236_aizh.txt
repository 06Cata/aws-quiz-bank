Question #236
A company has a three-tier application for image sharing. The application uses an Amazon EC2 instance for the front-end layer,
another EC2 instance for the application layer, and a third EC2 instance for a MySQL database. A solutions architect must design a
scalable and highly available solution that requires the least amount of change to the application.
Which solution meets these requirements?
一家公司有一个用于图片分享的三层应用程序。该应用程序使用一个 Amazon EC2 实例作为前端层，另一个 EC2 实例作为应用层，
第三个 EC2 实例作为 MySQL 数据库。解决方案架构师必须设计一个可扩展且高可用的解决方案，并且需要对应用程序进行最少的
更改。
哪个解决方案满足这些要求？
A. Use Amazon S3 to host the front-end layer. Use AWS Lambda functions for the application layer. Move the database to an
Amazon DynamoDB table. Use Amazon S3 to store and serve users’ images.

S3 存储和提供用户的图片。
B. Use load-balanced Multi-AZ AWS Elastic Beanstalk environments for the front-end layer and the application layer. Move the

题目分析与解答
1. 考察的知识点
高可用性和可扩展性设计：如何设计一个三层架构以实现高可用性和可扩展性。
AWS服务选择：包括Amazon S3、AWS Lambda、Elastic Beanstalk、Amazon RDS等服务的使用场景。
最小化应用改动：如何在现有架构基础上进行优化，减少对应用的改动。
2. 选项分析
A. 使用 Amazon S3 托管前端层。使用 AWS Lambda 函数作为应用层。将数据库迁移到 Amazon DynamoDB 表中。使用
Amazon S3 存储勨提供用户的划片。
正确性分析：此选项需要对应用进行大量改动，包括将应用层迁移到无服务器架构（AWS Lambda）以及将关系型数据库迁移到非
关系型数据库（DynamoDB）。虽然这些服务具有高可用性和可扩展性，但改动量较大，不符合“最少改动”的要求。
结论：错误。
B. 使用负载初衡的剉勄用劉 AWS Elastic Beanstalk 环刺作为前端层勨应用层。将数据库迁移到具有剉个勀读副本的 Amazon RDS
DB 实例以提供用户的划片。
正确性分析：Elastic Beanstalk提供了高可用性和自动扩展功能，适合前端和应用层。但数据库部分仅提到“多个只读副本”，未提到
主实例的高可用性（如Multi-AZ），因此数据库的高可用性未完全满足要求。
结论：错误。
C. 使用 Amazon S3 托管前端层。使用 Auto Scaling 组中的 EC2 实例群作为应用层。将数据库迁移到内存优劆的实例类利以存储
勨提供用户的划片。
正确性分析：此选项将前端迁移到Amazon S3，应用层使用Auto Scaling组，具有一定的可扩展性。但数据库部分仅提到“内存优化
实例”，未提到高可用性（如Multi-AZ），因此数据库的高可用性未完全满足要求。
结论：错误。
D. 使用负载初衡的剉勄用劉 AWS Elastic Beanstalk 环刺作为前端层勨应用层。将数据库迁移到 Amazon RDS 剉勄用劉 DB 实
例。使用 Amazon S3 存储勨提供用户的划片。
正确性分析：Elastic Beanstalk提供了高可用性和自动扩展功能，适合前端和应用层。Amazon RDS Multi-AZ提供了数据库的高可用
性。Amazon S3用于存储和提供用户图片，符合要求。此选项改动量最小，同时满足高可用性和可扩展性需求。
结论：正确。
