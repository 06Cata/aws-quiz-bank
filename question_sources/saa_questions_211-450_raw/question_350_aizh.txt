Question #350
A company uses a 100 GB Amazon RDS for Microsoft SQL Server Single-AZ DB instance in the us-east-1 Region to store customer
transactions. The company needs high availability and automatic recovery for the DB instance.
The company must also run reports on the RDS database several times a year. The report process causes transactions to take
longer than usual to post to the customers’ accounts. The company needs a solution that will improve the performance of the
report process.
Which combination of steps will meet these requirements? (Choose two.)
一家公司在 us-east-1 区域使用一个 100 GB 的 Amazon RDS for Microsoft SQL Server 单可用区 DB 实例来存储客户交易数据。该
公司需要为 DB 实例提供高可用性和自动恢复功能。
该公司还需要每年多次在 RDS 数据库上运行报告。报告处理会导致交易比平时更长时间才能记录到客户账户中。公司需要一种解决
方案来改善报告处理的性能。
哪些步骤组合可以满足这些要求？（选择两个。）
A. Modify the DB instance from a Single-AZ DB instance to a Multi-AZ deployment. Most Voted
将 DB 实例从单可用区 DB 实例修改为多可用区部署。
B. Take a snapshot of the current DB instance. Restore the snapshot to a new RDS deployment in another Availability Zone.
对当前 DB 实例进行快照。将快照恢复到另一个可用区的新 RDS 部署。
C. Create a read replica of the DB instance in a different Availability Zone. Point all requests for reports to the read replica.
Most Voted
在不同的可用区创建 DB 实例的只读副本。将所有报告请求指向只读副本。
D. Migrate the database to RDS Custom.
将数据库迁移到 RDS Custom。
E. Use RDS Proxy to limit reporting requests to the maintenance window.
使用 RDS Proxy 将报告请求限制到维护窗口。
Correct Answer: AC
正确答案：AC
题目分析与解答
1. 考察的知识点
Amazon RDS的高可用性配置（Multi-AZ部署）。
Amazon RDS的性能优化（使用Read Replica）。
如何解决报告运行导致性能下降的问题。
2. 选项分析
A. Modify the DB instance from a Single-AZ DB instance to a Multi-AZ deployment.
正确：Multi-AZ部署提供高可用性和自动故障恢复功能，满足题目中对高可用性和自动恢复的需求。
这是解决高可用性问题的推荐方法。
B. Take a snapshot of the current DB instance. Restore the snapshot to a new RDS deployment in another Availability
Zone.
错误：虽然可以通过快照恢复到另一个可用区，但这并不能提供自动故障恢复功能，也无法解决报告性能问题。
此方法不符合题目要求的高可用性和性能优化需求。
C. Create a read replica of the DB instance in a different Availability Zone. Point all requests for reports to the read replica.
正确：Read Replica可以分担读取负载，尤其是报告请求，从而提高主数据库的性能。
这是解决报告性能问题的推荐方法。
D. Migrate the database to RDS Custom.
错误：RDS Custom主要用于支持需要自定义操作系统或数据库的应用程序，不适用于解决高可用性或性能问题。
此选项与题目需求无关。
E. Use RDS Proxy to limit reporting requests to the maintenance window.
错误：RDS Proxy主要用于优化数据库连接管理，并不直接解决报告性能问题或提供高可用性。
限制报告请求到维护窗口并不符合题目需求。

理由：选项A解决了高可用性和自动恢复问题，选项C解决了报告性能问题。这两个选项完全符合题目要求。
