• Amazon EC2 instances in different AWS Regions

The company wants to protect the solution against DDoS attacks.

题目分析与解答
1. 考察的知识点
本题主要考察AWS的DDoS防护服务，包括AWS Shield Advanced和AWS WAF的使用场景。
了解AWS Global Accelerator的功能及其与DDoS防护的集成。
2. 选项分析
A. Subscribe to AWS Shield Advanced. Add the accelerator as a resource to protect.
正确：AWS Shield Advanced是专门设计用于保护资源免受DDoS攻击的服务。将AWS Global Accelerator作为受保护资源添加，可
以有效防护DDoS攻击，因为Global Accelerator的端点是流量的入口点。
此选项符合题目要求，且是最佳实践。
B. Subscribe to AWS Shield Advanced. Add the EC2 instances as resources to protect.
错误：虽然AWS Shield Advanced可以保护EC2实例，但题目中提到的流量入口点是Global Accelerator的端点，而不是直接访问EC2
实例。因此，保护EC2实例并不能有效防护DDoS攻击。
C. Create an AWS WAF web ACL that includes a rate-based rule. Associate the web ACL with the accelerator.
错误：AWS WAF主要用于防护应用层攻击（如SQL注入、跨站脚本攻击），而不是针对DDoS攻击。虽然可以创建基于速率的规
则，但这并不是针对DDoS攻击的最佳解决方案。
D. Create an AWS WAF web ACL that includes a rate-based rule. Associate the web ACL with the EC2 instances.
错误：同选项C，AWS WAF主要用于应用层防护，而不是针对DDoS攻击。将web ACL与EC2实例关联并不能有效防护DDoS攻击。
3. 我选的答案
A. Subscribe to AWS Shield Advanced. Add the accelerator as a resource to protect.
4. 官方答案
A. Subscribe to AWS Shield Advanced. Add the accelerator as a resource to protect.
5. 与官方答案比较
我的答案与官方答案一致。
原因：AWS Shield Advanced是针对DDoS攻击的最佳解决方案，而Global Accelerator是流量的入口点，保护它可以有效防护DDoS
攻击。
