题目分析与解答
1. 考察的知识点
EC2实例类型选择：Spot Instances、Reserved Instances、On-Demand Instances的成本和使用场景。
数据库服务选择：Amazon RDS、Amazon Aurora、Amazon S3的适用场景和成本优化。
成本优化：如何在满足应用需求的同时选择最具性价比的AWS服务。
2. 选项分析
A. Migrate the application layer to Amazon EC2 Spot Instances. Migrate the data storage layer to Amazon S3.
EC2 Spot Instances适合短期、非关键任务的工作负载，但不适合需要24/7运行的应用程序，因为Spot Instances可能会被中断。
Amazon S3是对象存储服务，不适合存储关系型数据库数据。
此选项不符合题目要求，错误。
B. Migrate the application layer to Amazon EC2 Reserved Instances. Migrate the data storage layer to Amazon RDS On-
Demand Instances.
EC2 Reserved Instances适合长期运行的应用程序，成本较低，符合要求。
RDS On-Demand Instances成本较高，不是最具性价比的选择。
此选项部分正确，但不是最优解。
C. Migrate the application layer to Amazon EC2 Reserved Instances. Migrate the data storage layer to Amazon Aurora
Reserved Instances.
EC2 Reserved Instances适合长期运行的应用程序，成本较低，符合要求。
Amazon Aurora是高性能、可扩展的关系型数据库服务，支持自动扩展，且使用Reserved Instances进一步降低成本，符合题目要
求。
此选项是最优解，正确。
D. Migrate the application layer to Amazon EC2 On-Demand Instances. Migrate the data storage layer to Amazon RDS
Reserved Instances.
EC2 On-Demand Instances成本较高，不适合长期运行的应用程序。
RDS Reserved Instances成本较低，适合长期运行的数据库，但EC2的选择不符合成本优化要求。
此选项部分正确，但不是最优解。
3. 我的答案
C. Migrate the application layer to Amazon EC2 Reserved Instances. Migrate the data storage layer to Amazon Aurora
Reserved Instances.
4. 官方答案
C. Migrate the application layer to Amazon EC2 Reserved Instances. Migrate the data storage layer to Amazon Aurora
Reserved Instances.
