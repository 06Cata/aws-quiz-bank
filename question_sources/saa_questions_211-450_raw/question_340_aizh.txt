Question #340
A media company hosts its website on AWS. The website application’s architecture includes a fleet of Amazon EC2 instances
behind an Application Load Balancer (ALB) and a database that is hosted on Amazon Aurora. The company’s cybersecurity team
reports that the application is vulnerable to SQL injection.
How should the company resolve this issue?
一家媒体公司将其网站托管在 AWS 上。该网站应用程序的架构包括一个位于 Application Load Balancer (ALB) 后面的 Amazon EC2
实例集群，以及一个托管在 Amazon Aurora 上的数据库。公司的网络安全团队报告该应用程序容易受到 SQL 注入攻击。
公司应该如何解决这个问题？
A. Use AWS WAF in front of the ALB. Associate the appropriate web ACLs with AWS WAF. Most Voted

创建一个 ALB 监听器规则，以固定响应回复 SQL 注入。

订阅 AWS Shield Advanced，以自动阻止所有 SQL 注入尝试。

题目分析与解答
1. 考察的知识点
本题考察的是AWS安全服务的使用，特别是如何防止SQL注入攻击。
Correct Answer: A
正确答案：A

A (100%)
涉及的服务包括AWS WAF、ALB、AWS Shield Advanced和Amazon Inspector。
2. 选项分析
A. Use AWS WAF in front of the ALB. Associate the appropriate web ACLs with AWS WAF.
正确：AWS WAF（Web Application Firewall）是专门设计用于保护Web应用程序免受常见攻击（如SQL注入）的服务。
通过在ALB前使用AWS WAF并配置适当的Web ACL（访问控制列表），可以有效检测和阻止SQL注入攻击。
B. Create an ALB listener rule to reply to SQL injections with a fixed response.
错误：ALB监听器规则主要用于路由流量或返回固定响应，但它无法检测或阻止SQL注入攻击。
ALB本身不具备防护SQL注入的功能。
C. Subscribe to AWS Shield Advanced to block all SQL injection attempts automatically.
错误：AWS Shield Advanced主要用于保护应用程序免受DDoS攻击，而不是SQL注入攻击。
SQL注入属于应用层攻击，AWS Shield Advanced无法直接处理此类问题。
D. Set up Amazon Inspector to block all SQL injection attempts automatically.
错误：Amazon Inspector是一个漏洞扫描服务，用于识别EC2实例中的安全问题，但它无法实时阻止SQL注入攻击。
Amazon Inspector的功能与SQL注入防护不直接相关。
3. 我的答案
A. Use AWS WAF in front of the ALB. Associate the appropriate web ACLs with AWS WAF.
