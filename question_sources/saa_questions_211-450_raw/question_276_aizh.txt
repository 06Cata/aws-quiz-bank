Question #276
A company has a multi-tier application deployed on several Amazon EC2 instances in an Auto Scaling group. An Amazon RDS for
Oracle instance is the application’ s data layer that uses Oracle-specific PL/SQL functions. Traffic to the application has been
steadily increasing. This is causing the EC2 instances to become overloaded and the RDS instance to run out of storage. The Auto
Scaling group does not have any scaling metrics and defines the minimum healthy instance count only. The company predicts that
traffic will continue to increase at a steady but unpredictable rate before leveling off.
What should a solutions architect do to ensure the system can automatically scale for the increased traffic? (Choose two.)

直到趋于平稳。

题目分析与解答
1. 考察的知识点
Amazon RDS存储自动扩展功能的配置。
Auto Scaling组的配置，包括使用适当的指标进行自动扩展。
如何处理流量增长导致的资源瓶颈问题。
2. 选项分析
A. Configure storage Auto Scaling on the RDS for Oracle instance. 正确。 RDS支持存储自动扩展功能，可以确保数据库在存储
空间不足时自动扩展。这是解决RDS存储不足问题的最佳实践。
B. Migrate the database to Amazon Aurora to use Auto Scaling storage. 错误。 虽然Amazon Aurora支持存储自动扩展，但题
目明确指出使用的是RDS for Oracle，并且依赖Oracle-specific PL/SQL功能。迁移到Aurora可能会导致功能不兼容。
C. Configure an alarm on the RDS for Oracle instance for low free storage space. 错误。 配置警报可以帮助监控存储空间，但
无法解决存储不足的问题。题目要求的是自动扩展，而不是仅仅监控。
D. Configure the Auto Scaling group to use the average CPU as the scaling metric. 正确。 使用平均CPU作为扩展指标是解决
EC2实例过载问题的最佳实践。随着流量增加，CPU使用率会升高，触发Auto Scaling扩展。
E. Configure the Auto Scaling group to use the average free memory as the scaling metric. 错误。 虽然内存是一个重要的指
标，但题目没有提到内存不足的问题。CPU使用率通常是更常用的扩展指标。
