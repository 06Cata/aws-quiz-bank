Question #336
A company hosts a multi-tier web application that uses an Amazon Aurora MySQL DB cluster for storage. The application tier is
hosted on Amazon EC2 instances. The company’s IT security guidelines mandate that the database credentials be encrypted and
rotated every 14 days.
What should a solutions architect do to meet this requirement with the LEAST operational effort?
一家公司托管了一个多层Web应用程序，该应用程序使用Amazon Aurora MySQL数据库集群进行存储。应用程序层托管在Amazon
EC2实例上。公司的IT安全指南要求数据库凭证必须加密并且每14天轮换一次。
解决方案架构师应该如何以最少的操作工作量满足这一要求？
A. Create a new AWS Key Management Service (AWS KMS) encryption key. Use AWS Secrets Manager to create a new secret
that uses the KMS key with the appropriate credentials. Associate the secret with the Aurora DB cluster. Configure a custom
rotation period of 14 days. Most Voted
创建一个新的AWS Key Management Service (AWS KMS)加密密钥。使用AWS Secrets Manager创建一个新密钥，该密钥使用
KMS密钥和适当的凭证。将该密钥与Aurora数据库集群关联。配置一个14天的自定义轮换周期。
B. Create two parameters in AWS Systems Manager Parameter Store: one for the user name as a string parameter and one that
uses the SecureString type for the password. Select AWS Key Management Service (AWS KMS) encryption for the password
parameter, and load these parameters in the application tier. Implement an AWS Lambda function that rotates the password
every 14 days.
在AWS Systems Manager Parameter Store中创建两个参数：一个用于用户名的字符串参数，另一个使用SecureString类型存储
密码。为密码参数选择AWS Key Management Service (AWS KMS)加密，并在应用程序层加载这些参数。实现一个AWS Lambda
函数，每14天轮换一次密码。
C. Store a file that contains the credentials in an AWS Key Management Service (AWS KMS) encrypted Amazon Elastic File
System (Amazon EFS) file system. Mount the EFS file system in all EC2 instances of the application tier. Restrict the access to
the file on the file system so that the application can read the file and that only super users can modify the file. Implement an
AWS Lambda function that rotates the key in Aurora every 14 days and writes new credentials into the file.
将包含凭证的文件存储在AWS Key Management Service (AWS KMS)加密的Amazon Elastic File System (Amazon EFS)文件系统
中。在应用程序层的所有EC2实例中挂载EFS文件系统。限制文件系统上的文件访问权限，使应用程序可以读取文件，而只有超
级用户可以修改文件。实现一个AWS Lambda函数，每14天轮换Aurora中的密钥，并将新凭证写入文件。
D. Store a file that contains the credentials in an AWS Key Management Service (AWS KMS) encrypted Amazon S3 bucket that
the application uses to load the credentials. Download the file to the application regularly to ensure that the correct credentials
are used. Implement an AWS Lambda function that rotates the Aurora credentials every 14 days and uploads these credentials
to the file in the S3 bucket.
将包含凭证的文件存储在AWS Key Management Service (AWS KMS)加密的Amazon S3桶中，供应用程序加载凭证。定期将文件
下载到应用程序，以确保使用正确的凭证。实现一个AWS Lambda函数，每14天轮换Aurora凭证，并将这些凭证上传到S3桶中
的文件。
题目分析与解答
1. 考察的知识点
Amazon Aurora数据库集群的集成与管理
AWS Secrets Manager的使用及凭证加密与轮换
AWS Key Management Service (KMS) 的加密功能
如何实现最少的操作工作量（Least Operational Effort）
2. 选项分析
A. Create a new AWS Key Management Service (AWS KMS) encryption key. Use AWS Secrets Manager to create a new
secret that uses the KMS key with the appropriate credentials. Associate the secret with the Aurora DB cluster. Configure a
custom rotation period of 14 days.
正确：AWS Secrets Manager是专门设计用于管理和轮换数据库凭证的服务。它支持与Amazon Aurora的直接集成，并允许设置自
定义轮换周期。
正确：使用AWS KMS加密密钥可以确保凭证的安全性。
正确：此选项完全符合“最少操作工作量”的要求，因为Secrets Manager自动处理凭证轮换，无需额外的手动干预。
B. Create two parameters in AWS Systems Manager Parameter Store: one for the user name as a string parameter and one
that uses the SecureString type for the password. Select AWS Key Management Service (AWS KMS) encryption for the
password parameter, and load these parameters in the application tier. Implement an AWS Lambda function that rotates
the password every 14 days.
错误：虽然Parameter Store可以存储加密的参数，但它不支持自动轮换数据库凭证。
错误：需要额外实现一个Lambda函数来轮换密码，这增加了操作工作量。
C. Store a file that contains the credentials in an AWS Key Management Service (AWS KMS) encrypted Amazon Elastic File
System (Amazon EFS) file system. Mount the EFS file system in all EC2 instances of the application tier. Restrict the access
to the file on the file system so that the application can read the file and that only super users can modify the file.
Implement an AWS Lambda function that rotates the key in Aurora every 14 days and writes new credentials into the file.
错误：使用EFS存储凭证文件并依赖Lambda函数进行轮换增加了复杂性。
错误：需要额外的操作来管理文件系统的挂载和权限，这不符合“最少操作工作量”的要求。
D. Store a file that contains the credentials in an AWS Key Management Service (AWS KMS) encrypted Amazon S3 bucket
that the application uses to load the credentials. Download the file to the application regularly to ensure that the correct
credentials are used. Implement an AWS Lambda function that rotates the Aurora credentials every 14 days and uploads
these credentials to the file in the S3 bucket.
错误：使用S3存储凭证文件并依赖Lambda函数进行轮换增加了复杂性。
错误：需要额外的操作来管理文件的下载和上传，这不符合“最少操作工作量”的要求。
