instance. Management says the application must be made more secure with the least amount of programming effort.
What should a solutions architect do to meet these requirements?

以最少的编程工作量提高安全性。
解决方案架构师应该怎么做以满足这些要求？

题目分析与解答
1. 考察的知识点
Amazon RDS数据库的安全性管理
AWS Secrets Manager的使用
凭证轮换的自动化
最小化编程工作量的解决方案
2. 选项分析
A. 使用 AWS Key Management Service (AWS KMS) 创建密钥。配置应用程序从 AWS KMS 加载数据库凭证。启用自动密钥轮
换。
错误：AWS KMS主要用于加密密钥管理，而不是直接存储和轮换数据库凭证。此选项需要额外的编程工作来实现凭证管理，不符
合“最少编程工作量”的要求。
B. 在 RDS for MySQL 数据库上为应用程序用户创建凭证，并将凭证存储在 AWS Secrets Manager 中。配置应用程序从 Secrets
Manager 加载数据库凭证。创建一个 AWS Lambda 函数以在 Secrets Manager 中轮换凭证。
错误：虽然使用AWS Secrets Manager是正确的方向，但手动创建AWS Lambda函数来轮换凭证增加了编程工作量，不符合“最少编
程工作量”的要求。
C. 在 RDS for MySQL 数据库上为应用程序用户创建凭证，并将凭证存储在 AWS Secrets Manager 中。配置应用程序从 Secrets
Manager 加载数据库凭证。使用 Secrets Manager 为 RDS for MySQL 数据库中的应用程序用户设置凭证轮换计划。
正确：AWS Secrets Manager支持自动凭证轮换功能，且无需额外编程工作。此选项符合“最少编程工作量”的要求，同时提高了安全
性。
D. 在 RDS for MySQL 数据库上为应用程序用户创建凭证，并将凭证存储在 AWS Systems Manager Parameter Store 中。配置应
用程序从 Parameter Store 加载数据库凭证。使用 Parameter Store 为 RDS for MySQL 数据库中的应用程序用户设置凭证轮换计
划。
错误：AWS Systems Manager Parameter Store不支持自动凭证轮换功能。此选项需要额外的编程工作来实现轮换，不符合“最少编
程工作量”的要求。
