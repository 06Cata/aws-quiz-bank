Question #239
A solutions architect needs to design a new microservice for a company’s application. Clients must be able to call an HTTPS
endpoint to reach the microservice. The microservice also must use AWS Identity and Access Management (IAM) to authenticate
calls. The solutions architect will write the logic for this microservice by using a single AWS Lambda function that is written in Go
1.x.
Which solution will deploy the function in the MOST operationally efficient way?
解决方案架构师需要为公司的应用程序设计一个新的微服务。客户端必须能够调用一个HTTPS端点以访问该微服务。该微服务还必
须使用AWS Identity and Access Management (IAM) 来验证调用。解决方案架构师将使用一个用Go 1.x编写的单一AWS Lambda函
数来编写该微服务的逻辑。
哪种解决方案能够以最具操作效率的方式部署该函数？
A. Create an Amazon API Gateway REST API. Configure the method to use the Lambda function. Enable IAM authentication on
the API. Most Voted

题目分析与解答
1. 考察的知识点
Amazon API Gateway的功能与配置，包括支持Lambda函数和IAM身份验证。
Lambda Function URL的使用场景及其支持的身份验证类型。
Amazon CloudFront与Lambda@Edge的功能及其适用场景。
CloudFront Functions的功能及其限制。
2. 选项分析
A. Create an Amazon API Gateway REST API. Configure the method to use the Lambda function. Enable IAM
authentication on the API.
正确：API Gateway可以直接集成Lambda函数，并支持IAM身份验证。这种方式是最常见且高效的解决方案，特别适合需要HTTPS
端点和IAM身份验证的场景。
B. Create a Lambda function URL for the function. Specify AWS_IAM as the authentication type.
错误：虽然Lambda Function URL支持IAM身份验证，但它的功能较为简单，缺乏API Gateway提供的高级功能（如流量管理、缓
存、监控等）。此外，题目要求“最具操作效率”的解决方案，API Gateway更适合这种场景。
C. Create an Amazon CloudFront distribution. Deploy the function to Lambda@Edge. Integrate IAM authentication logic
into the Lambda@Edge function.
错误：Lambda@Edge主要用于内容交付优化，而不是直接处理API请求。将IAM身份验证逻辑集成到Lambda@Edge中会增加复杂
性，不符合“最具操作效率”的要求。
D. Create an Amazon CloudFront distribution. Deploy the function to CloudFront Functions. Specify AWS_IAM as the
authentication type.
错误：CloudFront Functions主要用于轻量级的内容处理，不支持复杂的身份验证逻辑（如IAM）。此外，CloudFront Functions不
适合直接处理API请求。
