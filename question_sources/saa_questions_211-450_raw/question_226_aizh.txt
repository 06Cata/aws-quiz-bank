EC2 instance. The EC2 instance receives the raw data, transforms the raw data, and stores all the data in an Amazon S3 bucket. The

operational overhead.
Which combination of steps should a solutions architect take to meet these requirements? (Choose two.)

题目分析与解答
1. 考察的知识点
高可扩展性架构设计
使用托管服务减少运维开销
数据流处理与存储优化
2. 选项分析
A. Use AWS Glue to process the raw data in Amazon S3.
正确：AWS Glue 是一个完全托管的 ETL 服务，可以高效地处理和转换存储在 Amazon S3 中的数据。
符合题目要求的“高可扩展性”和“减少运维开销”。
B. Use Amazon Route 53 to route traffic to different EC2 instances.
错误：Amazon Route 53 是一个 DNS 服务，主要用于域名解析和流量路由，不适合处理数据流或扩展数据处理能力。
不符合题目要求的“高可扩展性”和“减少运维开销”。
C. Add more EC2 instances to accommodate the increasing amount of incoming data.
错误：增加 EC2 实例虽然可以扩展处理能力，但会增加运维开销，不符合题目要求的“减少运维开销”。
手动管理 EC2 实例的扩展性较低，且成本较高。
D. Send the raw data to Amazon Simple Queue Service (Amazon SQS). Use EC2 instances to process the data.
错误：虽然 Amazon SQS 可以缓冲数据流，但使用 EC2 实例处理数据仍然需要手动管理，增加了运维开销。
不符合题目要求的“减少运维开销”。
E. Use Amazon API Gateway to send the raw data to an Amazon Kinesis data stream. Configure Amazon Kinesis Data
Firehose to use the data stream as a source to deliver the data to Amazon S3.
正确：Amazon API Gateway 是一个托管服务，可以高效地接收数据请求并将数据发送到 Amazon Kinesis 数据流。
Amazon Kinesis Data Firehose 是一个完全托管的服务，可以将数据流直接传送到 Amazon S3，符合题目要求的“高可扩展性”和“减
少运维开销”。
