ElastiCache cluster and an App VPC for the application’s Amazon EC2 instances. Both VPCs are in the us-east-1 Region.
The solutions architect must implement a solution to provide the application’s EC2 instances with access to the ElastiCache cluster.
Which solution will meet these requirements MOST cost-effectively?

题目分析与解答
1. 考察的知识点
Amazon VPC对等连接（VPC Peering）的使用和配置。
跨VPC通信的路由表配置。
安全组规则的设置。
成本优化的解决方案选择。
2. 选项分析
A. Create a peering connection between the VPCs. Add a route table entry for the peering connection in both VPCs.
Configure an inbound rule for the ElastiCache cluster’s security group to allow inbound connection from the application’s
security group.
正确性分析：VPC对等连接是一个简单且成本最低的解决方案，可以实现两个VPC之间的直接通信。通过配置路由表和安全组规
则，EC2实例可以访问ElastiCache集群。
成本分析：VPC对等连接没有额外的中间资源（如Transit Gateway），因此成本最低。
结论：此选项正确。
B. Create a Transit VPC. Update the VPC route tables in the Cache VPC and the App VPC to route traffic through the
Transit VPC. Configure an inbound rule for the ElastiCache cluster's security group to allow inbound connection from the
application’s security group.
正确性分析：Transit VPC是一种复杂的网络架构，通常用于跨多个VPC或跨区域的通信。对于仅两个VPC之间的通信，这种方法过
于复杂。
成本分析：Transit VPC需要额外的资源（如VPN网关或第三方设备），成本较高。
结论：此选项错误。
C. Create a peering connection between the VPCs. Add a route table entry for the peering connection in both VPCs.
Configure an inbound rule for the peering connection’s security group to allow inbound connection from the application’s
security group.
正确性分析：VPC对等连接本身没有安全组，安全组是针对资源（如EC2实例或ElastiCache集群）配置的。因此此选项的描述不正
确。
成本分析：虽然使用了VPC对等连接，但安全组的配置方式错误，无法实现通信。
结论：此选项错误。
D. Create a Transit VPC. Update the VPC route tables in the Cache VPC and the App VPC to route traffic through the
Transit VPC. Configure an inbound rule for the Transit VPC’s security group to allow inbound connection from the
application’s security group.
正确性分析：Transit VPC的安全组规则配置方式不适用于ElastiCache集群，因为ElastiCache的安全组规则是直接针对集群资源配置
的。
成本分析：Transit VPC的成本较高，且不必要。
结论：此选项错误。
