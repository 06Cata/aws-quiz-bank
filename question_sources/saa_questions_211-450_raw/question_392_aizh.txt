Question #392
A company wants to deploy a new public web application on AWS. The application includes a web server tier that uses Amazon
EC2 instances. The application also includes a database tier that uses an Amazon RDS for MySQL DB instance.
The application must be secure and accessible for global customers that have dynamic IP addresses.
How should a solutions architect configure the security groups to meet these requirements?
一家公司希望在 AWS 上部署一个新的公共 Web 应用程序。该应用程序包括一个使用 Amazon EC2 实例的 Web 服务器层。该应用
程序还包括一个使用 Amazon RDS for MySQL DB 实例的数据库层。
该应用程序必须是安全的，并且能够被拥有动态 IP 地址的全球客户访问。
解决方案架构师应该如何配置安全组以满足这些要求？

题目分析与解答
1. 考察的知识点
安全组的配置与使用：如何正确配置安全组以确保应用的安全性和可访问性。
网络安全：如何限制数据库的访问权限，仅允许来自特定来源的流量。
全球访问：如何允许动态 IP 地址的客户访问公共 Web 应用。
2. 选项分析
A:
Web 服务器安全组允许来自 0.0.0.0/0 的端口 443 入站流量：正确，因为这是一个公共 Web 应用，全球客户需要访问，且客户的 IP
是动态的，因此需要允许所有 IP 地址访问。
数据库安全组允许来自 Web 服务器安全组的端口 3306 入站流量：正确，因为数据库应该仅允许来自 Web 服务器的流量，而不是
直接暴露给外部。
B:
Web 服务器安全组允许来自客户 IP 地址的端口 443 入站流量：错误，因为客户的 IP 地址是动态的，无法提前确定具体的 IP 范
围。
数据库安全组允许来自 Web 服务器安全组的端口 3306 入站流量：正确，但由于 Web 服务器的配置错误，整体方案不可行。
C:
Web 服务器安全组允许来自客户 IP 地址的端口 443 入站流量：错误，因为客户的 IP 地址是动态的，无法提前确定具体的 IP 范
围。
数据库安全组允许来自客户 IP 地址的端口 3306 入站流量：错误，因为数据库不应该直接暴露给客户，而是仅允许来自 Web 服务
器的流量。
D:
Web 服务器安全组允许来自 0.0.0.0/0 的端口 443 入站流量：正确，因为这是一个公共 Web 应用，全球客户需要访问。
数据库安全组允许来自 0.0.0.0/0 的端口 3306 入站流量：错误，因为数据库不应该暴露给所有 IP 地址，这会导致严重的安全问题。
