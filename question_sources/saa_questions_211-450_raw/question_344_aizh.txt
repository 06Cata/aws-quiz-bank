Question #344
A company has a Java application that uses Amazon Simple Queue Service (Amazon SQS) to parse messages. The application
cannot parse messages that are larger than 256 KB in size. The company wants to implement a solution to give the application the
ability to parse messages as large as 50 MB.
Which solution will meet these requirements with the FEWEST changes to the code?

2. 选项分析
A. Use the Amazon SQS Extended Client Library for Java to host messages that are larger than 256 KB in Amazon S3.
正确：Amazon SQS本身的消息大小限制为256 KB，但可以通过Amazon SQS Extended Client Library将大消息存储在Amazon S3
中，并在SQS中存储引用。这是官方推荐的解决方案，且对现有代码改动最少。
B. Use Amazon EventBridge to post large messages from the application instead of Amazon SQS.
错误：Amazon EventBridge并不是设计用于处理大消息的服务，且更改消息传递机制需要对代码进行较大改动。
C. Change the limit in Amazon SQS to handle messages that are larger than 256 KB.
错误：Amazon SQS的消息大小限制是硬性规定，无法通过配置更改。
D. Store messages that are larger than 256 KB in Amazon Elastic File System (Amazon EFS). Configure Amazon SQS to
reference this location in the messages.
错误：Amazon SQS无法直接与Amazon EFS集成，也无法在消息中引用EFS存储位置。这种解决方案不可行。
