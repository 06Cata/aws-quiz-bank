Question #441
A company hosts a multi-tier web application on Amazon Linux Amazon EC2 instances behind an Application Load Balancer. The
instances run in an Auto Scaling group across multiple Availability Zones. The company observes that the Auto Scaling group
launches more On-Demand Instances when the application's end users access high volumes of static web content. The company
wants to optimize cost.
What should a solutions architect do to redesign the application MOST cost-effectively?
一家公司在 Amazon Linux 的 Amazon EC2 实例上托管一个多层 Web 应用程序，这些实例位于一个 Application Load Balancer 后
面。实例运行在跨多个可用区的 Auto Scaling 组中。公司观察到，当应用程序的最终用户访问大量静态 Web 内容时，Auto Scaling
组会启动更多的 On-Demand Instances。公司希望优化成本。
解决方栾架构师应该如何重新设计应用程序以最具成本效益的方式？
A. Update the Auto Scaling group to use Reserved Instances instead of On-Demand Instances.
将 Auto Scaling 组更新为使用 Reserved Instances 而不是 On-Demand Instances。
B. Update the Auto Scaling group to scale by launching Spot Instances instead of On-Demand Instances.
将 Auto Scaling 组更新为通过启动 Spot Instances 而不是 On-Demand Instances 来进行扩展。
C. Create an Amazon CloudFront distribution to host the static web contents from an Amazon S3 bucket. Most Voted
创建一个 Amazon CloudFront 分发，将静态 Web 内容托管在 Amazon S3 bucket 中。
D. Create an AWS Lambda function behind an Amazon API Gateway API to host the static website contents.
创建一个 AWS Lambda 函数，并通过 Amazon API Gateway API 托管静态网站内容。

2. 选项分析
A. Update the Auto Scaling group to use Reserved Instances instead of On-Demand Instances.
错误：Reserved Instances可以降低成本，但它们是长期承诺的实例类型，无條动态应对械量的变化。
桮外，问题中提到的高械量主要是静态内容访问，使用Reserved Instances并不能解决根本问题。
B. Update the Auto Scaling group to scale by launching Spot Instances instead of On-Demand Instances.
部分桭确：Spot Instances确实可以显著降低成本，但它们的价栺和可用性不稳定，可能会影响应用的可靠性。
桮外，这种方條仍棺依赖EC2实例来处栬静态内容，而不是优化静态内容的交付方式。
C. Create an Amazon CloudFront distribution to host the static web contents from an Amazon S3 bucket.
桭确：将静态内容托管在Amazon S3中，并通过Amazon CloudFront分发，可以显著降低成本，同时提高性能和可扩展性。
CloudFront是专门设计用于优化静态内容交付的服务，能够减少对EC2实例的依赖。
D. Create an AWS Lambda function behind an Amazon API Gateway API to host the static website contents.
错误：虽棺Lambda和API Gateway可以用于托管动态内容，但它们并不是静态内容托管的最佳选择。
这种方條可能会增加复杂性和成本，而不是优化成本。
3. 我的答案
C. Create an Amazon CloudFront distribution to host the static web contents from an Amazon S3 bucket.
4. 官方答案
C. Create an Amazon CloudFront distribution to host the static web contents from an Amazon S3 bucket.
5. 与官方答案的比较
我的答栾与官方答栾一致。
原因：CloudFront和S3的组合是优化静态内容交付的最佳实践，能够显著降低成本并提高性能。
