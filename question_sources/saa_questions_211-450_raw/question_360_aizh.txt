Question #360
A company uses Amazon API Gateway to run a private gateway with two REST APIs in the same VPC. The BuyStock RESTful web
service calls the CheckFunds RESTful web service to ensure that enough funds are available before a stock can be purchased. The
company has noticed in the VPC flow logs that the BuyStock RESTful web service calls the CheckFunds RESTful web service over
the internet instead of through the VPC. A solutions architect must implement a solution so that the APIs communicate through
the VPC.
Which solution will meet these requirements with the FEWEST changes to the code?

题目分析与解答
1. 考察的知识点
本题考察的是如何在Amazon API Gateway中实现私有通信，确保REST API之间通过VPC内部通信而不是通过互联网。
涉及的知识点包括：VPC接口端点（Interface Endpoint）、网关端点（Gateway Endpoint）、API Gateway的私有集成以及服务间通
信的优化。
2. 选项分析
A. Add an X-API-Key header in the HTTP header for authorization.
错误：X-API-Key头仅用于授权和认证，无法解决API之间通过互联网通信的问题。
此选项与题目要求无关，因为它不会改变通信路径。
B. Use an interface endpoint.
正确：接口端点（Interface Endpoint）允许API Gateway通过VPC内部通信，而不是通过互联网。
接口端点是专门为私有通信设计的，且对代码改动最少。
C. Use a gateway endpoint.
错误：网关端点（Gateway Endpoint）主要用于S3和DynamoDB的私有通信，不适用于API Gateway。
此选项无法解决API Gateway的通信问题。
D. Add an Amazon Simple Queue Service (Amazon SQS) queue between the two REST APIs.
错误：SQS队列用于异步通信，虽然可以实现API之间的消息传递，但会增加复杂性且不符合题目要求的“最少代码改动”。
此选项不适合解决直接通信路径问题。
3. 我选的答案
B. Use an interface endpoint.
4. 官方答案
B. Use an interface endpoint.
5. 与官方答案的比较
我的答案与官方答案一致。
原因：接口端点是解决API Gateway私有通信的最佳选择，符合题目要求的“最少代码改动”。
