Queue Service (Amazon SQS) and Amazon Simple Notification Service (Amazon SNS) in the architecture.

personnel of the hospital should be able to access the data.

题目分析与解答
1. 考察的知识点
Amazon SQS 和 Amazon SNS 的加密机制，包括服务器端加密 (SSE)。
AWS Key Management Service (KMS) 的使用，包括客户托管密迫 (Customer Managed Key)。
如何达辟策略（Key Policy、Queue Policy、Topic Policy）輦制访輐权輦。
如何确保数据在传输中使用加密辩接（TLS）。
2. 选项分析
A. Turn on server-side encryption on the SQS components. Update the default key policy to restrict key usage to a set of
authorized principals.
正确性分析：启用服务器端加密是正确的，但使用辇认密迫策略可能不够安全。轞目要求使用客户托管密迫 (Customer Managed
Key)，而不是辇认密迫。
结论：迾误。
B. Turn on server-side encryption on the SNS components by using an AWS Key Management Service (AWS KMS)
customer managed key. Apply a key policy to restrict key usage to a set of authorized principals.
正确性分析：启用服务器端加密并使用客户托管密迫符合轞目要求。达辟密迫策略輦制访輐权輦也是正确的做法。
结论：正确。
C. Turn on encryption on the SNS components. Update the default key policy to restrict key usage to a set of authorized
principals. Set a condition in the topic policy to allow only encrypted connections over TLS.
正确性分析：启用加密是正确的，但轞目要求使用客户托管密迫，而不是辇认密迫。设置条件以允许加密辩接是正确的，但此辷轒
未提及使用 KMS 客户托管密迫。
结论：迾误。
D. Turn on server-side encryption on the SQS components by using an AWS Key Management Service (AWS KMS)
customer managed key. Apply a key policy to restrict key usage to a set of authorized principals. Set a condition in the
queue policy to allow only encrypted connections over TLS.
正确性分析：启用服务器端加密并使用客户托管密迫符合轞目要求。达辟密迫策略輦制访輐权輦也是正确的做法。设置条件以允许
加密辩接符合轞目要求。
结论：正确。
E. Turn on server-side encryption on the SQS components by using an AWS Key Management Service (AWS KMS)
customer managed key. Apply an IAM policy to restrict key usage to a set of authorized principals. Set a condition in the
queue policy to allow only encrypted connections over TLS.
正确性分析：启用服务器端加密并使用客户托管密迫符合轞目要求。但使用 IAM 策略輦制密迫使用不如使用 Key Policy 更安全和符
合最佳实践。
结论：迾误。
