Question #900
A company wants to use Amazon Elastic Container Service (Amazon ECS) to run its on-premises application in a hybrid
environment. The application currently runs on containers on premises.
The company needs a single container solution that can scale in an on-premises, hybrid, or cloud environment. The company must
run new application containers in the AWS Cloud and must use a load balancer for HTTP traffic.
Which combination of actions will meet these requirements? (Choose two.)

该公司需要一个单一的容器解决方案，可以在本地、混合或云环境中进行扩展。公司必须在 AWS 云中运行新的应用程序容器，并
且必须使用负载均衡器处理 HTTP 流量。

Correct Answer: AB
题目分析与解答
1. 考察的知识点
Amazon ECS的不同启动类型（Fargate、EC2、ECS Anywhere）。
负载均衡器的选择（Application Load Balancer vs Network Load Balancer）。
混合环境中运行容器的解决方案。
2. 选项分析
A. Set up an ECS cluster that uses the AWS Fargate launch type for the cloud application containers. Use an Amazon ECS
Anywhere external launch type for the on-premises application containers.
正确：AWS Fargate是无服务器的容器运行方式，适合云端应用程序容器。Amazon ECS Anywhere允许在本地运行容器并与AWS集
成，满足混合环境需求。
B. Set up an Application Load Balancer for cloud ECS services.
正确：Application Load Balancer支持HTTP/HTTPS流量，是运行Web应用程序的最佳选择。
C. Set up a Network Load Balancer for cloud ECS services.
错误：Network Load Balancer主要用于处理TCP/UDP流量，而题目明确要求处理HTTP流量，因此不适合。
D. Set up an ECS cluster that uses the AWS Fargate launch type. Use Fargate for the cloud application containers and the
on-premises application containers.
错误：AWS Fargate仅支持云端运行容器，无法直接用于本地环境。题目要求支持混合环境，因此此选项不符合要求。
E. Set up an ECS cluster that uses the Amazon EC2 launch type for the cloud application containers. Use Amazon ECS
Anywhere with an AWS Fargate launch type for the on-premises application containers.
错误：Amazon ECS Anywhere不支持AWS Fargate启动类型用于本地环境。此选项与技术限制不符。
