Question #836
A company serves its website by using an Auto Scaling group of Amazon EC2 instances in a single AWS Region. The website does
not require a database.
The company is expanding, and the company's engineering team deploys the website to a second Region. The company wants to
distribute traffic across both Regions to accommodate growth and for disaster recovery purposes. The solution should not serve
traffic from a Region in which the website is unhealthy.
Which policy or resource should the company use to meet these requirements?

决方案不应从网站不健康的区域提供流量。
公司应该使用哪种策略或资源来满足这些要求？

题目分析与解答
1. 考察的知识点
Amazon Route 53的路由策略，包括简单路由策略和多值答案路由策略。
跨区域流量分配和健康检查的实现。
负载均衡器的使用范围和限制。
2. 选项分析
A. An Amazon Route 53 simple routing policy
分析：简单路由策略无法实现跨区域流量分配，也不支持健康检查功能。因此无法满足题目中“分配流量”和“健康检查”的要求。
结论：错误。
B. An Amazon Route 53 multivalue answer routing policy
分析：多值答案路由策略支持跨区域流量分配，并且可以结合健康检查功能，确保流量不会被分配到不健康的区域。这完全符合题
目要求。
结论：正确。
C. An Application Load Balancer in one Region with a target group that specifies the EC2 instance IDs from both Regions
分析：Application Load Balancer无法跨区域工作。它只能在单个区域内分配流量，因此无法满足题目中跨区域流量分配的要求。
结论：错误。
D. An Application Load Balancer in one Region with a target group that specifies the IP addresses of the EC2 instances
from both Regions
分析：虽然可以通过IP地址指定跨区域的目标，但Application Load Balancer仍然无法跨区域工作。此外，这种配置复杂且不支持健
康检查功能。
结论：错误。
3. 我的答案
B. An Amazon Route 53 multivalue answer routing policy
4. 官方答案
B. An Amazon Route 53 multivalue answer routing policy
5. 与官方答案的比较
我的答案与官方答案一致。
理由：多值答案路由策略支持跨区域流量分配，并结合健康检查功能，完全满足题目要求。
