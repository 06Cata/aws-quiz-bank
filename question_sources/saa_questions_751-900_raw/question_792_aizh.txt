2. 选项分析
A. Configure an Amazon Cognito user pool for user authentication. Implement Amazon API Gateway REST APIs with a
Cognito authorizer.
正确性分析：Amazon Cognito用户池专门用于用户身份验证，能够高效地管理用户注册、登录和身份验证。API Gateway REST API
支持Cognito授权器，可以直接集成用户池进行认证，简化操作并提高效率。
结论：正确。
B. Configure an Amazon Cognito identity pool for user authentication. Implement Amazon API Gateway HTTP APIs with a
Cognito authorizer.
正确性分析：Amazon Cognito身份池主要用于临时访问AWS资源，而不是直接用于用户身份验证。HTTP API支持Cognito授权器，
但REST API更适合复杂的API场景。
结论：错误。
C. Configure an AWS Lambda function to handle user authentication. Implement Amazon API Gateway REST APIs with a
Lambda authorizer.
正确性分析：使用Lambda函数进行用户身份验证是可行的，但需要额外的开发和维护工作，增加了操作复杂性。相比之下，
Cognito用户池提供了更高的操作效率。
结论：错误。
D. Configure an IAM user to handle user authentication. Implement Amazon API Gateway HTTP APIs with an IAM
authorizer.
正确性分析：IAM用户不适合大规模的用户身份验证场景，且IAM授权器主要用于AWS服务间的访问控制，而不是面向外部用户的
认证。
结论：错误。
