Question #761
A company needs to use its on-premises LDAP directory service to authenticate its users to the AWS Management Console. The
directory service is not compatible with Security Assertion Markup Language (SAML).
Which solution meets these requirements?
一家公司需要使用其本地的 LDAP 目录服务来为用户认证访问 AWS Management Console。该目录服务不兼容 Security Assertion
Markup Language (SAML)。
满足这些需求的解决方案是什么？
A. Enable AWS IAM Identity Center (AWS Single Sign-On) between AWS and the on-premises LDAP.
启用 AWS IAM Identity Center (AWS Single Sign-On)，连接 AWS 和本地 LDAP。
B. Create an IAM policy that uses AWS credentials, and integrate the policy into LDAP.
创建一个使用 AWS 凭证的 IAM 策略，并将该策略集成到 LDAP 中。
C. Set up a process that rotates the IAM credentials whenever LDAP credentials are updated.
设置一个流程，在 LDAP 凭证更新时轮换 IAM 凭证。
D. Develop an on-premises custom identity broker application or process that uses AWS Security Token Service (AWS STS) to
get short-lived credentials. Most Voted
开发一个本地的自定义身份代理应用程序或流程，使用 AWS Security Token Service (AWS STS) 来获取短期凭证。

AWS Security Token Service (STS) 的使用
如何集成非SAML兼容的身份提供商与AWS
2. 选项分析
A. Enable AWS IAM Identity Center (AWS Single Sign-On) between AWS and the on-premises LDAP.
错误：AWS IAM Identity Center (AWS Single Sign-On) 需要身份提供商支持 SAML 或其他兼容协议，而题目明确指出 LDAP 不支持
SAML。
B. Create an IAM policy that uses AWS credentials, and integrate the policy into LDAP.
错误：IAM 策略用于定义权限，而不是身份认证。将 IAM 策略直接集成到 LDAP 中无法解决身份认证问题。
C. Set up a process that rotates the IAM credentials whenever LDAP credentials are updated.
错误：轮换 IAM 凭证无法解决身份认证问题，也无法实现用户通过 LDAP 认证后访问 AWS 管理控制台的需求。
D. Develop an on-premises custom identity broker application or process that uses AWS Security Token Service (AWS STS)
to get short-lived credentials.
正确：开发一个自定义身份代理应用程序可以充当桥梁，将 LDAP 的身份认证结果转换为 AWS STS 的短期凭证，从而允许用户访问
AWS 管理控制台。
