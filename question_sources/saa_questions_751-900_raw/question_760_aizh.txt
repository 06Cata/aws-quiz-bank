container needs 50 GB of storage available for temporary files. The infrastructure must be serverless.
Which solution meets these requirements with the LEAST operational overhead?

题目分析与解答
1. 考察的知识点
理解AWS的无服务器架构（Serverless Architecture）。
熟悉AWS Lambda、Amazon ECS（Fargate和EC2启动类型）、Amazon EFS、Amazon EBS和Amazon S3的使用场景和限制。
选择最少的操作开销（Least Operational Overhead）的解决方案。
2. 选项分析
A. Create an AWS Lambda function that uses the Docker container image with an Amazon S3 mounted volume that has
more than 50 GB of space.
错误：AWS Lambda不支持直接挂载Amazon S3作为存储卷。S3是对象存储，不能直接作为文件系统使用。
此外，Lambda的临时存储空间限制为512 MB，无法满足50 GB的需求。
B. Create an AWS Lambda function that uses the Docker container image with an Amazon Elastic Block Store (Amazon
EBS) volume that has more than 50 GB of space.
错误：AWS Lambda不支持挂载Amazon EBS卷。EBS卷只能与EC2实例或某些容器服务（如ECS）一起使用。
同样，Lambda的临时存储空间限制为512 MB，无法满足50 GB的需求。
C. Create an Amazon Elastic Container Service (Amazon ECS) cluster that uses the AWS Fargate launch type. Create a task
definition for the container image with an Amazon Elastic File System (Amazon EFS) volume. Create a service with that
task definition.
正确：AWS Fargate是无服务器的容器运行服务，符合题目要求的“serverless”。
Amazon EFS可以提供超过50 GB的存储，并且可以挂载到容器中作为文件系统使用。
操作开销较低，因为Fargate不需要管理底层的EC2实例。
D. Create an Amazon Elastic Container Service (Amazon ECS) cluster that uses the Amazon EC2 launch type with an
Amazon Elastic Block Store (Amazon EBS) volume that has more than 50 GB of space. Create a task definition for the
container image. Create a service with that task definition.
错误：虽然Amazon EC2启动类型可以使用EBS卷满足50 GB的存储需求，但EC2启动类型需要管理底层的EC2实例，增加了操作开
销。
不符合题目要求的“serverless”架构。
3. 我的选择
答案：C
4. 官方答案
答案：C
5. 与官方答案比较
我的答案与官方答案一致。
理由：选项C使用了AWS Fargate（无服务器架构）和Amazon EFS（满足50 GB存储需求），并且操作开销最低，完全符合题目要
求。
