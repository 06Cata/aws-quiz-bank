Question #861
A company wants to relocate its on-premises MySQL database to AWS. The database accepts regular imports from a client-facing
application, which causes a high volume of write operations. The company is concerned that the amount of traffic might be
causing performance issues within the application.
How should a solutions architect design the architecture on AWS?
一家公司希望将其本地 MySQL 数据库迁移到 AWS。该数据库接受来自面向客户的应用程序的定期导入，这会导致大量写操作。公
司担心流量可能会导致应用程序性能问题。
解决方案架构师应该如何在 AWS 上设计架构？

Correct Answer: A
正确答案：A
题目分析与解答
1. 考察的知识点
Amazon RDS的使用与配置，包括存储类型选择（Provisioned IOPS SSD vs General Purpose SSD）。
性能监控工具（Amazon CloudWatch）的应用。
数据库性能优化策略，特别是针对高写入量的场景。
2. 选项分析
A. Provision an Amazon RDS for MySQL DB instance with Provisioned IOPS SSD storage. Monitor write operation metrics
by using Amazon CloudWatch. Adjust the provisioned IOPS if necessary.
正确：Provisioned IOPS SSD存储专为高性能、高写入量场景设计，能够提供稳定的IOPS性能。
正确：使用Amazon CloudWatch监控写操作指标是标准的性能监控方法，可以帮助识别瓶颈并调整IOPS配置。
此选项完全符合题目要求，解决了高写入量可能导致的性能问题。
B. Provision an Amazon RDS for MySQL DB instance with General Purpose SSD storage. Place an Amazon ElastiCache
cluster in front of the DB instance. Configure the application to query ElastiCache instead.
错误：General Purpose SSD存储不适合高写入量场景，性能可能无法满足需求。
错误：ElastiCache主要用于缓存读操作，而题目关注的是写操作性能问题，ElastiCache无法解决写入瓶颈。
C. Provision an Amazon DocumentDB (with MongoDB compatibility) instance with a memory optimized instance type.
Monitor Amazon CloudWatch for performance-related issues. Change the instance class if necessary.
错误：Amazon DocumentDB是为文档数据库设计的，与MySQL不兼容，无法满足题目要求。
错误：题目明确提到的是MySQL数据库，而不是DocumentDB。
D. Provision an Amazon Elastic File System (Amazon EFS) file system in General Purpose performance mode. Monitor
Amazon CloudWatch for IOPS bottlenecks. Change to Provisioned Throughput performance mode if necessary.
错误：Amazon EFS是文件存储服务，与关系型数据库（如MySQL）的需求不匹配。
错误：题目关注的是数据库性能优化，而不是文件存储性能。
