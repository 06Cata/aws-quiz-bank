Question #751
A solutions architect is designing an AWS Identity and Access Management (IAM) authorization model for a company's AWS
account. The company has designated five specific employees to have full access to AWS services and resources in the AWS
account.
The solutions architect has created an IAM user for each of the five designated employees and has created an IAM user group.
Which solution will meet these requirements?
解决方案架构师正在为公司AWS账户设计一个AWS Identity and Access Management (IAM)授权模型。公司指定了五名特定员工拥
有对AWS账户中AWS服务和资源的完全访问权限。
解决方案架构师已经为这五名指定员工中的每一位创建了一个IAM用户，并创建了一个IAM用户组。
哪种解决方案可以满足这些要求？
A. Attach the AdministratorAccess resource-based policy to the IAM user group. Place each of the five designated employee

题目分析与解答
1. 考察的知识点
IAM用户和用户组的权限管理。
基于身份的策略（Identity-based policy）与基于资源的策略（Resource-based policy）的区别。
AdministratorAccess策略的使用场景。
2. 选项分析
A. Attach the AdministratorAccess resource-based policy to the IAM user group. Place each of the five designated
employee IAM users in the IAM user group.
错误：资源基于策略（Resource-based policy）是直接附加到资源上的，而不是附加到用户组或用户。IAM用户组无法使用资源基
于策略。
B. Attach the SystemAdministrator identity-based policy to the IAM user group. Place each of the five designated
employee IAM users in the IAM user group.
错误：AWS中没有名为“SystemAdministrator”的预定义策略。此选项无效。
C. Attach the AdministratorAccess identity-based policy to the IAM user group. Place each of the five designated
employee IAM users in the IAM user group.
正确：AdministratorAccess是AWS提供的预定义策略，授予完全访问权限。将此策略附加到IAM用户组，并将用户添加到该组，可
以满足题目要求。
D. Attach the SystemAdministrator resource-based policy to the IAM user group. Place each of the five designated
employee IAM users in the IAM user group.
错误：资源基于策略（Resource-based policy）无法附加到IAM用户组。此外，“SystemAdministrator”不是AWS的预定义策略。
