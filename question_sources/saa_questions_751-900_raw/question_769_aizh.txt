Question #769
A company hosts an application used to upload files to an Amazon S3 bucket. Once uploaded, the files are processed to extract
metadata, which takes less than 5 seconds. The volume and frequency of the uploads varies from a few files each hour to
hundreds of concurrent uploads. The company has asked a solutions architect to design a cost-effective architecture that will meet
these requirements.
What should the solutions architect recommend?

题目分析与解答
1. 考察的知识点
Amazon S3事件通知机制
AWS Lambda的无服务器计算能力
如何设计成本优化的架构
2. 选项分析
A. Configure AWS CloudTrail trails to log S3 API calls. Use AWS AppSync to process the files.
错误：CloudTrail主要用于记录API调用以进行审计和监控，而不是实时处理文件。
错误：AWS AppSync是用于构建GraphQL API的服务，不适合处理文件上传后的实时处理需求。
B. Configure an object-created event notification within the S3 bucket to invoke an AWS Lambda function to process the
files.
正确：S3支持事件通知，可以在对象创建时触发Lambda函数。
正确：AWS Lambda是无服务器计算服务，能够快速处理文件，且成本低，适合处理不确定的负载。
C. Configure Amazon Kinesis Data Streams to process and send data to Amazon S3. Invoke an AWS Lambda function to
process the files.
错误：Kinesis Data Streams主要用于实时流数据处理，而题目中没有提到流数据的需求。
错误：将数据发送到S3后再处理文件会增加复杂性和成本，不符合题目要求的成本优化。
D. Configure an Amazon Simple Notification Service (Amazon SNS) topic to process the files uploaded to Amazon S3.
Invoke an AWS Lambda function to process the files.
错误：SNS主要用于消息分发，而不是直接处理S3事件通知。
错误：S3事件通知可以直接触发Lambda函数，无需通过SNS中转。

B
