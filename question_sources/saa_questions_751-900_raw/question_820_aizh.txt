Question #820
A company is migrating a legacy application from an on-premises data center to AWS. The application relies on hundreds of cron
jobs that run between 1 and 20 minutes on different recurring schedules throughout the day.
The company wants a solution to schedule and run the cron jobs on AWS with minimal refactoring. The solution must support
running the cron jobs in response to an event in the future.
Which solution will meet these requirements?

段以不同的周期性计划运行，运行时间从1分钟到20分钟不等。

哪种解决方案可以满足这些要求？

题目分析与解答
1. 考察的知识点
Amazon EventBridge Scheduler的功能和使用场景。
AWS Fargate的无服务器容器运行能力。
如何在AWS上实现定时任务（cron jobs）的迁移和运行。
对比AWS Lambda、AWS Batch、AWS Step Functions等服务的适用场景。
2. 选项分析
A. Create a container image for the cron jobs. Use Amazon EventBridge Scheduler to create a recurring schedule. Run the
cron job tasks as AWS Lambda functions.
分析：
Amazon EventBridge Scheduler确实可以创建周期性计划，适合运行定时任务。
AWS Lambda适合运行短时间任务，但题目提到的cron jobs运行时间可能长达20分钟，可能超出Lambda的最大执行时间限制（15
分钟）。
因此，Lambda不适合运行这些cron jobs。
结论：错误。
B. Create a container image for the cron jobs. Use AWS Batch on Amazon Elastic Container Service (Amazon ECS) with a
scheduling policy to run the cron jobs.
分析：
AWS Batch适合运行批处理任务，但它主要用于处理大规模并行计算任务，而不是简单的定时任务。
题目要求的是运行cron jobs，而不是复杂的批处理任务，因此AWS Batch不是最佳选择。
结论：错误。
C. Create a container image for the cron jobs. Use Amazon EventBridge Scheduler to create a recurring schedule. Run the
cron job tasks on AWS Fargate.
分析：
Amazon EventBridge Scheduler可以轻松创建周期性计划，适合运行定时任务。
AWS Fargate是一种无服务器容器运行服务，支持运行长时间任务，适合运行这些cron jobs。
此选项满足题目要求，且支持未来事件触发。
结论：正确。
D. Create a container image for the cron jobs. Create a workflow in AWS Step Functions that uses a Wait state to run the
cron jobs at a specified time. Use the RunTask action to run the cron job tasks on AWS Fargate.
分析：
AWS Step Functions适合构建复杂的工作流，但题目并未要求复杂的工作流管理。
使用Wait状态来实现定时任务的运行效率较低，且不如EventBridge Scheduler直接支持定时任务。
虽然可以使用RunTask在AWS Fargate上运行任务，但整体方案复杂度较高，不符合“最小重构”的要求。
结论：错误。
