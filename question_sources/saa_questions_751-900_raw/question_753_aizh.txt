Question #753
A company has a nightly batch processing routine that analyzes report files that an on-premises file system receives daily through
SFTP. The company wants to move the solution to the AWS Cloud. The solution must be highly available and resilient. The solution
also must minimize operational effort.
Which solution meets these requirements?
一家公司有一个夜间批处理例程，用于分析通过 SFTP 接收的本地文件系统每天收到的报告文件。 该公司希望将解决方案迁移到
AWS 云。解决方案必须高度可用且具有弹性。 同时解决方案还必须尽量减少运营工作量。
哪种解决方案满足这些要求？

AWS Transfer for SFTP的使用
Amazon Elastic File System (EFS)的特性和用途
高可用性和弹性架构设计

2. 选项分析
A. Deploy AWS Transfer for SFTP and an Amazon Elastic File System (Amazon EFS) file system for storage. Use an Amazon
EC2 instance in an Auto Scaling group with a scheduled scaling policy to run the batch operation.
分析：
AWS Transfer for SFTP是一个托管服务，能够轻松地接收SFTP文件，减少了管理SFTP服务器的操作工作量。
Amazon EFS是一个高可用、弹性且可扩展的文件存储服务，适合存储批处理文件。
使用Auto Scaling组和计划扩展策略可以确保批处理操作的高可用性和弹性。
此选项符合题目要求：高可用性、弹性和最小化操作工作量。
结论：正确。
B. Deploy an Amazon EC2 instance that runs Linux and an SFTP service. Use an Amazon Elastic Block Store (Amazon EBS)
volume for storage. Use an Auto Scaling group with the minimum number of instances and desired number of instances
set to 1.
分析：
运行SFTP服务需要手动管理EC2实例和SFTP服务器，增加了操作工作量。
Amazon EBS是单实例存储，无法提供高可用性和弹性。
Auto Scaling组设置最小实例数和期望实例数为1，无法实现弹性扩展。
此选项不符合题目要求：高可用性和最小化操作工作量。
结论：错误。
C. Deploy an Amazon EC2 instance that runs Linux and an SFTP service. Use an Amazon Elastic File System (Amazon EFS)
file system for storage. Use an Auto Scaling group with the minimum number of instances and desired number of
instances set to 1.
分析：
运行SFTP服务需要手动管理EC2实例和SFTP服务器，增加了操作工作量。
Amazon EFS提供高可用性和弹性，但手动管理SFTP服务仍然是一个问题。
Auto Scaling组设置最小实例数和期望实例数为1，无法实现弹性扩展。
此选项部分符合题目要求，但未能最小化操作工作量。
结论：错误。
D. Deploy AWS Transfer for SFTP and an Amazon S3 bucket for storage. Modify the application to pull the batch files from
Amazon S3 to an Amazon EC2 instance for processing. Use an EC2 instance in an Auto Scaling group with a scheduled
scaling policy to run the batch operation.
分析：
AWS Transfer for SFTP是托管服务，减少了操作工作量。
Amazon S3是高可用、弹性且成本低的存储服务，但需要修改应用程序以支持从S3拉取文件，增加了开发工作量。
使用Auto Scaling组和计划扩展策略可以确保批处理操作的高可用性和弹性。
此选项符合高可用性和弹性要求，但增加了开发工作量，未能完全最小化操作工作量。
结论：错误。
