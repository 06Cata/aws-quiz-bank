authorization techniques that are available on AWS. The company also wants to implement a serverless architecture for
authorization and authentication that has low login latency.
The solution must integrate with the web application and serve web content globally. The application currently has a small user
base, but the company expects the application's user base to increase.
Which solution will meet these requirements?

题目分析与解答
1. 考察的知识点
Amazon Cognito的身份验证功能。
Lambda@Edge的边缘计算能力，用于授权。
Amazon CloudFront的全球内容分发能力。
如何设计低延迟、可扩展的无服务器架构。
2. 选项分析
A. Configure Amazon Cognito for authentication. Implement Lambda@Edge for authorization. Configure Amazon
CloudFront to serve the web application globally.
正确：Amazon Cognito提供身份验证服务，支持小型用户群并可扩展到更大的用户群。
正确：Lambda@Edge可以在CloudFront边缘节点执行授权逻辑，减少延迟。
正确：Amazon CloudFront可以全球分发内容，满足全球用户访问需求。
此选项完全符合题目要求。
B. Configure AWS Directory Service for Microsoft Active Directory for authentication. Implement AWS Lambda for
authorization. Use an Application Load Balancer to serve the web application globally.
错误：AWS Directory Service适用于企业内部目录服务集成，不适合小型用户群或全球分发场景。
错误：AWS Lambda虽然可以实现授权，但无法在边缘节点执行，可能导致较高的延迟。
错误：Application Load Balancer无法实现全球分发，无法满足题目要求。
C. Configure Amazon Cognito for authentication. Implement AWS Lambda for authorization. Use Amazon S3 Transfer
Acceleration to serve the web application globally.
正确：Amazon Cognito适合身份验证需求。
错误：AWS Lambda无法在边缘节点执行授权逻辑，可能导致较高的延迟。
错误：Amazon S3 Transfer Acceleration主要用于加速文件上传和下载，不适合直接分发动态Web内容。
D. Configure AWS Directory Service for Microsoft Active Directory for authentication. Implement Lambda@Edge for
authorization. Use AWS Elastic Beanstalk to serve the web application globally.
错误：AWS Directory Service不适合小型用户群或全球分发场景。
正确：Lambda@Edge可以在边缘节点执行授权逻辑。
错误：AWS Elastic Beanstalk无法实现全球分发，无法满足题目要求。

A
