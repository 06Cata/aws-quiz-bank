题目分析与解答
1. 考察的知识点
本题考察的是AWS服务的调度能力、无服务器计算（Serverless）和容器化技术的应用。
重点关注AWS Fargate、Amazon ECS、Amazon EventBridge Scheduler、AWS Lambda、EC2 Spot Instances、AWS DataSync等服务
的特点和适用场景。
2. 选项分析
A. Create a container for the job. Schedule the job to run as an AWS Fargate task on an Amazon Elastic Container Service
(Amazon ECS) cluster by using Amazon EventBridge Scheduler.
正确性分析：AWS Fargate是一个无服务器容器运行服务，支持运行长时间任务且不会中断。结合Amazon ECS和EventBridge
Scheduler，可以可靠地调度任务并确保其运行完成。
优点：Fargate任务是无状态的，且不会受到底层基础设施的中断影响，非常适合运行关键任务。
结论：此选项正确。
B. Configure the job to run in an AWS Lambda function. Create a scheduled rule in Amazon EventBridge to invoke the
Lambda function.
正确性分析：AWS Lambda适合运行短时间任务（最长15分钟），而题目要求任务至少运行1小时，因此Lambda不适合此场景。
缺点：Lambda的运行时间限制无法满足任务需求。
结论：此选项错误。
C. Configure an Auto Scaling group of Amazon EC2 Spot Instances that run Amazon Linux. Configure a crontab entry on
the instances to run the analysis.
正确性分析：EC2 Spot Instances价格低廉，但可能会被中断，无法保证任务的连续性。而题目明确要求任务不能容忍中断。
缺点：Spot Instances的中断风险使其不适合运行关键任务。
结论：此选项错误。
D. Configure an AWS DataSync task to run the job. Configure a cron expression to run the task on a schedule.
正确性分析：AWS DataSync主要用于数据迁移和同步，而不是运行计算任务。此选项与题目要求不符。
缺点：DataSync不适合运行分析任务。
结论：此选项错误。
