Question #758
A company wants to deploy its containerized application workloads to a VPC across three Availability Zones. The company needs a
solution that is highly available across Availability Zones. The solution must require minimal changes to the application.
Which solution will meet these requirements with the LEAST operational overhead?
一家公司希望将其容器化的应用工作负载部署到跨三个可用区的VPC中。该公司需要一个在可用区之间高度可用的解决方案。该解
决方案必须对应用程序的更改要求最小。
哪个解决方案能够以最少的运营开销满足这些需求？
A. Use Amazon Elastic Container Service (Amazon ECS). Configure Amazon ECS Service Auto Scaling to use target tracking
scaling. Set the minimum capacity to 3. Set the task placement strategy type to spread with an Availability Zone attribute.

题目分析与解答
1. 考察的知识点
容器化应用的部署与管理。
跨可用区的高可用架构设计。
自动扩展与任务放置策略。
最小化应用程序的修改。
2. 选项分析
A. Use Amazon Elastic Container Service (Amazon ECS). Configure Amazon ECS Service Auto Scaling to use target tracking
scaling. Set the minimum capacity to 3. Set the task placement strategy type to spread with an Availability Zone attribute.
分析：
Amazon ECS 是 AWS 提供的托管容器服务，支持跨可用区的高可用性。
通过配置 Service Auto Scaling 和任务放置策略，可以确保任务在多个可用区之间均匀分布，从而实现高可用性。
最小容量设置为 3，确保每个可用区至少有一个任务运行。
此选项对应用程序的修改需求最小，且操作开销较低。
正确选项。
B. Use Amazon Elastic Kubernetes Service (Amazon EKS) self-managed nodes. Configure Application Auto Scaling to use
target tracking scaling. Set the minimum capacity to 3.
分析：
Amazon EKS 是 AWS 提供的托管 Kubernetes 服务，支持容器化应用的部署。
使用自管理节点需要额外的运维工作，例如节点的管理和更新，增加了操作开销。
虽然可以实现高可用性，但与 ECS 相比，EKS 的复杂性更高，且需要更多的应用程序配置修改。
错误选项。
C. Use Amazon EC2 Reserved Instances. Launch three EC2 instances in a spread placement group. Configure an Auto
Scaling group to use target tracking scaling. Set the minimum capacity to 3.
分析：
EC2 Reserved Instances 提供固定的计算资源，但不具备容器化应用的管理功能。
Spread placement group 可以实现跨可用区的分布，但需要手动管理容器部署，增加了操作开销。
此选项需要对应用程序进行较多修改，不符合“最小化应用程序修改”的要求。
错误选项。
D. Use an AWS Lambda function. Configure the Lambda function to connect to a VPC. Configure Application Auto Scaling
to use Lambda as a scalable target. Set the minimum capacity to 3.
分析：
AWS Lambda 是无服务器计算服务，适合事件驱动的应用程序，但不适合容器化工作负载的部署。
虽然可以连接到 VPC，但 Lambda 的扩展机制与容器化应用的需求不完全匹配。
此选项需要对应用程序进行较大修改，不符合题目要求。
错误选项。
