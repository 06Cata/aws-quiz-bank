题目分析与解答
1. 考察的知识点
本题考察的是AWS迁移服务相关的知识，包括AWS Migration Hub、AWS Application Discovery Service、AWS Systems Manager等
服务的功能和用途。
重点在于如何收集本地服务器和工作负载的使用和配置数据，以便规划迁移到AWS。
2. 选项分析
A. Set the home AWS Region in AWS Migration Hub. Use AWS Systems Manager to collect data about the on-premises
servers.
错误：AWS Systems Manager主要用于管理和操作服务器（如运行命令、补丁管理等），而不是专门用于收集迁移相关的使用和配
置数据。
虽然可以通过Systems Manager的一些功能收集部分数据，但它不是规划迁移的最佳工具。
B. Set the home AWS Region in AWS Migration Hub. Use AWS Application Discovery Service to collect data about the on-
premises servers.
正确：AWS Application Discovery Service是专门设计用于收集本地服务器和工作负载的使用和配置数据的服务。
它与AWS Migration Hub集成，可以帮助用户规划迁移到AWS的过程。
C. Use the AWS Schema Conversion Tool (AWS SCT) to create the relevant templates. Use AWS Trusted Advisor to collect
data about the on-premises servers.
错误：AWS Schema Conversion Tool (AWS SCT)主要用于数据库迁移的架构转换，与收集服务器使用和配置数据无关。
AWS Trusted Advisor是一个优化工具，用于检查AWS账户的最佳实践，而不是用于收集本地服务器数据。
D. Use the AWS Schema Conversion Tool (AWS SCT) to create the relevant templates. Use AWS Database Migration
Service (AWS DMS) to collect data about the on-premises servers.
错误：AWS Database Migration Service (AWS DMS)主要用于迁移数据库，而不是用于收集本地服务器的使用和配置数据。
AWS SCT和AWS DMS都与数据库迁移相关，与本题要求的服务器数据收集不匹配。
