Question #780
A company has hired an external vendor to perform work in the company’s AWS account. The vendor uses an automated tool that
is hosted in an AWS account that the vendor owns. The vendor does not have IAM access to the company’s AWS account. The
company needs to grant the vendor access to the company’s AWS account.
Which solution will meet these requirements MOST securely?
一家公司聘请了一家外部供应商在公司的AWS账户中执行工作。 供应商使用一个自动化工具，该工具托管在供应商拥有的AWS账户
中。 供应商没有对公司AWS账户的IAM访问权限。 公司需要授予供应商访问公司AWS账户的权限。
哪种解决方案可以最安全地满足这些要求？
A. Create an IAM role in the company’s account to delegate access to the vendor’s IAM role. Attach the appropriate IAM
policies to the role for the permissions that the vendor requires. Most Voted

的权限。
B. Create an IAM user in the company’s account with a password that meets the password complexity requirements. Attach the

正确答案：A
题目分析与解答
1. 考察的知识点
IAM角色与跨账户访问的实现
安全性最佳实践：最小权限原则
如何安全地授权外部供应商访问AWS资源
2. 选项分析
A. Create an IAM role in the company’s account to delegate access to the vendor’s IAM role. Attach the appropriate IAM
policies to the role for the permissions that the vendor requires.
正确：这是AWS推荐的跨账户访问最佳实践。通过创建IAM角色并使用信任策略，允许供应商的AWS账户中的IAM角色进行访问。
这样可以避免直接创建用户或共享密码，确保安全性。
附加的IAM策略可以限制供应商的权限，符合最小权限原则。
B. Create an IAM user in the company’s account with a password that meets the password complexity requirements.
Attach the appropriate IAM policies to the user for the permissions that the vendor requires.
错误：创建IAM用户并共享密码不符合安全性最佳实践。密码可能被泄露或滥用，且无法实现自动化工具的跨账户访问。
C. Create an IAM group in the company’s account. Add the automated tool’s IAM user from the vendor account to the
group. Attach the appropriate IAM policies to the group for the permissions that the vendor requires.
错误：IAM组仅适用于同一账户内的用户管理，无法跨账户使用。供应商的IAM用户无法直接加入公司账户的IAM组。
D. Create an IAM user in the company’s account that has a permission boundary that allows the vendor’s account. Attach
the appropriate IAM policies to the user for the permissions that the vendor requires.
错误：权限边界用于限制IAM用户或角色的权限，但无法实现跨账户访问。创建IAM用户仍然涉及密码管理问题，不符合安全性最
佳实践。

A
