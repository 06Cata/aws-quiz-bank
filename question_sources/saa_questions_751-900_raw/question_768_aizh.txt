Question #768
A company has a business-critical application that runs on Amazon EC2 instances. The application stores data in an Amazon
DynamoDB table. The company must be able to revert the table to any point within the last 24 hours.
Which solution meets these requirements with the LEAST operational overhead?
一家公司有一个对业务至关重要的应用程序，该应用程序运行在 Amazon EC2 实例上。该应用程序将数据存储在一个 Amazon
DynamoDB 表中。公司必须能够将表恢复到过去 24 小时内的任何时间点。
哪种解决方案以最少的运营开销满足这些要求？
A. Configure point-in-time recovery for the table. Most Voted
为表配置点时间恢复功能。

使用 AWS Lambda 函数每小时对表进行一次按需备份。

an Amazon S3 bucket.
在表上启用流功能，以捕获过去 24 小时内表的所有更改日志。将流的副本存储在一个 Amazon S3 存储桶中。
题目分析与解答
1. 考察的知识点
本题考察的是Amazon DynamoDB的备份与恢复功能，特别是点时间恢复（Point-in-Time Recovery, PITR）的使用。
Correct Answer: A
正确答案：A

社区投票分布
A (100%)
还涉及AWS Backup、Lambda自动化备份以及DynamoDB Streams的功能理解。
2. 选项分析
A. Configure point-in-time recovery for the table.
正确：DynamoDB的点时间恢复功能允许用户将表恢复到过去任何时间点（最多35天内），并且操作简单，几乎不需要额外的管理
工作。
这是满足题目要求的最佳解决方案，因为它提供了自动化的恢复能力，且操作开销最低。
B. Use AWS Backup for the table.
错误：虽然AWS Backup可以备份DynamoDB表，但它需要手动配置备份计划，并且无法直接支持“恢复到过去24小时内的任意时间
点”的需求。
相比点时间恢复，AWS Backup的操作开销更高。
C. Use an AWS Lambda function to make an on-demand backup of the table every hour.
错误：使用Lambda函数进行按需备份需要额外的开发和维护工作，且备份频率（每小时）无法完全满足“恢复到任意时间点”的要
求。
操作开销较高，不是最佳选择。
D. Turn on streams on the table to capture a log of all changes to the table in the last 24 hours. Store a copy of the stream
in an Amazon S3 bucket.
错误：DynamoDB Streams主要用于捕获表的更改记录，适合实时数据处理或事件驱动架构，但不支持直接恢复到过去的任意时间
点。
需要额外的开发工作来实现数据恢复，操作开销较高。
3. 我的答案
A. Configure point-in-time recovery for the table.
4. 官方答案
A. Configure point-in-time recovery for the table.
5. 与官方答案的比较
我的答案与官方答案一致。
原因：点时间恢复功能是满足题目要求的最佳解决方案，具有最低的操作开销，并且完全支持恢复到过去24小时内的任意时间点。
