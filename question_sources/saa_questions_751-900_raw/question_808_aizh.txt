definition uses need to be scanned for Common Vulnerabilities and Exposures (CVEs). New container images that are created also
need to be scanned.
Which solution will meet these requirements with the FEWEST changes to the workloads?

题目分析与解答
1. 考察的知识点
Amazon Elastic Container Registry (ECR) 的功能，包括容器镜像存储和扫描。
如何实现容器镜像的自动化扫描以检测漏洞 (CVEs)。
AWS服务的集成与最佳实践。
2. 选项分析
A. Use Amazon Elastic Container Registry (Amazon ECR) as a private image repository to store the container images.
Specify scan on push filters for the ECR basic scan.
正确：Amazon ECR 是 AWS 提供的专用容器镜像存储服务，支持基本扫描功能，可以在镜像推送时自动扫描以检测 CVEs。
最少的工作量：直接使用 ECR 的内置功能，无需额外配置或开发。
B. Store the container images in an Amazon S3 bucket. Use Amazon Macie to scan the images. Use an S3 Event
Notification to initiate a Macie scan for every event with an s3:ObjectCreated:Put event type.
错误：Amazon Macie 是用于数据分类和敏感数据保护的服务，不适用于扫描容器镜像的漏洞。
不符合需求：无法检测 CVEs。
C. Deploy the workloads to Amazon Elastic Kubernetes Service (Amazon EKS). Use Amazon Elastic Container Registry
(Amazon ECR) as a private image repository. Specify scan on push filters for the ECR enhanced scan.
错误：虽然 ECR 支持增强扫描，但题目中并未要求迁移到 Amazon EKS，迁移工作量较大。
不符合“最少更改”的要求：迁移到 EKS 会增加复杂性。
D. Store the container images in an Amazon S3 bucket that has versioning enabled. Configure an S3 Event Notification for
s3:ObjectCreated:* events to invoke an AWS Lambda function. Configure the Lambda function to initiate an Amazon
Inspector scan.
错误：Amazon Inspector 主要用于扫描 EC2 实例和容器运行时的漏洞，而不是直接扫描存储在 S3 中的容器镜像。
复杂性高：需要额外配置 S3 事件通知和 Lambda 函数。
