AWS Regions. The company wants to improve the availability and performance of the application. The company also wants to

resources. Static IP addresses are required.
What should a solutions architect recommend to accomplish this?

题目分析与解答
1. 考察的知识点
全球用户访问优化：如何提高应用的可用性和性能。
安全性：保护应用免受常见的网络攻击（如SQL注入、跨站脚本攻击等）。
静态IP地址需求：如何实现静态IP地址的分配。
AWS服务的组合使用：AWS Global Accelerator、AWS WAF、ALB/NLB、CloudFront等服务的功能和适用场景。
2. 选项分析
A. Put the EC2 instances behind Network Load Balancers (NLBs) in each Region. Deploy AWS WAF on the NLBs. Create an
accelerator using AWS Global Accelerator and register the NLBs as endpoints.
正确性分析：NLB支持静态IP地址，AWS Global Accelerator可以提供全球加速和静态IP地址，但NLB不支持直接与AWS WAF集成。
AWS WAF通常与ALB或CloudFront集成，因此此选项在安全性方面不符合要求。
结论：错误。
B. Put the EC2 instances behind Application Load Balancers (ALBs) in each Region. Deploy AWS WAF on the ALBs. Create
an accelerator using AWS Global Accelerator and register the ALBs as endpoints.
正确性分析：ALB支持与AWS WAF集成，可以保护应用免受常见的网络攻击。AWS Global Accelerator提供静态IP地址和全球加速功
能。此选项满足所有需求，包括性能优化、安全性和静态IP地址。
结论：正确。
C. Put the EC2 instances behind Network Load Balancers (NLBs) in each Region. Deploy AWS WAF on the NLBs. Create an
Amazon CloudFront distribution with an origin that uses Amazon Route 53 latency-based routing to route requests to the
NLBs.
正确性分析：NLB不支持与AWS WAF直接集成，因此安全性需求无法满足。此外，CloudFront分发和Route 53的延迟路由可以优化
性能，但无法提供静态IP地址。
结论：错误。
D. Put the EC2 instances behind Application Load Balancers (ALBs) in each Region. Create an Amazon CloudFront
distribution with an origin that uses Amazon Route 53 latency-based routing to route requests to the ALBs. Deploy AWS
WAF on the CloudFront distribution.
正确性分析：CloudFront可以与AWS WAF集成，提供安全性保护，但CloudFront无法提供静态IP地址。此外，Route 53的延迟路由
可以优化性能，但不满足静态IP地址的需求。
结论：错误。
