Question #827
A company is planning to deploy its application on an Amazon Aurora PostgreSQL Serverless v2 cluster. The application will
receive large amounts of traffic. The company wants to optimize the storage performance of the cluster as the load on the
application increases.
Which solution will meet these requirements MOST cost-effectively?
一家公司计划将其应用程序部署在一个 Amazon Aurora PostgreSQL Serverless v2 集群上。该应用程序将接收大量流量。公司希望
在应用程序负载增加时优化集群的存储性能。
哪种解决方案将最具成本效益地满足这些需求？
A. Configure the cluster to use the Aurora Standard storage configuration.
将集群配置为使用 Aurora Standard 存储配置。
B. Configure the cluster storage type as Provisioned IOPS.
将集群存储类型配置为 Provisioned IOPS。
C. Configure the cluster storage type as General Purpose.
将集群存储类型配置为 General Purpose。
D. Configure the cluster to use the Aurora I/O-Optimized storage configuration. Most Voted
将集群配置为使用 Aurora I/O-Optimized 存储配置。
题目分析与解答
1. 考察的知识点
本题考察的是 Amazon Aurora 的存储配置选项，尤其是如何优化存储性能以应对高负载，同时保持成本效益。
Correct Answer: D
正确答案：D
Community vote distribution
社区投票分布
D (81%)
Other
涉及 Aurora 的不同存储类型，包括 Aurora Standard、Provisioned IOPS、General Purpose 和 Aurora I/O-Optimized。
2. 选项分析
A. Configure the cluster to use the Aurora Standard storage configuration.
错误：Aurora Standard 是默认的存储配置，但它在高负载情况下可能无法提供最佳性能。
虽然成本较低，但不适合需要高性能存储的场景。
B. Configure the cluster storage type as Provisioned IOPS.
错误：Provisioned IOPS 提供了高性能存储，但成本较高，不是最具成本效益的解决方案。
适用于需要极高 IOPS 的场景，但 Aurora Serverless v2 通常不需要这种配置。
C. Configure the cluster storage type as General Purpose.
错误：General Purpose 存储类型适合中等性能需求，但在高负载情况下可能无法满足性能要求。
成本较低，但性能不如 Aurora I/O-Optimized。
D. Configure the cluster to use the Aurora I/O-Optimized storage configuration.
正确：Aurora I/O-Optimized 是专门设计用于高性能和高负载场景的存储配置。
它消除了 I/O 请求成本，同时提供了更高的性能，是最具成本效益的解决方案。
3. 我的答案
我的选择是：D
4. 官方答案
官方答案是：D

原因：Aurora I/O-Optimized 存储配置在高负载场景下提供了最佳性能，同时消除了 I/O 请求成本，是最具成本效益的选择。
