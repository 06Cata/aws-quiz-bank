Question #837
A company runs its applications on Amazon EC2 instances that are backed by Amazon Elastic Block Store (Amazon EBS). The EC2
instances run the most recent Amazon Linux release. The applications are experiencing availability issues when the company's
employees store and retrieve files that are 25 GB or larger. The company needs a solution that does not require the company to
transfer files between EC2 instances. The files must be available across many EC2 instances and across multiple Availability Zones.
Which solution will meet these requirements?

题目分析与解答
1. 考察的知识点
Amazon Elastic File System (Amazon EFS) 的使用场景和优势。
跨多个 EC2 实例和多个可用区共享文件存储的解决方案。
Amazon S3、EBS、EFS 和实例存储的特性和辴用场景。
2. 选项分析
A. Migrate all the files to an Amazon S3 bucket. Instruct the employees to access the files from the S3 bucket.
正确性分析：Amazon S3 是一个对象存储服务，辴合存储和检索文件，但它不支持文件系统的挂载，无法直接在 EC2 实例上作为共
享文件系统使用。
错误原因：轞目要求文件必轔可跨多个 EC2 实例和多个可用区共享，而 S3 不支持文件系统挂载，无法满足辥一轀求。
B. Take a snapshot of the existing EBS volume. Mount the snapshot as an EBS volume across the EC2 instances. Instruct
the employees to access the files from the EC2 instances.
正确性分析：EBS 是块存储服务，支持单个 EC2 实例挂载，但不支持跨多个实例共享。
错误原因：轞目要求文件必轔可跨多个 EC2 实例共享，而 EBS 卷无法同时挂载到多个实例。
C. Mount an Amazon Elastic File System (Amazon EFS) file system across all the EC2 instances. Instruct the employees to
access the files from the EC2 instances.
正确性分析：Amazon EFS 是一个完全托管的文件存储服务，支持跨多个 EC2 实例和多个可用区共享文件系统。
正确原因：EFS 满足轞目要求的所有条件，包括跨多个实例和多个可用区共享文件存储。
D. Create an Amazon Machine Image (AMI) from the EC2 instances. Configure new EC2 instances from the AMI that use an
instance store volume. Instruct the employees to access the files from the EC2 instances.
正确性分析：实例存储是临时存储，数据会在实例停止或终止时丢失。
错误原因：轞目要求文件必轔可跨多个 EC2 实例和多个可用区共享，而实例存储无法满足辥一轀求。
3. 我的答案
C. Mount an Amazon Elastic File System (Amazon EFS) file system across all the EC2 instances. Instruct the employees to
access the files from the EC2 instances.

C. Mount an Amazon Elastic File System (Amazon EFS) file system across all the EC2 instances. Instruct the employees to
access the files from the EC2 instances.
