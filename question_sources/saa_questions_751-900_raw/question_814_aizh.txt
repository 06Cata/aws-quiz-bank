Message Queuing Protocol (AMQP) to communicate with a message queue. The data center cannot scale fast enough to meet the
company’s expanding business needs. The company wants to migrate the workloads to AWS.
Which solution will meet these requirements with the LEAST operational overhead?

题目分析与解答
1. 考察的知识点
容器化应用的迁移与管理：Amazon ECS 和 Amazon EKS 的使用场景。
消息队列服务：Amazon MQ 和 Amazon SQS 的适用场景。
高可用性与操作开销：如何选择最少的运维工作量的解决方案。
2. 选项分析
A. Migrate the container application to Amazon Elastic Container Service (Amazon ECS). Use Amazon Simple Queue
Service (Amazon SQS) to retrieve the messages.
Amazon ECS 是一种托管的容器服务，适合运行容器化应用，但它不直接支持 Kubernetes 的工作负载迁移。
AMQP 是一种高级消息队列协议，而 Amazon SQS 不支持 AMQP 协议，因此无法满足消息队列的需求。
此选项错误。
B. Migrate the container application to Amazon Elastic Kubernetes Service (Amazon EKS). Use Amazon MQ to retrieve the
messages.
Amazon EKS 是托管的 Kubernetes 服务，能够直接支持 Kubernetes 工作负载的迁移。
Amazon MQ 支持 AMQP 协议，能够满足消息队列的需求。
此选项正确。
C. Use highly available Amazon EC2 instances to run the application. Use Amazon MQ to retrieve the messages.
使用 Amazon EC2 实例运行应用程序需要手动管理基础设施，增加了运维开销。
虽然 Amazon MQ 支持 AMQP，但此选项的操作开销较高，不符合“最少运维开销”的要求。
此选项错误。
D. Use AWS Lambda functions to run the application. Use Amazon Simple Queue Service (Amazon SQS) to retrieve the
messages.
AWS Lambda 是无服务器计算服务，但它不适合运行需要持续运行的容器化应用程序。
Amazon SQS 不支持 AMQP 协议，因此无法满足消息队列的需求。
此选项错误。
