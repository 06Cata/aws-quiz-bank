Question #93
Which of the following is a recommended design principle of the AWS Well-Architected Framework?
以下哪项是 AWS Well-Architected Framework 的推荐设计原则？
A. Reduce downtime by making infrastructure changes infrequently and in large increments.
通过不频繁且大幅度的基础设施变更来减少停机时间。
B. Invest the time to configure infrastructure manually.
花时间手动配置基础设施。
C. Learn to improve from operational failures. Most Voted
从操作失败中学习并改进。
D. Use monolithic application design for centralization.
使用单体应用设计进行集中化。
题目分析与解答
1. 考察的知识点
本题考察的是AWS Well-Architected Framework的设计原则，特别是如何设计可靠、弹性和高效的云架构。
2. 选项分析
A. Reduce downtime by making infrastructure changes infrequently and in large increments.
错误：AWS Well-Architected Framework推荐的是小规模、频繁的变更，而不是大规模、低频率的变更。这样可以降低风险并更快
地发现问题。
B. Invest the time to configure infrastructure manually.
错误：AWS推荐使用自动化工具（如AWS CloudFormation）来配置基础设施，而不是手动操作。手动配置容易出错且效率低下。
C. Learn to improve from operational failures.
Correct Answer: C
正确答案：C
