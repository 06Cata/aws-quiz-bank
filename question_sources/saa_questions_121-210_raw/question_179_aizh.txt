题目分析与解答
1. 考察的知识点
本题考察的是如何使用 AWS Systems Manager Parameter Store 安全存储敏感信息（如数据库用户名和密码），以及如何通过 IAM
角色和权限管理来确保安全访问。
涉及的服务包括：AWS Systems Manager Parameter Store、IAM 角色与策略、AWS Key Management Service (KMS)。
2. 选项分析
A. Create an IAM role that has read access to the Parameter Store parameter. Allow Decrypt access to an AWS Key
Management Service (AWS KMS) key that is used to encrypt the parameter. Assign this IAM role to the EC2 instance.
正确：此选项描述了最佳实践。通过创建 IAM 角色并赋予其读取 Parameter Store 参数的权限，同时允许解密 KMS 加密的参数，
确保 EC2 实例能够安全访问敏感信息。
IAM 角色是分配给 EC2 实例的最佳方式，因为它避免了硬编码凭证，并且可以动态管理权限。
B. Create an IAM policy that allows read access to the Parameter Store parameter. Allow Decrypt access to an AWS Key
Management Service (AWS KMS) key that is used to encrypt the parameter. Assign this IAM policy to the EC2 instance.
错误：IAM 策略不能直接分配给 EC2 实例。IAM 策略需要附加到 IAM 角色或用户，而 EC2 实例只能通过 IAM 角色来获取权限。
C. Create an IAM trust relationship between the Parameter Store parameter and the EC2 instance. Specify Amazon RDS as
a principal in the trust policy.
错误：IAM 信任关系的主体应该是 EC2 实例或相关的 IAM 角色，而不是 Amazon RDS。此选项的描述不符合 AWS 的权限管理逻
辑。
D. Create an IAM trust relationship between the DB instance and the EC2 instance. Specify Systems Manager as a principal
in the trust policy.
错误：DB 实例和 EC2 实例之间不需要直接的 IAM 信任关系。Systems Manager Parameter Store 的权限管理是通过 IAM 角色和策
略来实现的，而不是通过信任关系。
