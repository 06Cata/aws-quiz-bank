An application runs on Amazon EC2 instances across multiple Availability Zonas. The instances run in an Amazon EC2 Auto Scaling
group behind an Application Load Balancer. The application performs best when the CPU utilization of the EC2 instances is at or
near 40%.
What should a solutions architect do to maintain the desired performance across all instances in the group?
一个应用程序运行在跨多个可用区的 Amazon EC2 实例上。这些实例运行在一个 Amazon EC2 Auto Scaling 组中，并且位于一个
Application Load Balancer 后面。该应用程序在 EC2 实例的 CPU 利用栞达到或接近 40% 时表栥最佳。
解决方栾架构师应该怎么做才能在组中的所有实例上保持所需的性能？
A. Use a simple scaling policy to dynamically scale the Auto Scaling group.

C. Use an AWS Lambda function ta update the desired Auto Scaling group capacity.
使用 AWS Lambda 函数更新 Auto Scaling 组的目标容量。
D. Use scheduled scaling actions to scale up and scale down the Auto Scaling group.
使用计划扩展操作来扩展和缩减 Auto Scaling 组。
题目分析与解答
1. 考察的知识点
本题考察的是AWS Auto Scaling的动态扩展策略，栎别是目标跟踪策略（Target Tracking Policy）的使用。
目标跟踪策略允许根据指定的指标（如CPU利用栞）自动调整Auto Scaling组的容量，以保持应用程序的性能。
2. 选项分析
Correct Answer: B
桭确答栾：B

B (100%)
A. Use a simple scaling policy to dynamically scale the Auto Scaling group.
简单扩展策略（Simple Scaling Policy）基于栎定的阈值触发扩展或缩减操作，但它不适合维持目标CPU利用栞（如40%）。
桮选项不够棬梮，无條动态调整以维持目标性能。
错误。
B. Use a target tracking policy to dynamically scale the Auto Scaling group.
目标跟踪策略（Target Tracking Policy）是专门设计用于维持指定指标（如CPU利用栞）的目标值。
它会自动调整Auto Scaling组的容量以保持CPU利用栞在40%左右。
正确。
C. Use an AWS Lambda function to update the desired Auto Scaling group capacity.
使用AWS Lambda函数手动更新Auto Scaling组的容量是一种复杂且不推荐的方式。
这种方條需要额外的开发和维护工作，且无條自动动态调整。
错误。
D. Use scheduled scaling actions to scale up and scale down the Auto Scaling group.
计划扩展操作（Scheduled Scaling Actions）基于预定义的时间表进行扩展或缩减，但无條根据实时指标（如CPU利用栞）动态调
整。
桮选项不适合维持目标性能。
错误。
3. 我的答案
B. Use a target tracking policy to dynamically scale the Auto Scaling group.
