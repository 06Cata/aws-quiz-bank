front-end code and the backend code as possible. However, the company wants to break the application into smaller applications.
A different team will manage each application. The company needs a highly scalable solution that minimizes operational
overhead.
Which solution will meet these requirements?

题目分析与解答
1. 考察的知识点
应用迁移到AWS的最佳实践。
微服务架构的实现方式。
高可扩展性和低运维开销的解决方案。
AWS服务的适用场景：AWS Lambda、Amazon API Gateway、Amazon EC2、Amazon ECS。
2. 选项分析
A. Host the application on AWS Lambda. Integrate the application with Amazon API Gateway.
正确性分析：虽然AWS Lambda和API Gateway可以实现无服务器架构并减少运维开销，但题目提到要保留现有的前端和后端代码，
并将应用拆分为多个小型应用。Lambda适合运行小型函数，而不是托管复杂的应用程序。此外，Lambda可能需要重构代码以适应
其事件驱动模型。
结论：错误。
B. Host the application with AWS Amplify. Connect the application to an Amazon API Gateway API that is integrated with
AWS Lambda.
正确性分析：AWS Amplify主要用于构建和部署前端应用程序，而不是托管后端代码。虽然可以通过API Gateway和Lambda连接后
端，但这并不符合题目要求的“保留现有前端和后端代码”的需求。
结论：错误。
C. Host the application on Amazon EC2 instances. Set up an Application Load Balancer with EC2 instances in an Auto
Scaling group as targets.
正确性分析：EC2实例可以托管现有的前端和后端代码，并通过负载均衡器实现高可用性。然而，EC2实例的管理和维护需要较高的
运维开销，不符合题目要求的“最小化运维开销”。
结论：错误。
D. Host the application on Amazon Elastic Container Service (Amazon ECS). Set up an Application Load Balancer with
Amazon ECS as the target.
正确性分析：Amazon ECS支持容器化应用程序，可以将单体应用拆分为多个容器，每个容器由不同的团队管理。ECS与Application
Load Balancer集成后可以实现高可扩展性，同时容器化减少了运维开销，符合题目要求。
结论：正确。
3. 我选的答案
D. Host the application on Amazon Elastic Container Service (Amazon ECS). Set up an Application Load Balancer with
Amazon ECS as the target.
4. 官方答案
D. Host the application on Amazon Elastic Container Service (Amazon ECS). Set up an Application Load Balancer with
Amazon ECS as the target.
5. 与官方答案比较
比较结果：我的答案与官方答案一致。
原因分析：Amazon ECS提供了容器化解决方案，支持将单体应用拆分为多个小型应用，并通过负载均衡实现高可扩展性，同时减
少运维开销，完全符合题目要求。
