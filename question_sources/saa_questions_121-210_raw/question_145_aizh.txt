PHP and uses a MySQL database. The analytics software, the web server that provides PHP, and the database server are all hosted
on the EC2 instance. The application is showing signs of performance degradation during busy times and is presenting 5xx errors.
The company needs to make the application scale seamlessly.
Which solution will meet these requirements MOST cost-effectively?

题目分析与解答
1. 考察的知识点
Amazon RDS 和 Amazon Aurora 的使用场景和区别。
EC2 实例的扩展和负载均衡策略。
Auto Scaling 和 Spot Fleet 的成本优化能力。
Application Load Balancer 的功能和作用。
2. 选项分析
A. Migrate the database to an Amazon RDS for MySQL DB instance. Create an AMI of the web application. Use the AMI to
launch a second EC2 On-Demand Instance. Use an Application Load Balancer to distribute the load to each EC2 instance.
正确性：将数据库迁移到 Amazon RDS for MySQL 是一个合理的选择，可以提高数据库性能和可扩展性。
问题：虽然使用 Application Load Balancer 可以分配负载，但使用 On-Demand Instances 的成本较高，缺乏成本优化。
结论：此选项正确性较高，但不是最具成本效益的解决方案。
B. Migrate the database to an Amazon RDS for MySQL DB instance. Create an AMI of the web application. Use the AMI to
launch a second EC2 On-Demand Instance. Use Amazon Route 53 weighted routing to distribute the load across the two
EC2 instances.
正确性：将数据库迁移到 Amazon RDS for MySQL 是合理的选择。
问题：使用 Route 53 加权路由分配负载不如 Application Load Balancer 高效，且仍然使用 On-Demand Instances，成本较高。
结论：此选项正确性较低，且不符合“无缝扩展”的要求。
C. Migrate the database to an Amazon Aurora MySQL DB instance. Create an AWS Lambda function to stop the EC2
instance and change the instance type. Create an Amazon CloudWatch alarm to invoke the Lambda function when CPU
utilization surpasses 75%.
正确性：将数据库迁移到 Amazon Aurora MySQL 是合理的选择，可以提高性能。
问题：使用 Lambda 和 CloudWatch 来动态调整实例类型虽然可以优化性能，但无法实现无缝扩展，且操作复杂。
结论：此选项正确性较低，不符合“无缝扩展”的要求。
D. Migrate the database to an Amazon Aurora MySQL DB instance. Create an AMI of the web application. Apply the AMI
to a launch template. Create an Auto Scaling group with the launch template Configure the launch template to use a Spot
Fleet. Attach an Application Load Balancer to the Auto Scaling group.
正确性：将数据库迁移到 Amazon Aurora MySQL 是合理的选择，可以提高性能。
优势：使用 Auto Scaling Group 和 Spot Fleet 可以实现无缝扩展，同时优化成本。Application Load Balancer 可以高效分配负载。
结论：此选项是最具成本效益且符合“无缝扩展”要求的解决方案。
