A company is running a batch application on Amazon EC2 instances. The application consists of a backend with multiple Amazon
RDS databases. The application is causing a high number of reads on the databases. A solutions architect must reduce the number
of database reads while ensuring high availability.
What should the solutions architect do to meet this requirement?

2. 选项分析
A. Add Amazon RDS read replicas.
正确：Amazon RDS读副本可以有效减少主数据库的读取压力，同时提供高可用性。读副本是专门设计用于处理高读取负载的场
景。
符合题目要求：减少数据库读取次数并确保高可用性。
B. Use Amazon ElastiCache for Redis.
错误：虽然Redis可以作为缓存层减少数据库读取压力，但题目没有提到应用程序支持Redis或需要复杂的缓存逻辑。
Redis适用于需要高性能缓存的场景，但它并不直接提供数据库的高可用性功能。
C. Use Amazon Route 53 DNS caching.
错误：Route 53 DNS缓存主要用于优化域名解析速度，与减少数据库读取次数无关。
不符合题目要求：无法直接解决数据库读取压力问题。
D. Use Amazon ElastiCache for Memcached.
错误：Memcached与Redis类似，可以作为缓存层减少数据库读取压力，但题目没有提到应用程序支持Memcached。
Memcached也不直接提供数据库的高可用性功能。
3. 我选的答案
A. Add Amazon RDS read replicas.
4. 官方答案
A. Add Amazon RDS read replicas.
