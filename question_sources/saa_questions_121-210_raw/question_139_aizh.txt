A reporting team receives files each day in an Amazon S3 bucket. The reporting team manually reviews and copies the files from
this initial S3 bucket to an analysis S3 bucket each day at the same time to use with Amazon QuickSight. Additional teams are
starting to send more files in larger sizes to the initial S3 bucket.
The reporting team wants to move the files automatically analysis S3 bucket as the files enter the initial S3 bucket. The reporting
team also wants to use AWS Lambda functions to run pattern-matching code on the copied data. In addition, the reporting team
wants to send the data files to a pipeline in Amazon SageMaker Pipelines.
What should a solutions architect do to meet these requirements with the LEAST operational overhead?

A. 创建一个 Lambda 函数以将文件复制到分析 S3 bucket。为分析 S3 bucket 创建一个 S3 事件通知。配置 Lambda 和
SageMaker Pipelines 作为事件通知的目标。将事件类型配置为 s3:ObjectCreated:Put。
错误：此选项要求手动创建 Lambda 函数来复制文件，这增加了操作开销，不符合题目要求的“最少操作开销”。
错误：S3事件通知无法直接支持多个目标（如Lambda和SageMaker Pipelines），需要通过EventBridge来实现多目标支持。
B. 创建一个 Lambda 函数以将文件复制到分析 S3 bucket。配置分析 S3 bucket 发送事件通知到 Amazon EventBridge (Amazon
CloudWatch Events)。在 EventBridge (CloudWatch Events) 中配置一个 ObjectCreated 规则。将 Lambda 和 SageMaker
Pipelines 配置为该规则的目标。
错误：此选项仍然要求手动创建 Lambda 函数来复制文件，增加了操作开销。
正确：EventBridge可以支持多目标，但手动复制文件的方式不符合“最少操作开销”的要求。
C. 在 S3 buckets 之间配置 S3 复制。为分析 S3 bucket 创建一个 S3 事件通知。配置 Lambda 和 SageMaker Pipelines 作为事件

错误：S3事件通知无法直接支持多个目标（如Lambda和SageMaker Pipelines），需要通过EventBridge来实现多目标支持。
正确：S3复制可以自动将文件从初始S3 bucket复制到分析S3 bucket，减少了操作开销。
