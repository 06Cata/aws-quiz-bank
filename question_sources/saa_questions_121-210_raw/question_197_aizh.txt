changes while moving the application. The AWS application environment should be highly available.
Which combination of actions should the company take to meet these requirements? (Choose two.)

题目分析与解答
1. 考察的知识点
应用程序迁移到AWS的策略，包括重托管（Rehosting）、重新平台化（Replatforming）、重构（Refactoring）。
高可用架构设计，包括多可用区（Multi-AZ）部署。
AWS服务的使用，包括AWS Elastic Beanstalk、Amazon RDS、AWS Database Migration Service (DMS)。
2. 选项分析
A. Refactor the application as serverless with AWS Lambda functions running .NET Core.
错误：重构应用程序为无服务器架构需要大量开发工作，与题目要求的“最小化开发更改”不符。
B. Rehost the application in AWS Elastic Beanstalk with the .NET platform in a Multi-AZ deployment.
正确：AWS Elastic Beanstalk支持.NET平台，可以快速迁移应用程序，同时支持多可用区部署以实现高可用性。
C. Replatform the application to run on Amazon EC2 with the Amazon Linux Amazon Machine Image (AMI).
错误：题目提到应用程序运行在Windows Server上，而Amazon Linux AMI不支持Windows环境，迁移到Linux需要大量开发工作。
D. Use AWS Database Migration Service (AWS DMS) to migrate from the Oracle database to Amazon DynamoDB in a
Multi-AZ deployment.
错误：DynamoDB是NoSQL数据库，与现有的Oracle数据库（关系型数据库）架构不兼容，迁移到DynamoDB需要对应用程序进行
大量修改。
E. Use AWS Database Migration Service (AWS DMS) to migrate from the Oracle database to Oracle on Amazon RDS in a
Multi-AZ deployment.
正确：Amazon RDS支持Oracle数据库，可以使用AWS DMS进行迁移，同时支持多可用区部署以实现高可用性。
