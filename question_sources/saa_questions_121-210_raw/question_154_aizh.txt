A company needs to save the results from a medical trial to an Amazon S3 repository. The repository must allow a few scientists to
add new files and must restrict all other users to read-only access. No users can have the ability to modify or delete any files in the
repository. The company must keep every file in the repository for a minimum of 1 year after its creation date.
Which solution will meet these requirements?

A. Use S3 Object Lock in governance mode with a legal hold of 1 year.

题目分析与解答
1. 考察的知识点
Amazon S3 Object Lock功能，包括治理模式和合规模式。
S3存储桶策略的权限管理。
数据保留策略和防止删除或修改的机制。
2. 选项分析
A. 使用 S3 Object Lock 的治理模式，并设置 1 年的法律保留。
治理模式允许用户（具有特定权限的用户）绕过锁定并删除或修改对象，因此不符合题目要求“没有用户可以修改或删除文件”。
虽然设置了法律保留，但治理模式无法完全防止文件被修改或删除。
此选项错误。
B. 使用 S3 Object Lock 的合规模式，并设置 365 天的保留期。
合规模式是最严格的模式，任何用户（包括管理员）都无法删除或修改对象，完全符合题目要求。
设置365天的保留期确保文件至少保存1年，符合题目要求。
此选项正确。
C. 使用 IAM 角色限制所有用户删除或更改 S3 bucket 中的对象。使用 S3 bucket policy 仅允许 IAM 角色。
虽然可以通过IAM角色和S3存储桶策略限制用户权限，但无法确保文件至少保存1年，也无法完全防止管理员修改或删除文件。
此选项不符合题目要求。
此选项错误。
D. 配置 S3 bucket 每次添加对象时触发 AWS Lambda 函数。配置该函数以跟踪保存对象的哈希值，以便可以相应地标记被修改的
对象。
此选项涉及复杂的自定义解决方案，但无法防止文件被删除或修改。
题目要求的是防止修改或删除，而不是标记被修改的对象。
此选项错误。
3. 我选的答案
B. 使用 S3 Object Lock 的合规模式，并设置 365 天的保留期。
4. 官方答案
B. 使用 S3 Object Lock 的合规模式，并设置 365 天的保留期。
5. 与官方答案比较
我的答案与官方答案一致。
原因：合规模式是最严格的模式，完全符合题目要求，确保文件至少保存1年且无法被修改或删除。
