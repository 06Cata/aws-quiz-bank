题目分析与解答
1. 考察的知识点
Amazon S3 Object Lock的使用，包括治理模式和合规模式。
加密数据的方式，包括SSE-S3和AWS KMS的加密选项。
如何实现加密密钥的自动轮换。
如何满足数据不可删除和不可覆盖的合规性要求。
2. 选项分析
A. Store the documents in Amazon S3. Use S3 Object Lock in governance mode.
治理模式允许管理员绕过锁定规则，这不符合题目要求的“确保文件不能被覆盖或删除”的严格合规性要求。
因此此选项错误。
B. Store the documents in Amazon S3. Use S3 Object Lock in compliance mode.
合规模式确保文件不能被覆盖或删除，即使是管理员也无法绕过锁定规则。
此选项符合题目要求，因此正确。
C. Use server-side encryption with Amazon S3 managed encryption keys (SSE-S3). Configure key rotation.
SSE-S3使用Amazon S3托管的加密密钥，但它不支持用户控制的密钥轮换。
题目要求每年自动轮换加密密钥，因此此选项错误。
D. Use server-side encryption with AWS Key Management Service (AWS KMS) customer managed keys. Configure key
rotation.
AWS KMS客户托管密钥支持自动密钥轮换，符合题目要求。
此选项正确。
E. Use server-side encryption with AWS Key Management Service (AWS KMS) customer provided (imported) keys.
Configure key rotation.
客户提供的密钥需要手动管理密钥轮换，不支持自动轮换。
题目要求自动轮换加密密钥，因此此选项错误。
