A company runs an application on a large fleet of Amazon EC2 instances. The application reads and writes entries into an Amazon
DynamoDB table. The size of the DynamoDB table continuously grows, but the application needs only data from the last 30 days.
The company needs a solution that minimizes cost and development effort.
Which solution meets these requirements?
一家公司在大量的 Amazon EC2 实例上运行一个应用程序。该应用程序在 Amazon DynamoDB 表中读取和写入条目。DynamoDB
表的大小不断增长，但该应用程序仅需要过去 30 天的数据。公司需要一个解决方案以最小化成本和开发工作量。
满足这些要求的解决方案是什么？
A. Use an AWS CloudFormation template to deploy the complete solution. Redeploy the CloudFormation stack every 30 days,

题目分析与解答
1. 考察的知识点
Amazon DynamoDB的TTL（Time-to-Live）功能。
如何优化成本和减少开发工作量。
使用AWS服务的最佳实践。
2. 选项分析
A. 使用 AWS CloudFormation 模板部署完整解决方案。每 30 天重新部署 CloudFormation 堆栈，并删除原始堆栈。
错误：CloudFormation主要用于资源的自动化部署和管理，而不是数据管理。重新部署整个堆栈并删除原始堆栈的方式效率低下，
且不符合“最小化开发工作量”的要求。
B. 使用运行来自 AWS Marketplace 的监控应用程序的 EC2 实例。配置监控应用程序使用 Amazon DynamoDB Streams 存储表中
新项目创建时的时间戳。在 EC2 实例上运行一个脚本以删除时间戳超过 30 天的项目。
错误：这种方法需要额外的EC2实例和监控应用程序，增加了成本和复杂性。手动运行脚本也增加了开发工作量，不符合“最小化成
本和开发工作量”的要求。
C. 配置 Amazon DynamoDB Streams 在表中新项目创建时调用 AWS Lambda 函数。配置 Lambda 函数以删除表中超过 30 天的
项目。
错误：虽然使用Lambda函数可以自动化删除操作，但需要额外的开发工作来编写和维护Lambda函数逻辑。相比D选项，开发工作
量更大。
D. 扩展应用程序，为表中新创建的每个项目添加一个属性，该属性的值为当前时间戳加 30 天。配置 DynamoDB 使用该属性作为
TTL 属性。
正确：DynamoDB的TTL功能可以自动删除过期数据，无需额外的开发工作或资源。通过设置TTL属性，DynamoDB会自动清理超过
30天的数据，符合“最小化成本和开发工作量”的要求。
