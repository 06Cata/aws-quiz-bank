题目分析与解答
1. 考察的知识点
容器化应用的迁移与部署：了解如何将现有的容器化应用迁移到AWS。
计算服务选择：熟悉Amazon ECS、Amazon EKS以及AWS Fargate的使用场景。
数据库服务选择：了解MongoDB与AWS数据库服务的兼容性。
操作开销最小化：选择托管服务以减少管理工作。
2. 选项分析
A. Use Amazon Elastic Container Service (Amazon ECS) with Amazon EC2 worker nodes for compute and MongoDB on
EC2 for data storage.
正确性分析：Amazon ECS可以运行容器化应用，但使用EC2实例运行MongoDB会增加管理开销（如手动安装、维护、备份等）。
错误原因：不符合“最小化操作开销”的要求。
B. Use Amazon Elastic Container Service (Amazon ECS) with AWS Fargate for compute and Amazon DynamoDB for data
storage.
正确性分析：AWS Fargate减少了管理计算资源的开销，但DynamoDB是NoSQL数据库，与MongoDB的兼容性较低。
错误原因：需要代码修改以适配DynamoDB，不符合“不做代码或部署方法更改”的要求。
C. Use Amazon Elastic Kubernetes Service (Amazon EKS) with Amazon EC2 worker nodes for compute and Amazon
DynamoDB for data storage.
正确性分析：Amazon EKS支持Kubernetes，但DynamoDB与MongoDB不兼容，且EC2 worker nodes增加了管理开销。
错误原因：不符合“不做代码更改”和“最小化操作开销”的要求。
D. Use Amazon Elastic Kubernetes Service (Amazon EKS) with AWS Fargate for compute and Amazon DocumentDB (with
MongoDB compatibility) for data storage.
正确性分析：Amazon EKS支持Kubernetes，AWS Fargate减少了计算资源管理开销，Amazon DocumentDB与MongoDB兼容，满
足“不做代码更改”的要求。
正确原因：完全符合题目要求，最小化操作开销且无需代码或部署方法更改。
