题目分析与解答
1. 考察的知识点
本题考察的是如何优化 Amazon DynamoDB 的性能，特别是在读密集型场景下，且要求减少操作开销。
Correct Answer: B
正确答案：B

涉及的服务包括 Amazon DynamoDB、DynamoDB Accelerator (DAX)、Amazon ElastiCache (Redis 和 Memcached)、以及
DynamoDB 全球表。
2. 选项分析
A. Use Amazon ElastiCache for Redis.
Redis 是一种高性能的缓存解决方案，可以用于加速数据库查询。
然而，使用 Redis 需要额外的配置和开发工作来集成到现有应用中，这与题目中“不需要重新配置应用”的要求不符。
因此，此选项错误。
B. Use Amazon DynamoDB Accelerator (DAX).
DAX 是专门为 DynamoDB 提供的内存中缓存服务，能够显著提高读性能。
它与 DynamoDB 无缝集成，不需要对现有应用进行重大修改。
符合题目要求，因此此选项正确。
C. Replicate data by using DynamoDB global tables.
全球表用于跨区域复制数据，主要解决多区域可用性和延迟问题。
虽然全球表可以提高性能，但它并不是专门针对读密集型场景的优化方案，且可能需要额外的配置工作。
因此，此选项错误。
D. Use Amazon ElastiCache for Memcached with Auto Discovery enabled.
Memcached 是另一种缓存解决方案，支持自动发现功能。
与 Redis 类似，使用 Memcached 需要额外的开发工作来集成到现有应用中，这与题目中“不需要重新配置应用”的要求不符。
因此，此选项错误。
3. 我的答案
B. Use Amazon DynamoDB Accelerator (DAX).
