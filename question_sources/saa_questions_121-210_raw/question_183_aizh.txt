A company is building a new dynamic ordering website. The company wants to minimize server maintenance and patching. The
website must be highly available and must scale read and write capacity as quickly as possible to meet changes in user demand.
Which solution will meet these requirements?
一家公司正在构建一个新的动态订单网站。该公司希望尽量减少服务器维护和补丁更新。网站必须具有高可用性，并且必须尽可能
快速地扩展读写容量，以满足用户需求的变化。
哪种解决方案可以满足这些要求？
A. Host static content in Amazon S3. Host dynamic content by using Amazon API Gateway and AWS Lambda. Use Amazon
DynamoDB with on-demand capacity for the database. Configure Amazon CloudFront to deliver the website content.

题目分析与解答
1. 考察的知识点
无服务器架构（Serverless Architecture）
Amazon S3 用于托管静态内容
Amazon API Gateway 和 AWS Lambda 用于动态内容处理
Amazon DynamoDB 的按需容量模式
Amazon CloudFront 用于内容分发和加速
高可用性和自动扩展能力
2. 选项分析
A:
静态内容托管在 Amazon S3：正确，S3 是托管静态内容的最佳选择，支持高可用性和低成本。
动态内容使用 Amazon API Gateway 和 AWS Lambda：正确，无服务器架构减少了服务器维护和补丁工作，同时支持自动扩展。
数据库使用 Amazon DynamoDB 的按需容量：正确，按需容量模式支持自动扩展，适合动态负载。
使用 Amazon CloudFront：正确，CloudFront 提供全球内容分发和加速。
此选项完全符合题目要求。
B:
静态内容托管在 Amazon S3：正确。
动态内容使用 Amazon API Gateway 和 AWS Lambda：正确。
数据库使用 Amazon Aurora 和 Aurora Auto Scaling：错误，Aurora 是关系型数据库，虽然支持自动扩展，但相比 DynamoDB，
Aurora 的管理复杂性更高，不完全符合“最小化服务器维护和补丁”的要求。
使用 Amazon CloudFront：正确。
此选项不完全符合题目要求。
C:
所有内容托管在 Amazon EC2 实例上：错误，EC2 需要手动管理服务器，增加了维护和补丁工作量，不符合题目要求。
创建 Auto Scaling 组：正确，但 EC2 的管理复杂性较高。
使用 Application Load Balancer：正确，但不符合无服务器架构的要求。
数据库使用 Amazon DynamoDB 的预置写入容量：错误，预置容量模式需要手动调整，无法快速响应需求变化。
此选项不符合题目要求。
D:
