题目分析与解答
1. 考察的知识点
Amazon Pinpoint的功能和使用场景。
如何处理和存储SMS消息及其回复。
数据流的分析与归档（Amazon Kinesis）。
2. 选项分析
A. Create an Amazon Connect contact flow to send the SMS messages. Use AWS Lambda to process the responses.
Amazon Connect主要用于构建呼叫中心解决方案，而不是发送SMS消息。
虽然可以使用AWS Lambda处理回复，但Amazon Connect并不适合此场景。
错误原因：此选项不符合发送SMS消息和存储回复的需求。
B. Build an Amazon Pinpoint journey. Configure Amazon Pinpoint to send events to an Amazon Kinesis data stream for
analysis and archiving.
Amazon Pinpoint专门用于发送SMS消息和管理用户交互。
可以配置将事件发送到Amazon Kinesis数据流以进行分析和归档。
正确原因：此选项完全满足发送SMS消息、处理回复以及存储数据的需求。
C. Use Amazon Simple Queue Service (Amazon SQS) to distribute the SMS messages. Use AWS Lambda to process the
responses.
Amazon SQS主要用于消息队列，而不是直接发送SMS消息。
虽然可以使用AWS Lambda处理回复，但SQS不适合发送SMS消息。
错误原因：此选项不符合发送SMS消息的需求。
D. Create an Amazon Simple Notification Service (Amazon SNS) FIFO topic. Subscribe an Amazon Kinesis data stream to
the SNS topic for analysis and archiving.
Amazon SNS可以发送通知，但不支持用户回复SMS消息。
FIFO主题主要用于严格的消息顺序，而不是SMS消息发送。
错误原因：此选项不支持用户回复SMS消息的需求。
3. 我的答案
B. Build an Amazon Pinpoint journey. Configure Amazon Pinpoint to send events to an Amazon Kinesis data stream for
analysis and archiving.
