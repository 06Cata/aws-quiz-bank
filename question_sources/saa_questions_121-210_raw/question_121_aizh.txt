DB instance in a Multi-AZ deployment. Daily database snapshots are taken from this instance.
What should a solutions architect do to ensure the database and snapshots are always encrypted moving forward?

署。每天都会从该实例中获取数据库快照。

2. 选项分析
A. Encrypt a copy of the latest DB snapshot. Replace existing DB instance by restoring the encrypted snapshot.
正确：这是AWS推荐的方式。RDS实例无法直接启用加密，必须通过创建加密的快照并从该快照恢复一个新的加密实例来实现。
此方法确保数据库和未来的快照都被加密。
B. Create a new encrypted Amazon Elastic Block Store (Amazon EBS) volume and copy the snapshots to it. Enable
encryption on the DB instance.
错误：RDS实例的加密与EBS卷的加密是独立的。无法通过创建加密的EBS卷直接为RDS实例启用加密。
RDS实例的加密只能通过加密的快照恢复来实现。
C. Copy the snapshots and enable encryption using AWS Key Management Service (AWS KMS). Restore encrypted
snapshot to an existing DB instance.
错误：虽然可以使用KMS加密快照，但无法将加密的快照恢复到现有的未加密实例。
必须创建一个新的加密实例，而不是恢复到现有实例。
D. Copy the snapshots to an Amazon S3 bucket that is encrypted using server-side encryption with AWS Key Management
Service (AWS KMS) managed keys (SSE-KMS).
错误：将快照复制到S3并加密不会影响RDS实例本身的加密状态。
此方法仅适用于备份目的，而不是为RDS实例启用加密。
