where the AMIs were created. The company needs to design an application that captures AWS API calls and sends alerts whenever
the Amazon EC2 CreateImage API operation is called within the company’s account.
Which solution will meet these requirements with the LEAST operational overhead?

题目分析与解答
1. 考察的知识点
本题考察的是 AWS 服务集成与事件驱动架构的设计，包括 Amazon EventBridge、Amazon SNS、AWS CloudTrail 等服务的使用。
重点在于如何以最低的操作开销实现对 API 调用的实时监控和警报。
2. 选项分析
A. Create an AWS Lambda function to query AWS CloudTrail logs and to send an alert when a CreateImage API call is
detected.
错误：虽然可以通过 Lambda 查询 CloudTrail 日志来检测 API 调用，但这种方法需要定期运行 Lambda 函数以扫描日志，增加了操
作开销。
此外，这种方式不是实时的，可能会有延迟。
B. Configure AWS CloudTrail with an Amazon Simple Notification Service (Amazon SNS) notification that occurs when
updated logs are sent to Amazon S3. Use Amazon Athena to create a new table and to query on CreateImage when an API
call is detected.
错误：此方法依赖于 CloudTrail 将日志存储到 S3，然后使用 Athena 查询日志。这种方式需要手动配置 Athena 表，并且查询过程
可能会有延迟。
操作开销较高，不符合“最低操作开销”的要求。
C. Create an Amazon EventBridge (Amazon CloudWatch Events) rule for the CreateImage API call. Configure the target as
an Amazon Simple Notification Service (Amazon SNS) topic to send an alert when a CreateImage API call is detected.
正确：EventBridge 可以直接捕获 CreateImage API 调用事件，并将其发送到 SNS 主题以触发警报。
这种方法是实时的，且操作开销最低，因为不需要额外的日志查询或存储配置。
D. Configure an Amazon Simple Queue Service (Amazon SQS) FIFO queue as a target for AWS CloudTrail logs. Create an
AWS Lambda function to send an alert to an Amazon Simple Notification Service (Amazon SNS) topic when a
CreateImage API call is detected.
错误：此方法需要配置 SQS 队列和 Lambda 函数，增加了操作复杂性。
此外，依赖 CloudTrail 日志，可能会有延迟，不如 EventBridge 的实时性强。
