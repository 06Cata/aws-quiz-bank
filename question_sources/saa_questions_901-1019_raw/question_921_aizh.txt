Question #921
A company recently migrated a monolithic application to an Amazon EC2 instance and Amazon RDS. The application has tightly
coupled modules. The existing design of the application gives the application the ability to run on only a single EC2 instance.
The company has noticed high CPU utilization on the EC2 instance during peak usage times. The high CPU utilization corresponds
to degraded performance on Amazon RDS for read requests. The company wants to reduce the high CPU utilization and improve
read request performance.
Which solution will meet these requirements?

降低高 CPU 使用率并改善读取请求性能。
哪种解决方案可以满足这些需求？

题目分析与解答
1. 考察的知识点
Amazon EC2实例的性能优化和扩展。
Amazon RDS的读写性能优化，包括使用读副本。
Auto Scaling组的配置和使用。
2. 选项分析
A. Resize the EC2 instance to an EC2 instance type that has more CPU capacity. Configure an Auto Scaling group with a
minimum and maximum size of 1. Configure an RDS read replica for read requests.
正确：调整EC2实例以增加CPU容量可以解决高CPU利用率问题。
正确：配置Auto Scaling组的最小和最大大小为1，确保单实例运行符合当前应用的设计限制。
正确：添加RDS读副本可以分担读请求负载，从而提高数据库性能。
B. Resize the EC2 instance to an EC2 instance type that has more CPU capacity. Configure an Auto Scaling group with a
minimum and maximum size of 1. Add an RDS read replica and redirect all read/write traffic to the replica.
错误：RDS读副本仅用于处理读请求，不能处理写请求。将所有读/写流量重定向到读副本是不正确的。
正确：调整EC2实例以增加CPU容量可以解决高CPU利用率问题。
正确：配置Auto Scaling组的最小和最大大小为1，符合当前应用的设计限制。
C. Configure an Auto Scaling group with a minimum size of 1 and maximum size of 2. Resize the RDS DB instance to an
instance type that has more CPU capacity.
错误：当前应用设计为只能运行在单个EC2实例上，因此Auto Scaling组的最大大小设置为2不符合要求。
正确：调整RDS实例以增加CPU容量可以提高数据库性能，但没有解决读请求的分担问题。
D. Resize the EC2 instance to an EC2 instance type that has more CPU capacity. Configure an Auto Scaling group with a
minimum and maximum size of 1. Resize the RDS DB instance to an instance type that has more CPU capacity.
正确：调整EC2实例以增加CPU容量可以解决高CPU利用率问题。
正确：配置Auto Scaling组的最小和最大大小为1，符合当前应用的设计限制。

性能问题。
