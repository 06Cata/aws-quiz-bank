Question #975
A weather forecasting company collects temperature readings from various sensors on a continuous basis. An existing data
ingestion process collects the readings and aggregates the readings into larger Apache Parquet files. Then the process encrypts
the files by using client-side encryption with KMS managed keys (CSE-KMS). Finally, the process writes the files to an Amazon S3
bucket with separate prefixes for each calendar day.
The company wants to run occasional SQL queries on the data to take sample moving averages for a specific calendar day.
Which solution will meet these requirements MOST cost-effectively?
一家天气预报公司持续从各种传感器收集温度读数。现有的数据摄取流程会收集这些读数，并将读数汇总成更大的 Apache Parquet
文件。然后，该流程使用客户端加密和 KMS 管理密钥（CSE-KMS）对文件进行加密。最后，该流程将文件写入一个 Amazon S3 存
储桶，并为每个日历日设置单独的前缀。
该公司希望对数据运行偶尔的 SQL 查询，以获取特定日历日的样本移动平均值。
哪种解决方案可以最具成本效益地满足这些需求？

题目分析与解答
1. 考察的知识点
Amazon Athena的使用及其与Amazon S3的集成。
Amazon S3存储和加密文件的处理方式。
成本效益分析：选择最经济的解决方案。
SQL查询工具的适用场景。
2. 选项分析
A. Configure Amazon Athena to read the encrypted files. Run SQL queries on the data directly in Amazon S3.
正确：Amazon Athena是一种无服务器的交互式查询服务，可以直接查询存储在Amazon S3中的数据。它支持读取加密的Parquet文
件，并且可以通过KMS解密数据。
成本效益高：Athena按查询付费，无需预置资源，非常适合偶尔运行SQL查询的场景。
B. Use Amazon S3 Select to run SQL queries on the data directly in Amazon S3.
错误：Amazon S3 Select主要用于从单个对象中提取部分数据，而不是运行复杂的SQL查询。题目要求运行SQL查询以计算移动平均
值，S3 Select不适合这种场景。
C. Configure Amazon Redshift to read the encrypted files. Use Redshift Spectrum and Redshift query editor v2 to run SQL
queries on the data directly in Amazon S3.
错误：虽然Redshift Spectrum可以查询存储在S3中的数据，但Redshift需要预置集群，成本较高，不适合偶尔运行查询的场景。
D. Configure Amazon EMR Serverless to read the encrypted files. Use Apache SparkSQL to run SQL queries on the data
directly in Amazon S3.
错误：Amazon EMR Serverless适合处理大规模数据分析任务，但对于偶尔运行SQL查询的场景，EMR的复杂性和成本较高，不是最
优选择。
