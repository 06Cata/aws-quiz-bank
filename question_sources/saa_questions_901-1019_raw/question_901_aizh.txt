题目分析与解答
1. 考察的知识点
Amazon RDS的功能与优势，包括Multi-AZ部署。
AWS Key Management Service (KMS)的使用和数据加密。
如何选择适合的AWS服务来迁移和管理关系型数据库。
2. 选项分析
A. Migrate the databases to Amazon EC2 instances. Use an AWS Key Management Service (AWS KMS) AWS managed key
for encryption.
错误：虽然可以在EC2实例上运行SQL Server并使用KMS进行加密，但这种方法增加了运维开销，因为需要手动管理EC2实例的操作
系统、数据库软件的安装和维护。
不符合题目要求的“减少运维开销”。
B. Migrate the databases to a Multi-AZ Amazon RDS for SQL Server DB instance. Use an AWS Key Management Service
(AWS KMS) AWS managed key for encryption.
正确：Amazon RDS是托管服务，减少了运维开销，同时支持Multi-AZ部署以提高可用性。
RDS支持KMS进行数据加密，符合题目中对安全性的要求。
完全满足题目要求的“增加安全性”和“减少运维开销”。
C. Migrate the data to an Amazon S3 bucket. Use Amazon Macie to ensure data security.
错误：Amazon S3是对象存储服务，不适合存储关系型数据库数据。
Amazon Macie主要用于识别敏感数据，而不是直接管理数据库的安全性。
不符合题目要求的“迁移关系型数据库”。
D. Migrate the databases to an Amazon DynamoDB table. Use Amazon CloudWatch Logs to ensure data security.
错误：Amazon DynamoDB是NoSQL数据库服务，不适合迁移SQL Server关系型数据库。
CloudWatch Logs用于监控和日志记录，不直接提供数据安全功能。
不符合题目要求的“迁移关系型数据库”。
3. 我选的答案
B. Migrate the databases to a Multi-AZ Amazon RDS for SQL Server DB instance. Use an AWS Key Management Service
(AWS KMS) AWS managed key for encryption.
