Question #942
A company regularly uploads confidential data to Amazon S3 buckets for analysis.
The company's security policies mandate that the objects must be encrypted at rest. The company must automatically rotate the
encryption key every year. The company must be able to track key rotation by using AWS CloudTrail. The company also must
minimize costs for the encryption key.
Which solution will meet these requirements?
一家公司定期将机密数据上传到 Amazon S3 存储桶进行分析。
该公司的安全策略要求对象必须在静止状态下加密。 公司必须每年自动轮换加密密钥。 公司必须能够通过 AWS CloudTrail 跟踪密
钥轮换。 公司还必须尽量减少加密密钥的成本。
哪种解决方案可以满足这些要求？
A. Use server-side encryption with customer-provided keys (SSE-C)
使用客户提供的密钥进行服务器端加密 (SSE-C)
B. Use server-side encryption with Amazon S3 managed keys (SSE-S3)
使用 Amazon S3 管理密钥进行服务器端加密 (SSE-S3)
C. Use server-side encryption with AWS KMS keys (SSE-KMS) Most Voted
使用 AWS KMS 密钥进行服务器端加密 (SSE-KMS)
D. Use server-side encryption with customer managed AWS KMS keys
使用客户管理的 AWS KMS 密钥进行服务器端加密
Correct Answer: C
正确答案：C

题目分析与解答
1. 考察的知识点
Amazon S3的加密选项，包括SSE-C、SSE-S3、SSE-KMS。
AWS Key Management Service (KMS)的功能，包括密钥管理和自动轮换。
AWS CloudTrail的功能，用于跟踪密钥使用和轮换事件。
成本优化与安全性要求的平衡。
2. 选项分析
A. 使用客户提供的密钥进行服务器端加密 (SSE-C)
错误：SSE-C要求客户自己提供加密密钥，并且AWS不会管理密钥的轮换。无法满足“自动轮换密钥每年一次”的要求。
错误：SSE-C的密钥管理和跟踪需要客户自己实现，无法通过AWS CloudTrail跟踪密钥轮换事件。
B. 使用 Amazon S3 管理密钥进行服务器端加密 (SSE-S3)
错误：SSE-S3使用Amazon S3管理的密钥，但这些密钥无法自动轮换，也无法通过AWS CloudTrail跟踪密钥轮换事件。
错误：虽然SSE-S3成本较低，但无法满足“自动轮换密钥”和“跟踪密钥轮换”的要求。
C. 使用 AWS KMS 密钥进行服务器端加密 (SSE-KMS)
正确：SSE-KMS使用AWS KMS管理的密钥，支持自动轮换密钥，并且可以通过AWS CloudTrail跟踪密钥使用和轮换事件。
正确：SSE-KMS的成本较低，因为使用的是AWS管理的KMS密钥，而不是客户管理的密钥。
D. 使用客户管理的 AWS KMS 密钥进行服务器端加密
错误：虽然客户管理的KMS密钥也支持自动轮换和CloudTrail跟踪，但成本较高，因为客户需要额外支付管理密钥的费用。
错误：题目要求“最小化成本”，客户管理的密钥不符合这一要求。
3. 我选的答案
C. 使用 AWS KMS 密钥进行服务器端加密 (SSE-KMS)
4. 官方答案
C. 使用 AWS KMS 密钥进行服务器端加密 (SSE-KMS)
5. 与官方答案比较
我的答案与官方答案一致。
原因分析：SSE-KMS满足所有要求，包括自动轮换密钥、通过CloudTrail跟踪密钥轮换事件，以及成本优化。
