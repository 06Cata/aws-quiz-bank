Question #915
A video game company is deploying a new gaming application to its global users. The company requires a solution that will
provide near real-time reviews and rankings of the players.
A solutions architect must design a solution to provide fast access to the data. The solution must also ensure the data persists on
disks in the event that the company restarts the application.
Which solution will meet these requirements with the LEAST operational overhead?
一家视频游戏公司正在向其全球用户部署一款新的游戏应用程序。该公司需要一个解决方案来提供近乎实时的玩家评论和排名。
解决方案架构师必须设计一个解决方案，以提供对数据的快速访问。该解决方案还必须确保在公司重新启动应用程序时，数据能够
持久存储在磁盘上。
哪种解决方案能够以最少的运营开销满足这些需求？
A. Configure an Amazon CloudFront distribution with an Amazon S3 bucket as the origin. Store the player data in the S3
bucket.
配置一个 Amazon CloudFront 分发，并将 Amazon S3 bucket 作为源。将玩家数据存储在 S3 bucket 中。

在多个 AWS 区域中创建 Amazon EC2 实例。将玩家数据存储在 EC2 实例上。使用 Amazon Route 53 配置地理位置记录，将用

Correct Answer: C

题目分析与解答
1. 考察的知识点
本题主要考察AWS服务的选择与架构设计能力，尤其是针对高性能、低延迟数据访问的解决方案。
涉及的服务包括Amazon CloudFront、Amazon S3、Amazon EC2、Amazon Route 53、Amazon ElastiCache（Redis和
Memcached）。
重点在于数据持久性、实时性以及最小化运维开销。
2. 选项分析
A. Configure an Amazon CloudFront distribution with an Amazon S3 bucket as the origin. Store the player data in the S3
bucket.
错误原因：Amazon S3是一个对象存储服务，适合存储静态文件，但不适合实时数据访问。CloudFront可以加速内容分发，但无法
满足实时性和数据持久性要求。
B. Create Amazon EC2 instances in multiple AWS Regions. Store the player data on the EC2 instances. Configure Amazon
Route 53 with geolocation records to direct users to the closest EC2 instance.
错误原因：虽然EC2实例可以存储数据并通过Route 53实现地理位置路由，但需要手动管理实例的高可用性和数据持久性，运维开
销较高，不符合“最小化运维开销”的要求。
C. Deploy an Amazon ElastiCache for Redis cluster. Store the player data in the ElastiCache cluster.
正确原因：ElastiCache for Redis支持高性能、低延迟的数据访问，并且可以配置持久化以确保数据在应用重启时不会丢失。Redis还
支持复杂的数据结构，非常适合实时排名和评论功能。
D. Deploy an Amazon ElastiCache for Memcached cluster. Store the player data in the ElastiCache cluster.
错误原因：虽然Memcached也支持高性能数据访问，但它不支持数据持久化功能，因此无法满足“数据在应用重启时不会丢失”的要
求。
3. 我的答案
C. Deploy an Amazon ElastiCache for Redis cluster. Store the player data in the ElastiCache cluster.
4. 官方答案
C. Deploy an Amazon ElastiCache for Redis cluster. Store the player data in the ElastiCache cluster.
5. 与官方答案比较
我的答案与官方答案一致。
理由：ElastiCache for Redis是唯一满足实时性、数据持久性以及最小化运维开销的选项。
C (100%)
