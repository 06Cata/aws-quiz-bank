Question #1002
An online gaming company is transitioning user data storage to Amazon DynamoDB to support the company's growing user base.
The current architecture includes DynamoDB tables that contain user profiles, achievements, and in-game transactions.
The company needs to design a robust, continuously available, and resilient DynamoDB architecture to maintain a seamless
gaming experience for users.
Which solution will meet these requirements MOST cost-effectively?
一家在线游戏公司正在将用户数据存储迁移到 Amazon DynamoDB，以支持公司不断增长的用户群。当前架构包括包含用户资料、
成就和游戏内交易的 DynamoDB 表。
公司需要设计一个强大的、持续可用的、具有弹性的 DynamoDB 架构，以确保用户拥有无缝的游戏体验。
哪种解决方案最具成本效益地满足这些需求？
A. Create DynamoDB tables in a single AWS Region. Use on-demand capacity mode. Use global tables to replicate data across
multiple Regions.
在单个 AWS 区域中创建 DynamoDB 表。使用按需容量模式。使用全球表将数据复制到多个区域。
B. Use DynamoDB Accelerator (DAX) to cache frequently accessed data. Deploy tables in a single AWS Region and enable auto
scaling. Configure Cross-Region Replication manually to additional Regions.
使用 DynamoDB Accelerator (DAX) 缓存频繁访问的数据。在单个 AWS 区域中部署表并启用自动扩展。手动配置跨区域复制到
其他区域。
C. Create DynamoDB tables in multiple AWS Regions. Use on-demand capacity mode. Use DynamoDB Streams for Cross-
Region Replication between Regions.
在多个 AWS 区域中创建 DynamoDB 表。使用按需容量模式。使用 DynamoDB Streams 在区域之间进行跨区域复制。
D. Use DynamoDB global tables for automatic multi-Region replication. Deploy tables in multiple AWS Regions. Use
provisioned capacity mode. Enable auto scaling. Most Voted
使用 DynamoDB 全球表进行自动多区域复制。在多个 AWS 区域中部署表。使用预置容量模式。启用自动扩展。
Correct Answer: D
正确答案：D
题目分析与解答
1. 考察的知识点
Amazon DynamoDB的全球表（Global Tables）功能，用于实现跨区域数据复制和高可用性。
按需容量模式与预置容量模式的区别及使用场景。
DynamoDB的自动扩展功能。
如何设计成本效益高且具有高可用性和容错能力的架构。
2. 选项分析
A. Create DynamoDB tables in a single AWS Region. Use on-demand capacity mode. Use global tables to replicate data
across multiple Regions.
正确性分析：此选项提到使用全球表（Global Tables）进行跨区域复制，但全球表需要在多个区域中创建表，而不是单区域。此
外，按需容量模式虽然灵活，但在高流量场景下可能成本较高。
结论：错误。
B. Use DynamoDB Accelerator (DAX) to cache frequently accessed data. Deploy tables in a single AWS Region and enable
auto scaling. Configure Cross-Region Replication manually to additional Regions.
正确性分析：DAX可以提高读取性能，但它不解决跨区域复制的问题。手动配置跨区域复制复杂且容易出错，不符合“连续可
用”和“弹性”的要求。
结论：错误。
C. Create DynamoDB tables in multiple AWS Regions. Use on-demand capacity mode. Use DynamoDB Streams for Cross-
Region Replication between Regions.
正确性分析：使用DynamoDB Streams进行跨区域复制需要额外的管理工作，且不如全球表（Global Tables）自动化和高效。此
外，按需容量模式在高流量场景下可能成本较高。
结论：错误。
D. Use DynamoDB global tables for automatic multi-Region replication. Deploy tables in multiple AWS Regions. Use
provisioned capacity mode. Enable auto scaling.
正确性分析：全球表（Global Tables）提供自动化的多区域复制，确保高可用性和容错能力。预置容量模式结合自动扩展可以在成
本和性能之间取得平衡，是最符合题目要求的解决方案。
结论：正确。
