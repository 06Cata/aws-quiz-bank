Question #997
A company is launching a new application that requires a structured database to store user profiles, application settings, and
transactional data. The database must be scalable with application traffic and must offer backups.
Which solution will meet these requirements MOST cost-effectively?
一家公司正在推出一个新应用程序，该应用程序需要一个结构化数据库来存储用户档案、应用程序设置和事务数据。数据库必须能
够随着应用程序流量进行扩展，并且必须提供备份功能。
哪种解决方案最具成本效益地满足这些需求？
A. Deploy a self-managed database on Amazon EC2 instances by using open source software. Use Spot Instances for cost

backups with a retention period of 7 days.

题目分析与解答
1. 考察的知识点
Amazon RDS 和 Amazon Aurora 的功能与特点
数据库的可扩展性和备份机制
成本优化的最佳实践
选择合适的数据库解决方案（关系型 vs NoSQL）
2. 选项分析
A. Deploy a self-managed database on Amazon EC2 instances by using open source software. Use Spot Instances for cost
optimization. Configure automated backups to Amazon S3.
正确性分析：虽然使用 Spot Instances 可以降低成本，但自托管数据库需要额外的管理工作，包括安装、维护、扩展和备份配置。
这增加了复杂性和运营成本。
备份到 Amazon S3 是可行的，但不如 Amazon RDS 或 Aurora 的自动备份机制方便。
错误原因：不符合“最成本效益”的要求，因为管理开销较高。
B. Use Amazon RDS. Use on-demand capacity mode for the database with General Purpose SSD storage. Configure
automatic backups with a retention period of 7 days.
正确性分析：Amazon RDS 提供托管的关系型数据库服务，支持自动备份和扩展，使用 General Purpose SSD 存储是一个合理的选
择。
错误原因：虽然 RDS 是一个不错的选择，但使用按需容量模式可能会导致成本高于 Aurora Serverless，尤其是在流量波动较大的情
况下。
C. Use Amazon Aurora Serverless for the database. Use serverless capacity scaling. Configure automated backups to
Amazon S3.
正确性分析：Amazon Aurora Serverless 是一个无服务器的关系型数据库解决方案，支持自动扩展和按需计费，非常适合流量波动
的应用场景。
备份机制自动集成到 Amazon S3，简化了管理工作。
正确原因：最符合题目要求的“最成本效益”解决方案。
D. Deploy a self-managed NoSQL database on Amazon EC2 instances. Use Reserved Instances for cost optimization.
Configure automated backups directly to Amazon S3 Glacier Flexible Retrieval.
正确性分析：NoSQL 数据库不适合存储结构化数据（如用户配置文件和事务数据）。
虽然 Reserved Instances 可以降低成本，但自托管数据库仍然需要额外的管理工作。
错误原因：不符合题目要求的“结构化数据库”和“最成本效益”解决方案。
3. 我的答案
C. Use Amazon Aurora Serverless for the database. Use serverless capacity scaling. Configure automated backups to
Amazon S3.
4. 官方答案
C. Use Amazon Aurora Serverless for the database. Use serverless capacity scaling. Configure automated backups to
Amazon S3.
5. 与官方答案比较
我的答案与官方答案一致。
原因分析：Amazon Aurora Serverless 提供了无服务器架构，支持自动扩展和按需计费，备份机制集成到 Amazon S3，最符合题目
要求的“最成本效益”解决方案。
