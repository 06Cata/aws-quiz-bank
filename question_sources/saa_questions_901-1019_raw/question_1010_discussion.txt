A company runs multiple workloads on virtual machines (VMs) in an on-premises data center. The company is expanding rapidly.
The on-premises data center is not able to scale fast enough to meet business needs. The company wants to migrate the
workloads to AWS.
The migration is time sensitive. The company wants to use a lift-and-shift strategy for non-critical workloads.
Which combination of steps will meet these requirements? (Choose three.)
A. Use the AWS Schema Conversion Tool (AWS SCT) to collect data about the VMs.
B. Use AWS Application Migration Service. Install the AWS Replication Agent on the VMs. Most Voted
C. Complete the initial replication of the VMs. Launch test instances to perform acceptance tests on the VMs. Most Voted
D. Stop all operations on the VMs. Launch a cutover instance. Most Voted
E. Use AWS App2Container (A2C) to collect data about the VMs.
F. Use AWS Database Migration Service (AWS DMS) to migrate the VMs.

A - SCT is used for database schema conversions when migrating databases, not for migrating VMs.
B - Application Migration Service is designed for lift-and-shift migrations. It uses the AWS Replication Agent to replicate VMs to AWS.
C - After replication, testing ensures that the workloads function correctly before completing the migration. (If you're wondering why, see this:
https://docs.aws.amazon.com/mgn/latest/ug/launch-cutover-gs.html)
D - This is a critical step in the migration process. After testing, cutover instances are launched, and the source VMs are stopped to finalize the
migration.
E - AWS App2Container is used for containerizing applications, not for migrating VMs.

While AWS DMS is used for database migration, the question specifies migrating entire VMs, not just databases.
