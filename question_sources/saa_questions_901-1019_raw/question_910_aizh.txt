its database runs on Amazon RDS. The company is anticipating a large increase in sales during an upcoming holiday weekend. A
solutions architect needs to build a solution to analyze the performance of the web application with a granularity of no more than
2 minutes.
What should the solutions architect do to meet this requirement?

题目分析与解答
1. 考察的知识点
Amazon CloudWatch的详细监控功能。
如何使用CloudWatch Metrics进行性能分析。
AWS服务之间的集成与数据分析能力。
2. 选项分析
A. Send Amazon CloudWatch logs to Amazon Redshift. Use Amazon QuickSight to perform further analysis.
错误：将CloudWatch日志发送到Redshift并使用QuickSight进行分析虽然可以实现数据分析，但这种方法不适合实时性能监控，尤
其是要求2分钟的粒度。
Redshift更适合处理批量数据，而不是实时数据分析。
B. Enable detailed monitoring on all EC2 instances. Use Amazon CloudWatch metrics to perform further analysis.
正确：启用EC2实例的详细监控后，CloudWatch可以提供更高粒度的指标（每分钟一次），满足2分钟粒度的要求。
CloudWatch Metrics是专门设计用于实时性能监控的工具。
C. Create an AWS Lambda function to fetch EC2 logs from Amazon CloudWatch Logs. Use Amazon CloudWatch metrics to
perform further analysis.
错误：使用Lambda函数从CloudWatch Logs中提取日志并进行分析是一种复杂且不必要的解决方案。
CloudWatch Metrics已经可以直接提供所需的性能数据，无需额外的Lambda函数。
D. Send EC2 logs to Amazon S3. Use Amazon Redshift to fetch logs from the S3 bucket to process raw data for further
analysis with Amazon QuickSight.
错误：将日志发送到S3并使用Redshift和QuickSight进行分析是一种适合长期数据分析的解决方案，但不适合实时性能监控。
这种方法无法满足2分钟粒度的要求。
3. 我选的答案
B. Enable detailed monitoring on all EC2 instances. Use Amazon CloudWatch metrics to perform further analysis.
4. 官方答案
B. Enable detailed monitoring on all EC2 instances. Use Amazon CloudWatch metrics to perform further analysis.
5. 与官方答案的比较
我的答案与官方答案一致。
原因：启用详细监控并使用CloudWatch Metrics是满足2分钟粒度性能分析的最佳解决方案。
