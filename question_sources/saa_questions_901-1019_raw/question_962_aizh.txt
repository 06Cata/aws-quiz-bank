Question #962
A company uses an Amazon DynamoDB table to store data that the company receives from devices. The DynamoDB table
supports a customer-facing website to display recent activity on customer devices. The company configured the table with
provisioned throughput for writes and reads.
The company wants to calculate performance metrics for customer device data on a daily basis. The solution must have minimal
effect on the table's provisioned read and write capacity.
Which solution will meet these requirements?

题目分析与解答
1. 考察的知识点
Amazon DynamoDB的使用场景与性能优化。
如何在不影响DynamoDB表的读写容量的情况下进行数据分析。
AWS服务之间的集成，包括AWS Glue、Amazon Athena、Amazon Redshift和Amazon EMR。
2. 选项分析
A. Use an Amazon Athena SQL query with the Amazon Athena DynamoDB connector to calculate performance metrics on
a recurring schedule.
正确性分析：Amazon Athena可以通过DynamoDB connector直接查询DynamoDB表，但这种方法会直接读取表数据，可能会影响
DynamoDB的读容量，违反了题目要求“对表的读写容量影响最小”。
结论：错误。
B. Use an AWS Glue job with the AWS Glue DynamoDB export connector to calculate performance metrics on a recurring
schedule.
正确性分析：AWS Glue DynamoDB export connector可以将DynamoDB表数据导出到其他存储（如S3），然后进行分析。这种方法
不会直接影响DynamoDB表的读写容量，符合题目要求。
结论：正确。
C. Use an Amazon Redshift COPY command to calculate performance metrics on a recurring schedule.
正确性分析：Amazon Redshift COPY命令需要将数据从DynamoDB导入到Redshift中，但导入过程可能会直接读取DynamoDB表数
据，影响其读容量，违反题目要求。
结论：错误。
D. Use an Amazon EMR job with an Apache Hive external table to calculate performance metrics on a recurring schedule.
正确性分析：Amazon EMR可以通过Hive外部表查询DynamoDB数据，但这种方法会直接读取DynamoDB表数据，可能会影响其读
容量，违反题目要求。
结论：错误。

小化对表的读写容量的影响，完全符合题目要求。
