题目分析与解答
1. 考察的知识点
本题主要考察AWS Secrets Manager和AWS Systems Manager Parameter Store的使用场景和区别。
同时涉及如何以最少的操作开销安全地存储和检索敏感信息（如用户名和密码）。
2. 选项分析
A. Store the employee credentials in AWS Systems Manager Parameter Store. Use AWS CloudFormation and the
BatchGetSecretValue API to retrieve usernames and passwords from Parameter Store.
错误：AWS Systems Manager Parameter Store适合存储配置数据和非敏感信息，但对于敏感信息（如用户名和密码），AWS
Secrets Manager是更合适的选择。
错误：BatchGetSecretValue API是Secrets Manager的API，不能用于Parameter Store。
B. Store the employee credentials in AWS Secrets Manager. Use AWS CloudFormation and AWS Batch with the
BatchGetSecretValue API to retrieve the usernames and passwords from Secrets Manager.
错误：虽然Secrets Manager是正确的存储选择，但AWS Batch并不是检索Secrets Manager数据的合适工具。
错误：AWS Batch主要用于运行批处理作业，与检索敏感信息无关。
C. Store the employee credentials in AWS Systems Manager Parameter Store. Use AWS CloudFormation and AWS Batch
with the BatchGetSecretValue API to retrieve the usernames and passwords from Parameter Store.
错误：同选项A，Parameter Store不适合存储敏感信息。
错误：BatchGetSecretValue API是Secrets Manager的API，不能用于Parameter Store。
错误：AWS Batch与检索敏感信息无关。
D. Store the employee credentials in AWS Secrets Manager. Use AWS CloudFormation and the BatchGetSecretValue API to
retrieve the usernames and passwords from Secrets Manager.
正确：AWS Secrets Manager专门设计用于存储和管理敏感信息（如用户名和密码）。
正确：BatchGetSecretValue API是Secrets Manager的API，用于检索存储的敏感信息。
正确：此选项使用了正确的服务和API，且操作开销最小。
3. 我的答案
D

D

我的答案与官方答案一致。
原因：AWS Secrets Manager是存储敏感信息的最佳选择，且BatchGetSecretValue API是正确的检索方法。
D (100%)
