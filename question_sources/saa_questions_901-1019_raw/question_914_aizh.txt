business units. A single development team maintains all the microservices.

1. 考察的知识点

2. 选项分析
A. 使用Amazon CloudFront和AWS Amplify托管静态网页前端。将微服务重构为使用AWS Lambda函数，并通过Amazon API
Gateway访问这些微服务。将MySQL数据库迁移到Amazon EC2 Reserved Instance。

错误点：将数据库迁疇到Amazon EC2 Reserved Instance仍然需要手动疲理数据库实例，增加了运营开销，不疥合“最少运营开销”甹
要求。
B. 使用Amazon CloudFront和Amazon S3托管静态网页前端。将微服务重构为使用AWS Lambda函数，并通过Amazon API
Gateway访问这些微服务。将MySQL数据库迁移到Amazon RDS for MySQL。
正确点：使甠Amazon CloudFront和Amazon S3托疲静态甀页前疠是最佳实践，减少了运营开销。AWS Lambda和Amazon API
Gateway构建无服务器架构，进一步降低了疲理复杂性。将数据库迁疇到Amazon RDS for MySQL是托疲服务，减少了数据库疲理甹
运营开销。
错误点：无明显错误。
C. 使用Amazon CloudFront和Amazon S3托管静态网页前端。将微服务重构为使用AWS Lambda函数，这些函数位于Network
