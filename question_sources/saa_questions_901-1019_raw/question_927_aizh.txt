题目分析与解答
1. 考察的知识点
本题考察的是AWS的DDoS防护解决方案，包括AWS Shield Advanced、AWS WAF、Amazon GuardDuty等服务的功能和适用场景。
2. 选项分析
A. Enable AWS Config. Configure an AWS Config managed rule that detects DDoS attacks.
错误：AWS Config主要用于资源配置的合规性检查和审计，不具备检测或防御DDoS攻击的功能。
B. Enable AWS WAF on the ALB. Create an AWS WAF web ACL with rules to detect and prevent DDoS attacks. Associate
the web ACL with the ALB.
部分正确：AWS WAF可以帮助防御某些类型的攻击（如SQL注入、跨站脚本攻击），但它并不是专门设计用于防御大规模DDoS攻
击的解决方案。
不足：AWS WAF无法提供主动的DDoS防护和事件响应服务。
C. Store the ALB access logs in an Amazon S3 bucket. Configure Amazon GuardDuty to detect and take automated
preventative actions for DDoS attacks.
错误：Amazon GuardDuty主要用于检测恶意活动和异常行为（如恶意IP地址、异常流量），但它不具备直接防御DDoS攻击的功
能。
GuardDuty无法提供主动的DDoS防护和事件响应服务。
D. Subscribe to AWS Shield Advanced. Configure hosted zones in Route 53. Add ALB resources as protected resources.
正确：AWS Shield Advanced是AWS提供的专门用于防御DDoS攻击的托管解决方案，包含主动监控、事件响应支持以及对受保护资
源的经济补偿。
它支持与Route 53和ALB集成，提供全面的DDoS防护。
3. 我的答案
D. Subscribe to AWS Shield Advanced. Configure hosted zones in Route 53. Add ALB resources as protected resources.
4. 官方答案
D. Subscribe to AWS Shield Advanced. Configure hosted zones in Route 53. Add ALB resources as protected resources.
