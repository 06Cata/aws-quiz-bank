Question #1017
A company has an application that runs on an Amazon Elastic Kubernetes Service (Amazon EKS) cluster on Amazon EC2 instances.
The application has a UI that uses Amazon DynamoDB and data services that use Amazon S3 as part of the application
deployment.
The company must ensure that the EKS Pods for the UI can access only Amazon DynamoDB and that the EKS Pods for the data
services can access only Amazon S3. The company uses AWS Identity and Access Management (IAM).
Which solution meals these requirements?
一家公司有一个应用程序运行在 Amazon Elastic Kubernetes Service (Amazon EKS) 集群上的 Amazon EC2 实例。该应用程序具有一
个使用 Amazon DynamoDB 的 UI，以及作为应用程序部署一部分的使用 Amazon S3 的数据服务。
公司必须确保 UI 的 EKS Pods 只能访问 Amazon DynamoDB，而数据服务的 EKS Pods 只能访问 Amazon S3。公司使用 AWS
Identity and Access Management (IAM)。
哪种解决方案满足这些要求？

题目分析与解答
1. 考察的知识点
Amazon EKS与IAM集成的最佳实践。
使用IAM Role for Service Accounts (IRSA)为EKS Pods提供细粒度权限控制。
如何为不同的EKS Pods分配特定的权限以访问AWS资源。
2. 选项分析
A. Create separate IAM policies for Amazon S3 and DynamoDB access with the required permissions. Attach both IAM
policies to the EC2 instance profile. Use role-based access control (RBAC) to control access to Amazon S3 or DynamoDB
for the respective EKS Pods.
错误：将IAM策略附加到EC2实例配置文件会导致所有EKS Pods共享相同的权限，无法实现细粒度的权限控制。
RBAC是Kubernetes内部的权限控制机制，无法直接控制AWS资源的访问权限。
B. Create separate IAM policies for Amazon S3 and DynamoDB access with the required permissions. Attach the Amazon
S3 IAM policy directly to the EKS Pods for the data services and the DynamoDB policy to the EKS Pods for the UI.
错误：IAM策略不能直接附加到EKS Pods。AWS中需要通过IAM角色和IRSA机制来实现EKS Pods的权限控制。
C. Create separate Kubernetes service accounts for the UI and data services to assume an IAM role. Attach the
AmazonS3FullAccess policy to the data services account and the AmazonDynamoDBFullAccess policy to the UI service
account.
正确：通过为不同的EKS Pods创建单独的Kubernetes服务账户，并使用IAM角色附加特定的权限策略，可以实现细粒度的权限控
制。
这是AWS推荐的最佳实践，使用IRSA机制将IAM角色与Kubernetes服务账户关联。
D. Create separate Kubernetes service accounts for the UI and data services to assume an IAM role. Use IAM Role for
Service Accounts (IRSA) to provide access to the EKS Pods for the UI to Amazon S3 and the EKS Pods for the data services
to DynamoDB.
错误：选项中的资源分配描述错误。UI Pods应该访问DynamoDB，而数据服务Pods应该访问S3。
虽然提到了IRSA机制，但资源权限分配不符合题目要求。
