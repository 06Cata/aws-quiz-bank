Question #966
A company wants to create an Amazon EMR cluster that multiple teams will use. The company wants to ensure that each team’s
big data workloads can access only the AWS services that each team needs to interact with. The company does not want the
workloads to have access to Instance Metadata Service Version 2 (IMDSv2) on the cluster’s underlying EC2 instances.
Which solution will meet these requirements?
一家公司希望创建一个供多个团队使用的 Amazon EMR 集群。 该公司希望确保每个团队的大数据工作负载只能访问每个团队需要
交互的 AWS 服务。 该公司不希望这些工作负载能够访问集群底层 EC2 实例上的 Instance Metadata Service Version 2 (IMDSv2)。
哪种解决方案可以满足这些要求？
A. Configure interface VPC endpoints for each AWS service that the teams need. Use the required interface VPC endpoints to
submit the big data workloads.
为每个团队需要的 AWS 服务配置接口 VPC 端点。 使用所需的接口 VPC 端点提交大数据工作负载。
B. Create EMR runtime roles. Configure the cluster to use the runtime roles. Use the runtime roles to submit the big data
workloads. Most Voted
创建 EMR 运行时角色。 配置集群以使用运行时角色。 使用运行时角色提交大数据工作负载。
C. Create an EC2 IAM instance profile that has the required permissions for each team. Use the instance profile to submit the
big data workloads.
创建一个 EC2 IAM 实例配置文件，该配置文件具有每个团队所需的权限。 使用实例配置文件提交大数据工作负载。
D. Create an EMR security configuration that has the EnableApplicationScopedIAMRole option set to false. Use the security
configuration to submit the big data workloads.
创建一个 EMR 安全配置，其中 EnableApplicationScopedIAMRole 选项设置为 false。 使用安全配置提交大数据工作负载。
Correct Answer: B

题目分析与解答
1. 考察的知识点
Amazon EMR集群的权限管理与安全配置。
如何为不同团队分配权限以确保安全性和隔离性。
Instance Metadata Service Version 2 (IMDSv2)的使用和限制。
2. 选项分析
A. 配置接口 VPC 端点并使用它们提交大数据工作负载。
正确性分析：虽然VPC端点可以限制对特定AWS服务的访问，但它无法直接解决团队权限隔离的问题，也无法禁用IMDSv2。
结论：此选项错误。
B. 创建EMR运行时角色并配置集群使用这些角色。
正确性分析：EMR运行时角色允许为每个团队分配特定权限，并且可以通过配置来禁用IMDSv2。这是解决问题的正确方法。
结论：此选项正确。
C. 创建EC2 IAM实例配置文件并使用它提交大数据工作负载。
正确性分析：EC2实例配置文件是为EC2实例分配权限的，而不是为EMR集群中的工作负载分配权限。此方法无法实现团队权限隔
离，也无法禁用IMDSv2。
结论：此选项错误。
D. 创建EMR安全配置并设置EnableApplicationScopedIAMRole为false。
正确性分析：此选项涉及EMR安全配置，但设置EnableApplicationScopedIAMRole为false并不能直接解决团队权限隔离的问题，也
无法禁用IMDSv2。
结论：此选项错误。
