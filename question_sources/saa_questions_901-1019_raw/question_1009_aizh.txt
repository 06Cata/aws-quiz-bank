Question #1009
A company runs an environment where data is stored in an Amazon S3 bucket. The objects are accessed frequently throughout
the day. The company has strict da ta encryption requirements for data that is stored in the S3 bucket. The company currently uses
AWS Key Management Service (AWS KMS) for encryption.
The company wants to optimize costs associated with encrypting S3 objects without making additional calls to AWS KMS.
Which solution will meet these requirements?
一家公司运行一个环境，其中数据存储在一个 Amazon S3 存储桶中。对象在一天中频繁访问。该公司对存储在 S3 存储桶中的数据

该公司希望优化与加密 S3 对象相关的成本，同时不需要额外调用 AWS KMS。
哪种解决方案可以满足这些要求？

题目分析与解答
1. 考察的知识点
Amazon S3的加密选项，包括SSE-S3、SSE-KMS、SSE-C和客户端加密。
AWS Key Management Service (KMS)的使用及优化成本的方式。
S3 Bucket Key的功能及其对成本优化的影响。
2. 选项分析
A. 使用 Amazon S3 托管密钥的服务器端加密 (SSE-S3)
错误。SSE-S3使用Amazon S3托管的密钥进行加密，虽然可以满足加密需求，但它不使用AWS KMS，因此无法满足题目中提到
的“当前使用AWS KMS”的要求。
B. 对新对象使用 S3 Bucket Key 和 AWS KMS 密钥的服务器端加密 (SSE-KMS)
正确。SSE-KMS使用AWS KMS进行加密，符合题目要求。启用S3 Bucket Key可以减少对AWS KMS的调用，从而优化成本，同时仍
然满足严格的加密要求。
C. 使用 AWS KMS 客户托管密钥的客户端加密
错误。客户端加密需要在客户端进行加密和解密操作，这会增加复杂性，并且不符合题目中“优化成本”的要求。此外，客户端加密
不涉及S3的服务器端加密选项。
D. 使用存储在 AWS KMS 中的客户提供密钥的服务器端加密 (SSE-C)
错误。SSE-C要求客户提供自己的加密密钥，并且这些密钥不会存储在AWS KMS中。这与题目中提到的“当前使用AWS KMS”的要求
不符。
3. 我选的答案
B. 对新对象使用 S3 Bucket Key 和 AWS KMS 密钥的服务器端加密 (SSE-KMS)
4. 官方答案
B. 对新对象使用 S3 Bucket Key 和 AWS KMS 密钥的服务器端加密 (SSE-KMS)
5. 与官方答案的比较
我的答案与官方答案一致。
原因：S3 Bucket Key通过减少对AWS KMS的调用优化了成本，同时仍然使用AWS KMS进行加密，完全符合题目要求。
