Question #971
A company that is in the ap-northeast-1 Region has a fleet of thousands of AWS Outposts servers. The company has deployed the
servers at remote locations around the world. All the servers regularly download new software versions that consist of 100 files.
There is significant latency before all servers run the new software versions.
The company must reduce the deployment latency for new software versions.
Which solution will meet this requirement with the LEAST operational overhead?

器。 所有服务器定期下载包含 100 个文件的新软件版本。 在所有服务器运行新软件版本之前存在显著的延迟。
该公司必须减少新软件版本的部署延迟。
哪种解决方案能够以最少的运营开销满足此需求？

题目分析与解答
1. 考察的知识点
Amazon S3 的使用和配置
Amazon CloudFront 的分发机制
如何优化全球范围内的内容分发以减少延迟
签名 URL 的使用
2. 选项分析
A. Create an Amazon S3 bucket in ap-northeast-1. Set up an Amazon CloudFront distribution in ap-northeast-1 that
includes a CachingDisabled cache policy. Configure the S3 bucket as the origin. Download the software by using signed
URLs.
分析：
虽然使用了 CloudFront，但禁用缓存（CachingDisabled）会导致每次请求都直接回源服务器，无法利用 CloudFront 的缓存优势。
这会增加延迟，尤其是对于全球范围的分发场景。
因此，这个选项不符合减少部署延迟的要求。
B. Create an Amazon S3 bucket in ap-northeast-1. Create a second S3 bucket in the us-east-1 Region. Configure
replication between the buckets. Set up an Amazon CloudFront distribution that uses ap-northeast-1 as the primary origin
and us-east-1 as the secondary origin. Download the software by using signed URLs.
分析：
配置了两个 S3 bucket 并设置了复制，但这增加了额外的复杂性和操作开销。
虽然 CloudFront 可以使用多个源，但这种配置并不直接解决全球范围的延迟问题。
此选项的操作开销较高，不符合“LEAST operational overhead”的要求。
C. Create an Amazon S3 bucket in ap-northeast-1. Configure Amazon S3 Transfer Acceleration. Download the software by
using the S3 Transfer Acceleration endpoint.
分析：
S3 Transfer Acceleration 可以加速跨区域的上传和下载，但它主要适用于单一客户端与 S3 bucket 的交互。
对于全球范围的分发，CloudFront 的边缘缓存更适合，因为它可以将内容缓存到离用户最近的边缘位置。
此选项虽然减少了延迟，但不如 CloudFront 的分发机制高效。
D. Create an Amazon S3 bucket in ap-northeast-1. Set up an Amazon CloudFront distribution. Configure the S3 bucket as
the origin. Download the software by using signed URLs.
分析：
使用 CloudFront 分发内容，可以将文件缓存到全球范围的边缘位置，从而显著减少延迟。
签名 URL 提供了安全性，确保只有授权用户可以下载文件。
此选项操作简单，符合“LEAST operational overhead”的要求。
