Question #922
A company needs to grant a team of developers access to the company's AWS resources. The company must maintain a high level
of security for the resources.
The company requires an access control solution that will prevent unauthorized access to the sensitive data.
Which solution will meet these requirements?
一家公司需要为一组开发人员提供访问公司AWS资源的权限。公司必须对这些资源保持高水平的安全性。
公司需要一种访问控制解决方案，以防止未经授权访问敏感数据。
哪种解决方案可以满足这些要求？
A. Share the IAM user credentials for each development team member with the rest of the team to simplify access
management and to streamline development workflows.
与开发团队的每个成员共享IAM用户凭证，以简化访问管理并优化开发工作流程。
B. Define IAM roles that have fine-grained permissions based on the principle of least privilege. Assign an IAM role to each
developer. Most Voted
定义基于最小权限原则的精细权限的IAM角色。为每个开发人员分配一个IAM角色。

创建IAM访问密钥以授予对AWS资源的编程访问权限。仅允许开发人员通过使用访问密钥进行API调用与AWS资源交互。

题目分析与解答
1. 考察的知识点
IAM（Identity and Access Management）的使用与最佳实践。
基于最小权限原则的权限管理。
如何安全地管理对AWS资源的访问。
2. 选项分析
A. Share the IAM user credentials for each development team member with the rest of the team to simplify access
management and to streamline development workflows.
错误：共享IAM用户凭证违反了安全最佳实践，因为它会导致凭证泄露的风险。
错误：无法实现对敏感数据的细粒度访问控制。
B. Define IAM roles that have fine-grained permissions based on the principle of least privilege. Assign an IAM role to
each developer.
正确：基于最小权限原则定义IAM角色是AWS推荐的最佳实践。
正确：IAM角色可以提供细粒度的权限控制，并且可以通过临时凭证安全地访问资源。
C. Create IAM access keys to grant programmatic access to AWS resources. Allow only developers to interact with AWS
resources through API calls by using the access keys.
错误：虽然可以通过访问密钥进行编程访问，但长期使用静态访问密钥存在安全风险。
错误：无法实现基于最小权限的细粒度控制。
D. Create an AWS Cognito user pool. Grant developers access to AWS resources by using the user pool.
错误：AWS Cognito主要用于管理应用程序用户身份，而不是直接管理对AWS资源的访问。
错误：无法满足对敏感数据的细粒度权限控制需求。
3. 我的答案
B
4. 官方答案
B
5. 与官方答案比较
我的答案与官方答案一致。
原因：选项B符合AWS的安全最佳实践，基于最小权限原则定义IAM角色可以有效地控制对敏感数据的访问，同时避免凭证泄露的风
险。
