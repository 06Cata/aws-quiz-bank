Question #990
A company is planning to migrate a legacy application to AWS. The application currently uses NFS to communicate to an on-
premises storage solution to store application data. The application cannot be modified to use any other communication
protocols other than NFS for this purpose.
Which storage solution should a solutions architect recommend for use after the migration?
一家公司计划将一个遗留应用程序迁移到AWS。该应用程序当前使用NFS与本地存储解决方案进行通信以存储应用程序数据。该应
用程序无法修改为使用除NFS以外的其他通信协议来完成此目的。
解决方案架构师应推荐在迁移后使用哪种存储解决方案？
A. AWS DataSync
AWS DataSync
B. Amazon Elastic Block Store (Amazon EBS)
Amazon Elastic Block Store (Amazon EBS)
C. Amazon Elastic File System (Amazon EFS) Most Voted
Amazon Elastic File System (Amazon EFS)
D. Amazon EMR File System (Amazon EMRFS)
Amazon EMR File System (Amazon EMRFS)
题目分析与解答
1. 考察的知识点
本题考察的是AWS存储服务的选择，特别是针对NFS协议的支持。
Correct Answer: C
正确答案：C
Community vote distribution
社区投票分布
C (100%)
需要了解AWS存储服务的特性，包括Amazon EFS、Amazon EBS、AWS DataSync和Amazon EMRFS。
2. 选项分析
A. AWS DataSync
错误：AWS DataSync是一种数据迁移服务，用于将数据从本地存储迁移到AWS存储服务（如EFS或S3）。它不提供直接的NFS存储
功能。
B. Amazon Elastic Block Store (Amazon EBS)
错误：Amazon EBS是块存储服务，主要用于与EC2实例配合使用。它不支持NFS协议，无法满足题目中应用的需求。
C. Amazon Elastic File System (Amazon EFS)
正确：Amazon EFS是一个完全托管的文件存储服务，支持NFS协议。它可以直接与迁移后的应用集成，满足题目中应用的需求。
D. Amazon EMR File System (Amazon EMRFS)
错误：Amazon EMRFS是专门为Amazon EMR（Elastic MapReduce）设计的文件系统，主要用于与S3集成。它不支持NFS协议，无
法满足题目中应用的需求。
3. 我的答案
C. Amazon Elastic File System (Amazon EFS)
4. 官方答案
C. Amazon Elastic File System (Amazon EFS)
