A company is designing a web application with an internet-facing Application Load Balancer (ALB).
The company needs the ALB to receive HTTPS web traffic from the public internet. The ALB must send only HTTPS traffic to the
web application servers hosted on the Amazon EC2 instances on port 443. The ALB must perform a health check of the web
application servers over HTTPS on port 8443.
Which combination of configurations of the security group that is associated with the ALB will meet these requirements? (Choose
three.)
A. Allow HTTPS inbound traffic from 0.0.0.0/0 for port 443. Most Voted
B. Allow all outbound traffic to 0.0.0.0/0 for port 443.
C. Allow HTTPS outbound traffic to the web application instances for port 443. Most Voted
D. Allow HTTPS inbound traffic from the web application instances for port 443.
E. Allow HTTPS outbound traffic to the web application instances for the health check on port 8443. Most Voted
F. Allow HTTPS inbound traffic from the web application instances for the health check on port 8443.

This question is poorly worded: assuming that, by default in security groups, all OUTBOUND connections are ALLOWE and all INBOUND connections
are DENIED, options C and E would not even need to be configured. What would be needed is to create a security group for the EC2 instances
allowing INBOUND connections from the ALB security group to the EC2 instances security group on ports 443 and 8443.
