题目分析与解答
1. 考察的知识点
Amazon EKS集群的安全性配置，特别是如何加密Kubernetes etcd存储中的敏感数据。
AWS Key Management Service (KMS)的使用和集成。
Amazon EKS的KMS密钥加密功能。
2. 选项分析
A. 创建一个新的 AWS Key Management Service (AWS KMS) 密钥。使用 AWS Secrets Manager 来管理、轮换和存储 Amazon
EKS 中的所有密钥。
错误：虽然AWS Secrets Manager可以管理和存储密钥，但它与Kubernetes etcd存储的加密无关。此选项未提及如何加密
Kubernetes etcd存储。
B. 创建一个新的 AWS Key Management Service (AWS KMS) 密钥。在 Amazon EKS 集群上启用 Amazon EKS KMS 密钥加密。
正确：此选项直接提到使用AWS KMS密钥来启用Amazon EKS的KMS加密功能，这是加密Kubernetes etcd存储的正确方法。
C. 使用默认选项创建 Amazon EKS 集群。将 Amazon Elastic Block Store (Amazon EBS) 容器存储接口 (CSI) 驱动作为附加组件。
错误：EBS CSI驱动用于管理EBS卷，而不是用于加密Kubernetes etcd存储。
D. 创建一个带有 alias/aws/ebs 别名的新 AWS Key Management Service (AWS KMS) 密钥。为账户启用默认的 Amazon Elastic
Block Store (Amazon EBS) 卷加密。
错误：此选项与EBS卷加密相关，而不是Kubernetes etcd存储的加密。
