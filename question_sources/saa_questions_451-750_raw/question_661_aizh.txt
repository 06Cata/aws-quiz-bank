Question #661
A company runs applications on AWS that connect to the company's Amazon RDS database. The applications scale on weekends
and at peak times of the year. The company wants to scale the database more effectively for its applications that connect to the
database.
Which solution will meet these requirements with the LEAST operational overhead?

题目分析与解答
1. 考察的知识点
Amazon RDS Proxy的功能与优势
数据库连接池的优化
如何减少操作开销并提高应用程序的可扩展性
2. 选项分析
A. 使用 Amazon DynamoDB 和连接池，结合数据库的目标组配置。
错误：Amazon DynamoDB 是一个NoSQL数据库，与Amazon RDS（关系型数据库）完全不同。将应用程序改为使用DynamoDB端
点需要重构应用程序逻辑，且不符合题目要求的“连接到现有RDS数据库”。
B. 使用 Amazon RDS Proxy 和数据库的目标组。
正确：Amazon RDS Proxy是专门设计用于优化数据库连接池的服务。它可以减少数据库连接的开销，支持自动扩展，并且与现有的
RDS数据库无缝集成。使用RDS Proxy可以显著降低操作开销。
C. 使用运行在 Amazon EC2 上的自定义代理作为数据库的中间层。
错误：虽然自定义代理可以实现类似的功能，但需要额外的开发和维护工作，包括管理EC2实例、配置代理软件等。这增加了操作
开销，不符合题目要求的“最少操作开销”。
D. 使用 AWS Lambda 函数提供连接池，结合数据库的目标组配置。
错误：AWS Lambda函数通常用于事件驱动的计算，而不是长期的连接池管理。使用Lambda函数进行连接池管理可能会导致性能问
题，并且需要额外的开发工作。
3. 我的答案
B. 使用 Amazon RDS Proxy 和数据库的目标组。
4. 官方答案
B. 使用 Amazon RDS Proxy 和数据库的目标组。
5. 与官方答案的比较
我的答案与官方答案一致。
原因：Amazon RDS Proxy是专门设计用于优化数据库连接池的服务，能够减少操作开销并提高应用程序的可扩展性，完全符合题目
要求。
