Question #699
A company has an application that uses Docker containers in its local data center. The application runs on a container host that
stores persistent data in a volume on the host. The container instances use the stored persistent data.
The company wants to move the application to a fully managed service because the company does not want to manage any
servers or storage infrastructure.
Which solution will meet these requirements?

题目分析与解答
1. 考察的知识点
了解AWS容器服务的选项，包括Amazon ECS和Amazon EKS。
理解AWS Fargate的无服务器特性。
熟悉AWS存储服务（如EFS和S3）的使用场景。
选择适合的持久性存储解决方案。
2. 选项分析
A. Use Amazon Elastic Kubernetes Service (Amazon EKS) with self-managed nodes. Create an Amazon Elastic Block Store
(Amazon EBS) volume attached to an Amazon EC2 instance. Use the EBS volume as a persistent volume mounted in the
containers.
错误：虽然EKS支持容器编排，但使用EKS和自管理节点需要管理EC2实例和EBS卷，这不符合题目要求的“完全托管服务”。
B. Use Amazon Elastic Container Service (Amazon ECS) with an AWS Fargate launch type. Create an Amazon Elastic File
System (Amazon EFS) volume. Add the EFS volume as a persistent storage volume mounted in the containers.
正确：AWS Fargate是无服务器的容器运行时，完全托管，不需要管理底层服务器。EFS是一个完全托管的文件存储服务，支持持久
性存储，适合挂载到容器中。
C. Use Amazon Elastic Container Service (Amazon ECS) with an AWS Fargate launch type. Create an Amazon S3 bucket.
Map the S3 bucket as a persistent storage volume mounted in the containers.
错误：虽然S3是完全托管的存储服务，但它是对象存储，不适合直接作为容器的持久性存储卷挂载。
D. Use Amazon Elastic Container Service (Amazon ECS) with an Amazon EC2 launch type. Create an Amazon Elastic File
System (Amazon EFS) volume. Add the EFS volume as a persistent storage volume mounted in the containers.
错误：使用EC2启动类型需要管理底层服务器，不符合题目要求的“完全托管服务”。
