Question #464
A company hosts an online shopping application that stores all orders in an Amazon RDS for PostgreSQL Single-AZ DB instance.
Management wants to eliminate single points of failure and has asked a solutions architect to recommend an approach to
minimize database downtime without requiring any changes to the application code.
Which solution meets these requirements?
一家公司托管了一个在线购物应用程序，该应用程序将所有订单存储在一个 Amazon RDS for PostgreSQL Single-AZ DB 实例中。管
理层希望消除单点故障，并要求解决方案架构师推荐一种方法，以尽量减少数据库停机时间，而无需对应用程序代码进行任何更
改。
哪种解决方案满足这些要求？
A. Convert the existing database instance to a Multi-AZ deployment by modifying the database instance and specifying the

通过修改数据库实例并指定 Multi-AZ 选项，将现有的数据库实例转换为 Multi-AZ 部署。

deployment with the snapshot.
创建一个新的 RDS Multi-AZ 部署。对当前的 RDS 实例进行快照，并使用该快照恢复新的 Multi-AZ 部署。

题目分析与解答
1. 考察的知识点
Amazon RDS的高可用性配置，特别是Multi-AZ部署。
如何最小化数据库停机时间。
无需更改应用程序代码的解决方案。
2. 选项分析
A. Convert the existing database instance to a Multi-AZ deployment by modifying the database instance and specifying
the Multi-AZ option.
正确：Amazon RDS支持通过修改现有数据库实例来启用Multi-AZ部署。这种方法可以最小化停机时间，并且不需要更改应用程序
代码。
符合题目要求：消除单点故障，最小化数据库停机时间，无需更改应用程序代码。
B. Create a new RDS Multi-AZ deployment. Take a snapshot of the current RDS instance and restore the new Multi-AZ
deployment with the snapshot.
错误：虽然可以通过快照恢复到一个新的Multi-AZ部署，但此过程会导致较长的停机时间，因为需要创建新实例并恢复数据。
不符合题目要求：无法最小化数据库停机时间。
C. Create a read-only replica of the PostgreSQL database in another Availability Zone. Use Amazon Route 53 weighted
record sets to distribute requests across the databases.
错误：只读副本不能用于写操作，因此无法完全消除单点故障。
不符合题目要求：无法解决主数据库的高可用性问题。
D. Place the RDS for PostgreSQL database in an Amazon EC2 Auto Scaling group with a minimum group size of two. Use
Amazon Route 53 weighted record sets to distribute requests across instances.
错误：RDS数据库不能直接放置在EC2 Auto Scaling组中。此选项与RDS的工作机制不符。
不符合题目要求：无法实现RDS的高可用性。
