Question #746
A company is deploying an application that processes streaming data in near-real time. The company plans to use Amazon EC2
instances for the workload. The network architecture must be configurable to provide the lowest possible latency between nodes.
Which combination of network solutions will meet these requirements? (Choose two.)
一家公司正在部署一个处理近实时流数据的应用程序。该公司计划使用 Amazon EC2 实例来运行该工作负载。网络架构必须可配
置，以提供节点之间尽可能低的延迟。
哪些网络解决方案的组合可以满足这些要求？（选择两个。）
A. Enable and configure enhanced networking on each EC2 instance. Most Voted

将 EC2 实例分组到不同的账户中。

在集群放置组中运行 EC2 实例。

为每个 EC2 实例附加多个弹性网络接口。
E. Use Amazon Elastic Block Store (Amazon EBS) optimized instance types.
使用 Amazon Elastic Block Store（Amazon EBS）优化的实例类型。

Amazon EC2增强型网络（Enhanced Networking）的配置与作用。
集群放置组（Cluster Placement Group）的使用场景及优势。
如何优化网络架构以实现最低延迟。
2. 选项分析
A. Enable and configure enhanced networking on each EC2 instance.
正确：增强型网络（Enhanced Networking）通过使用高性能网络接口（如Elastic Network Adapter或ENA）显著降低网络延迟并提
高吞吐量。
这是实现最低网络延迟的关键配置之一。
B. Group the EC2 instances in separate accounts.
错误：将实例分组到不同账户中不会降低网络延迟，反而可能增加跨账户通信的复杂性。
与题目要求的“最低可能的延迟”无关。
C. Run the EC2 instances in a cluster placement group.
正确：集群放置组（Cluster Placement Group）允许EC2实例在物理硬件上紧密分布，从而显著降低网络延迟。
这是实现最低延迟的另一关键配置。
D. Attach multiple elastic network interfaces to each EC2 instance.
错误：虽然附加多个网络接口可以增加网络带宽，但它不会直接降低节点之间的网络延迟。
题目要求的是最低延迟，而不是带宽优化。
E. Use Amazon Elastic Block Store (Amazon EBS) optimized instance types.
错误：EBS优化实例类型主要用于提高存储性能，与网络延迟无关。
题目要求的是网络架构优化，而非存储性能优化。
