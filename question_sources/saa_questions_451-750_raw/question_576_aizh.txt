Question #576
A company is building a RESTful serverless web application on AWS by using Amazon API Gateway and AWS Lambda. The users of
this web application will be geographically distributed, and the company wants to reduce the latency of API requests to these
users.
Which type of endpoint should a solutions architect use to meet these requirements?

的用户将分布在地理上，公司希望减少这些用户的 API 请求延迟。

题目分析与解答
1. 考察的知识点
本题考察的是Amazon API Gateway的不同类型的端点，包括私有端点、区域端点、接口 VPC 端点和边缘优化端点。
Correct Answer: D
正确答案：D
Community vote distribution
社区投票分布
D (100%)
重点在于如何选择合适的端点以优化用户的访问延迟，尤其是针对地理分布广泛的用户。
2. 选项分析
A. Private endpoint (私有端点)
私有端点用于在VPC内访问API Gateway，适合内部应用或需要高安全性场景。
不适合地理分布广泛的用户，因为它无法通过CloudFront优化延迟。
错误原因：无法满足减少全球用户访问延迟的需求。
B. Regional endpoint (区域端点)
区域端点直接在指定的AWS区域内提供API Gateway服务。
适合区域内的用户，但对于地理分布广泛的用户，延迟可能较高。
错误原因：无法通过全球分布的边缘位置优化延迟。
C. Interface VPC endpoint (接口 VPC 端点)
接口 VPC 端点用于通过AWS PrivateLink在VPC内访问服务。
主要用于安全性和私有网络访问场景，与减少全球用户延迟无关。
错误原因：不适合优化全球用户的访问延迟。
D. Edge-optimized endpoint (边缘优化端点)
边缘优化端点通过Amazon CloudFront将API Gateway的流量分发到全球的边缘位置。
适合地理分布广泛的用户，可以显著减少延迟。
正确原因：满足减少全球用户访问延迟的需求。
3. 我的答案
答案： D. Edge-optimized endpoint (边缘优化端点)
4. 官方答案
答案： D. Edge-optimized endpoint (边缘优化端点)
