Question #518
An application uses an Amazon RDS MySQL DB instance. The RDS database is becoming low on disk space. A solutions architect
wants to increase the disk space without downtime.
Which solution meets these requirements with the LEAST amount of effort?
一个应用程序使用了一个 Amazon RDS MySQL 数据库实例。该 RDS 数据库的磁盘空间即将耗尽。一位解决方案架构师希望在不中
断的情况下增加磁盘空间。
哪种解决方案以最少的努力满足这些要求？
A. Enable storage autoscaling in RDS Most Voted
在 RDS 中启用存储自动扩展
B. Increase the RDS database instance size
增加 RDS 数据库实例的大小
C. Change the RDS database instance storage type to Provisioned IOPS
将 RDS 数据库实例的存储类型更改为 Provisioned IOPS
D. Back up the RDS database, increase the storage capacity, restore the database, and stop the previous instance
备份 RDS 数据库，增加存储容量，恢复数据库，并停止之前的实例
题目分析与解答
1. 考察的知识点
Amazon RDS的存储管理功能，包括存储自动扩展（Storage Autoscaling）。
如何在不影响应用程序可用性的情况下扩展存储容量。
Correct Answer: A
正确答案：A
Community vote distribution
社区投票分布
A (100%)
2. 选项分析
A. Enable storage autoscaling in RDS
正确。Amazon RDS支持存储自动扩展功能，可以在数据库存储空间不足时自动增加存储容量，而无需停机。这是最简单且无中断
的方法。
B. Increase the RDS database instance size
错误。增加实例大小（例如从小型实例升级到大型实例）主要影响计算能力（CPU和内存），而不是存储容量。此选项无法解决存
储空间不足的问题。
C. Change the RDS database instance storage type to Provisioned IOPS
错误。更改存储类型（例如切换到Provisioned IOPS）可以提高性能，但不会直接增加存储容量。此外，更改存储类型可能需要停
机。
D. Back up the RDS database, increase the storage capacity, restore the database, and stop the previous instance
错误。此方法需要手动备份和恢复数据库，并且会导致停机时间，不符合题目要求的“无停机”条件。
3. 我的答案
A. Enable storage autoscaling in RDS
4. 官方答案
A. Enable storage autoscaling in RDS

我的答案与官方答案一致，没有差异。
