Question #601
A company runs its critical database on an Amazon RDS for PostgreSQL DB instance. The company wants to migrate to Amazon
Aurora PostgreSQL with minimal downtime and data loss.
Which solution will meet these requirements with the LEAST operational overhead?
一家公司在 Amazon RDS for PostgreSQL 数据库实例上运行其关键数据库。该公司希望以最小的停机时间和数据丢失迁移到
Amazon Aurora PostgreSQL。
哪种解决方案能够以最少的操作开销满足这些需求？
A. Create a DB snapshot of the RDS for PostgreSQL DB instance to populate a new Aurora PostgreSQL DB cluster.
创建 RDS for PostgreSQL 数据库实例的 DB 快照，以填充新的 Aurora PostgreSQL 数据库集群。
B. Create an Aurora read replica of the RDS for PostgreSQL DB instance. Promote the Aurora read replicate to a new Aurora
PostgreSQL DB cluster. Most Voted
创建 RDS for PostgreSQL 数据库实例的 Aurora 只读副本。将 Aurora 只读副本提升为新的 Aurora PostgreSQL 数据库集群。
C. Use data import from Amazon S3 to migrate the database to an Aurora PostgreSQL DB cluster.
使用来自 Amazon S3 的数据导入将数据库迁移到 Aurora PostgreSQL 数据库集群。
D. Use the pg_dump utility to back up the RDS for PostgreSQL database. Restore the backup to a new Aurora PostgreSQL DB
cluster.
使用 pg_dump 工具备份 RDS for PostgreSQL 数据库。将备份恢复到新的 Aurora PostgreSQL 数据库集群。

2. 选项分析
A. Create a DB snapshot of the RDS for PostgreSQL DB instance to populate a new Aurora PostgreSQL DB cluster.
错误：使用快照迁移会导致较长的停机时间，因为快照是静态的，无法实时更新数据。
不符合“最小停机时间和数据丢失”的要求。
B. Create an Aurora read replica of the RDS for PostgreSQL DB instance. Promote the Aurora read replicate to a new
Aurora PostgreSQL DB cluster.
正确：Aurora支持直接创建RDS PostgreSQL的只读副本，并且可以将其提升为独立的Aurora集群。
这种方法可以实现实时数据同步，确保最小的停机时间和数据丢失。
操作简单，符合“最小操作开销”的要求。
C. Use data import from Amazon S3 to migrate the database to an Aurora PostgreSQL DB cluster.
错误：使用S3导入数据需要额外的步骤，例如将数据导出到S3并配置导入任务。
这种方法无法实现实时数据同步，可能导致数据丢失。
操作复杂，不符合“最小操作开销”的要求。
D. Use the pg_dump utility to back up the RDS for PostgreSQL database. Restore the backup to a new Aurora PostgreSQL
DB cluster.
错误：pg_dump和恢复过程需要较长时间，且无法实现实时数据同步。
这种方法会导致较长的停机时间，不符合“最小停机时间”的要求。

Aurora PostgreSQL DB cluster.
