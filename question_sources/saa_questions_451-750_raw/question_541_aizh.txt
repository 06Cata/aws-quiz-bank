Question #541

a long time. Only customers who have paid a subscription fee can have the ability to sign in and use the web application.
Which combination of steps will meet these requirements MOST cost-effectively? (Choose three.)
一家公司希望在 AWS 上构建一个 Web 应用程序。客户访问网站的请求不可预测，并且可能会长时间处于空闲状态。只有支付订阅
费用的客户才能登录并使用该 Web 应用程序。
哪些步骤组合可以最具成本效益地满足这些要求？（选择三项）
A. Create an AWS Lambda function to retrieve user information from Amazon DynamoDB. Create an Amazon API Gateway

Lambda function.
创建一个 Amazon Elastic Container Service (Amazon ECS) 服务，放置在一个 Application Load Balancer 后面以从 Amazon RDS
检索用户信息。创建一个 Amazon API Gateway 端点以接受 RESTful API。将 API 调用发送到 Lambda 函数。

创建一个 Amazon Cognito 身份池以验证用户身份。

使用 AWS Amplify 提供带有 HTML、CSS 和 JS 的前端 Web 内容。使用集成的 Amazon CloudFront 配置。

正确答案：ACE
题目分析与解答
1. 考察的知识点
无服务器架构（Serverless Architecture）
用户认证与授权（Authentication and Authorization）
静态网站托管与内容分发（Static Website Hosting and Content Delivery）
成本优化（Cost Optimization）
2. 选项分析
A. Create an AWS Lambda function to retrieve user information from Amazon DynamoDB. Create an Amazon API Gateway
endpoint to accept RESTful APIs. Send the API calls to the Lambda function.
正确：使用AWS Lambda和API Gateway是无服务器架构的核心组件，能够按需运行代码并处理API请求，且成本低廉。
Lambda函数可以从DynamoDB中检索用户信息，满足动态数据需求。
B. Create an Amazon Elastic Container Service (Amazon ECS) service behind an Application Load Balancer to retrieve user
information from Amazon RDS. Create an Amazon API Gateway endpoint to accept RESTful APIs. Send the API calls to the
Lambda function.
错误：ECS和RDS的组合适用于更复杂的应用场景，但成本较高，不符合题目中“最具成本效益”的要求。
题目中没有提到需要使用容器化服务，因此不必要引入ECS。
C. Create an Amazon Cognito user pool to authenticate users.
正确：Amazon Cognito用户池是专门用于用户身份验证的服务，能够轻松管理用户注册、登录和访问控制。
用户池支持付费用户的身份验证需求，符合题目要求。
D. Create an Amazon Cognito identity pool to authenticate users.
错误：身份池主要用于临时访问AWS资源，而不是用户身份验证。题目要求的是用户认证，而不是资源访问授权。
E. Use AWS Amplify to serve the frontend web content with HTML, CSS, and JS. Use an integrated Amazon CloudFront
configuration.
正确：AWS Amplify可以快速部署前端内容，并集成CloudFront进行内容分发，提供低延迟和高性能的用户体验。
Amplify是构建现代Web应用的理想选择，且成本较低。
F. Use Amazon S3 static web hosting with PHP, CSS, and JS. Use Amazon CloudFront to serve the frontend web content.
错误：Amazon S3静态网站托管不支持动态内容（如PHP），而题目中提到需要用户登录功能，这需要动态处理。
虽然S3和CloudFront可以用于静态内容分发，但不适合此场景。
