Question #645
A company is required to use cryptographic keys in its on-premises key manager. The key manager is outside of the AWS Cloud
because of regulatory and compliance requirements. The company wants to manage encryption and decryption by using
cryptographic keys that are retained outside of the AWS Cloud and that support a variety of external key managers from different
vendors.
Which solution will meet these requirements with the LEAST operational overhead?
一家公司被要求在其本地密钥管理器中使用加密密钥。由于法规和合规性要求，该密钥管理器位于AWS云之外。该公司希望通过使
用保留在AWS云之外的加密密钥来管理加密和解密，并支持来自不同供应商的各种外部密钥管理器。
哪种解决方案能够以最少的操作开销满足这些要求？
A. Use AWS CloudHSM key store backed by a CloudHSM cluster.
使用由CloudHSM集群支持的AWS CloudHSM密钥存储。
B. Use an AWS Key Management Service (AWS KMS) external key store backed by an external key manager. Most Voted
使用由外部密钥管理器支持的AWS Key Management Service (AWS KMS) 外部密钥存储。
C. Use the default AWS Key Management Service (AWS KMS) managed key store.
使用默认的AWS Key Management Service (AWS KMS) 托管密钥存储。
D. Use a custom key store backed by an AWS CloudHSM cluster.
使用由AWS CloudHSM集群支持的自定义密钥存储。

Correct Answer: B

本题考察的是AWS KMS（Key Management Service）的功能和集成选项，特别是如何使用外部密钥管理器来满足合规性要求。
涉及的知识点包括：AWS KMS外部密钥存储（External Key Store）、CloudHSM、默认托管密钥存储和自定义密钥存储。
2. 选项分析
A. 使用由CloudHSM集群支持的AWS CloudHSM密钥存储。
CloudHSM是一个托管的硬件安全模块（HSM）服务，可以帮助用户管理自己的加密密钥。
然而，题目要求使用外部的密钥管理器，而CloudHSM是AWS内部的服务，不符合题目要求。
错误原因：不满足“密钥存储在AWS之外”的要求。
B. 使用由外部密钥管理器支持的AWS Key Management Service (AWS KMS) 外部密钥存储。
AWS KMS外部密钥存储（External Key Store, XKS）允许用户将加密密钥存储在AWS之外的外部密钥管理器中。
此选项完全符合题目要求：支持外部密钥管理器，满足合规性要求，并且操作开销最低。
正确原因：完全满足题目要求，并且是AWS推荐的解决方案。
C. 使用默认的AWS Key Management Service (AWS KMS) 托管密钥存储。
默认的AWS KMS托管密钥存储是AWS管理的密钥存储服务，密钥存储在AWS内部。
题目要求密钥存储在AWS之外，因此此选项不符合要求。
错误原因：密钥存储在AWS内部，不符合题目要求。
D. 使用由AWS CloudHSM集群支持的自定义密钥存储。
自定义密钥存储允许用户使用CloudHSM集群来管理自己的密钥。
然而，CloudHSM仍然是AWS内部的服务，不符合题目要求。
错误原因：密钥存储在AWS内部，不符合题目要求。
