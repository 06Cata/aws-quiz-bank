Question #615
A company runs a critical, customer-facing application on Amazon Elastic Kubernetes Service (Amazon EKS). The application has a
microservices architecture. The company needs to implement a solution that collects, aggregates, and summarizes metrics and
logs from the application in a centralized location.
Which solution meets these requirements?
一家公司在 Amazon Elastic Kubernetes Service (Amazon EKS) 上运行一个关键的面向客户的应用程序。该应用程序采用微服务架
构。公司需要实施一个解决方案，以在集中位置收集、汇总和总结来自应用程序的指标和日志。
哪种解决方案满足这些需求？
A. Run the Amazon CloudWatch agent in the existing EKS cluster. View the metrics and logs in the CloudWatch console.
在现有的 EKS 集群中运行 Amazon CloudWatch agent。在 CloudWatch 控制台中查看指标和日志。
B. Run AWS App Mesh in the existing EKS cluster. View the metrics and logs in the App Mesh console.
在现有的 EKS 集群中运行 AWS App Mesh。在 App Mesh 控制台中查看指标和日志。
C. Configure AWS CloudTrail to capture data events. Query CloudTrail by using Amazon OpenSearch Service.
配置 AWS CloudTrail 来捕获数据事件。通过使用 Amazon OpenSearch Service 查询 CloudTrail。
D. Configure Amazon CloudWatch Container Insights in the existing EKS cluster. View the metrics and logs in the CloudWatch
console. Most Voted

2. 选项分析
A. Run the Amazon CloudWatch agent in the existing EKS cluster. View the metrics and logs in the CloudWatch console.
错误：虽然CloudWatch agent可以收集指标和日志，但它并不是专门为容器化环境设计的解决方案。
CloudWatch Container Insights是更适合EKS的工具，因为它可以自动收集容器级别的指标和日志。
B. Run AWS App Mesh in the existing EKS cluster. View the metrics and logs in the App Mesh console.
错误：AWS App Mesh主要用于服务网格的流量管理和监控，而不是专门用于集中收集和汇总日志与指标。
App Mesh的功能与题目要求不完全匹配。
C. Configure AWS CloudTrail to capture data events. Query CloudTrail by using Amazon OpenSearch Service.
错误：CloudTrail主要用于记录API调用和事件，而不是用于收集应用程序的运行时指标和日志。
OpenSearch Service可以查询日志，但这不是题目要求的最佳解决方案。
D. Configure Amazon CloudWatch Container Insights in the existing EKS cluster. View the metrics and logs in the
CloudWatch console.
正确：CloudWatch Container Insights是专门为容器化环境设计的解决方案，可以自动收集和汇总EKS集群的指标和日志。
它支持微服务架构，并且可以在CloudWatch控制台中集中查看数据。
