The company has AWS CloudTrail turned on.
Which solution will meet these requirements with the LEAST effort?

哪种解决方案能够以最少的努力满足这些需求？
A. Use AWS Glue and write custom scripts to query CloudTrail logs for the errors.

题目分析与解答
1. 考察的知识点
本题考察的是如何高效分析和排查与IAM权限相关的错误（Access Denied和Unauthorized），以及如何利用AWS服务（如
CloudTrail、Athena等）进行日志查询和分析。
2. 选项分析
Correct Answer: C
正确答案：C
Community vote distribution
社区投票分布
C (77%)
D (23%)
A. 使用AWS Glue并编写自定义脚本查询CloudTrail日志以查找错误。
错误原因：AWS Glue主要用于ETL（提取、转换、加载）任务，适合处理结构化和半结构化数据，但需要编写复杂的自定义脚本，
工作量较大，不符合“最少努力”的要求。
B. 使用AWS Batch并编写自定义脚本查询CloudTrail日志以查找错误。
错误原因：AWS Batch主要用于运行批处理作业，适合大规模计算任务，但需要编写脚本和管理作业队列，工作量较大，不符合“最
少努力”的要求。
C. 使用Amazon Athena查询CloudTrail日志以识别错误。
正确原因：Amazon Athena是一种交互式查询服务，可以直接查询存储在S3中的CloudTrail日志，无需预处理或编写复杂脚本。它支
持SQL查询，能够快速定位IAM相关的错误，符合“最少努力”的要求。
D. 使用Amazon QuickSight查询CloudTrail日志。创建一个仪表板以识别错误。
错误原因：Amazon QuickSight主要用于数据可视化和创建仪表板，虽然可以用来分析数据，但需要额外的配置和数据建模，工作
量较大，不符合“最少努力”的要求。
3. 我选的答案
C. 使用Amazon Athena查询CloudTrail日志以识别错误。
4. 官方答案
C. 使用Amazon Athena查询CloudTrail日志以识别错误。
