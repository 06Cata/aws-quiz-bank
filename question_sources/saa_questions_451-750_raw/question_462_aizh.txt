Question #462
A company has an application that processes customer orders. The company hosts the application on an Amazon EC2 instance
that saves the orders to an Amazon Aurora database. Occasionally when traffic is high the workload does not process orders fast
enough.
What should a solutions architect do to write the orders reliably to the database as quickly as possible?
一家公司有一个处理客户订单的应用程序。该公司将应用程序托管在一个 Amazon EC2 实例上，该实例将订单保存到 Amazon
Aurora 数据库中。偶尔当流量较高时，工作负载无法足够快地处理订单。
解决方案架构师应该怎么做才能尽可能快速可靠地将订单写入数据库？
A. Increase the instance size of the EC2 instance when traffic is high. Write orders to Amazon Simple Notification Service
(Amazon SNS). Subscribe the database endpoint to the SNS topic.
在流量较高时增加 EC2 实例的大小。将订单写入 Amazon Simple Notification Service (Amazon SNS)。将数据库端点订阅到 SNS
主题。
B. Write orders to an Amazon Simple Queue Service (Amazon SQS) queue. Use EC2 instances in an Auto Scaling group behind
an Application Load Balancer to read from the SQS queue and process orders into the database. Most Voted
将订单写入 Amazon Simple Queue Service (Amazon SQS) 队列。在一个带有 Application Load Balancer 的 Auto Scaling 组中的
EC2 实例中读取 SQS 队列并将订单处理到数据库中。
C. Write orders to Amazon Simple Notification Service (Amazon SNS). Subscribe the database endpoint to the SNS topic. Use
EC2 instances in an Auto Scaling group behind an Application Load Balancer to read from the SNS topic.
将订单写入 Amazon Simple Notification Service (Amazon SNS)。将数据库端点订阅到 SNS 主题。使用一个带有 Application
Load Balancer 的 Auto Scaling 组中的 EC2 实例从 SNS 主题中读取并处理订单。
D. Write orders to an Amazon Simple Queue Service (Amazon SQS) queue when the EC2 instance reaches CPU threshold limits.
Use scheduled scaling of EC2 instances in an Auto Scaling group behind an Application Load Balancer to read from the SQS
queue and process orders into the database.
当 EC2 实例达到 CPU 阈值限制时，将订单写入 Amazon Simple Queue Service (Amazon SQS) 队列。使用一个带有 Application
Load Balancer 的 Auto Scaling 组中的 EC2 实例的计划扩展来读取 SQS 队列并将订单处理到数据库中。
Correct Answer: B
正确答案：B
题目分析与解答
1. 考察的知识点
Amazon EC2 Auto Scaling的使用
Amazon SQS队列的应用场景
高可用性和可扩展性架构设计
负载均衡器（Application Load Balancer）的作用
2. 选项分析
A. 增加 EC2 实例大小并使用 Amazon SNS：
错误。虽然增加实例大小可以提高单个实例的处理能力，但这不是一个可扩展的解决方案。此外，Amazon SNS是一个发布/订阅服
务，无法直接保证消息的顺序或可靠性，数据库端点也不能直接订阅SNS主题。
B. 使用 Amazon SQS 和 Auto Scaling：
正确。Amazon SQS是一个可靠的消息队列服务，可以缓冲订单请求，确保消息不会丢失。使用Auto Scaling组和负载均衡器可以动
态扩展EC2实例以处理高流量，确保订单处理的可靠性和效率。
C. 使用 Amazon SNS 和 Auto Scaling：
错误。Amazon SNS是一个发布/订阅服务，不适合用作消息队列。它无法保证消息的顺序或可靠性，且数据库端点不能直接订阅
SNS主题。
D. 使用 Amazon SQS和计划扩展：
错误。计划扩展是基于预定义的时间表，而不是实时流量变化。这种方法无法动态响应流量高峰，可能导致订单处理延迟。

答案：B
