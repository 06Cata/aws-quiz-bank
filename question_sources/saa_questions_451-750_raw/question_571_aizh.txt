API endpoints. The company also requires a specific public third-party certificate authority (CA) to sign the TLS certificate.
Which solution will meet these requirements?

方证书颁发机构（CA）来签署TLS证书。

题目分析与解答
1. 考察的知识点
使用AWS Certificate Manager (ACM)管理证书。
如何配置API Gateway的自定义域名。
TLS协议版本的支持与第三方CA证书的使用。
2. 选项分析
A. 使用本地机器创建一个由第三方CA签署的证书。将证书导入AWS Certificate Manager (ACM)。在Amazon API Gateway中创
建一个具有自定义域名的HTTP API。配置自定义域名以使用该证书。
正确：AWS Certificate Manager (ACM)支持导入由第三方CA签署的证书。
正确：API Gateway支持通过自定义域名使用ACM中的证书。
正确：可以通过API Gateway的自定义域名配置TLSv1.3。
B. 在AWS Certificate Manager (ACM)中创建一个由第三方CA签署的证书。在Amazon API Gateway中创建一个具有自定义域名
的HTTP API。配置自定义域名以使用该证书。
错误：AWS Certificate Manager (ACM)无法直接创建由第三方CA签署的证书。ACM只能创建由Amazon签署的证书。
C. 使用AWS Certificate Manager (ACM)创建一个由第三方CA签署的证书。将证书导入AWS Certificate Manager (ACM)。创建
一个具有Lambda函数URL的AWS Lambda函数。配置Lambda函数URL以使用该证书。
错误：Lambda函数URL不支持自定义域名或ACM证书配置。
错误：Lambda函数URL无法满足TLSv1.3的严格要求。

数。配置Lambda函数URL以使用该证书。
错误：AWS Certificate Manager (ACM)无法直接创建由第三方CA签署的证书。
错误：Lambda函数URL不支持自定义域名或ACM证书配置。
