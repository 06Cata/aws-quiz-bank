Question #466
A company designed a stateless two-tier application that uses Amazon EC2 in a single Availability Zone and an Amazon RDS
Multi-AZ DB instance. New company management wants to ensure the application is highly available.
What should a solutions architect do to meet this requirement?
一家公司设计了一个无状态的两层应用程序，该应用程序使用单个可用区中的 Amazon EC2 和一个 Amazon RDS Multi-AZ DB 实
例。新的公司管理层希望确保该应用程序具有高可用性。
解决方案架构师应该怎么做来满足这一要求？
A. Configure the application to use Multi-AZ EC2 Auto Scaling and create an Application Load Balancer Most Voted
将应用程序配置为使用 Multi-AZ EC2 Auto Scaling，并创建一个 Application Load Balancer
B. Configure the application to take snapshots of the EC2 instances and send them to a different AWS Region
将应用程序配置为对 EC2 实例进行快照并将其发送到另一个 AWS Region
C. Configure the application to use Amazon Route 53 latency-based routing to feed requests to the application
将应用程序配置为使用 Amazon Route 53 基于延迟的路由来处理应用程序的请求
D. Configure Amazon Route 53 rules to handle incoming requests and create a Multi-AZ Application Load Balancer
配置 Amazon Route 53 规则来处理传入请求，并创建一个 Multi-AZ Application Load Balancer
题目分析与解答
1. 考察的知识点
高可用架构设计：如何设计一个高可用的应用程序架构，包括使用多可用区（Multi-AZ）和负载均衡。
AWS服务：Amazon EC2 Auto Scaling、Application Load Balancer、Amazon Route 53、Amazon RDS Multi-AZ。
Correct Answer: A
正确答案：A
Community vote distribution
社区投票分布
A (100%)
2. 选项分析
A. Configure the application to use Multi-AZ EC2 Auto Scaling and create an Application Load Balancer
正确：使用Multi-AZ EC2 Auto Scaling可以确保实例在多个可用区中运行，从而提高应用程序的高可用性。
正确：创建Application Load Balancer可以分发流量到多个实例，进一步增强高可用性。
B. Configure the application to take snapshots of the EC2 instances and send them to a different AWS Region
错误：快照仅用于备份和灾难恢复，不直接提高应用程序的高可用性。
错误：发送快照到另一个AWS Region是跨区域备份，与高可用性无关。
C. Configure the application to use Amazon Route 53 latency-based routing to feed requests to the application
错误：基于延迟的路由可以优化用户请求的响应时间，但它不能解决单可用区的问题。
错误：此选项没有提到如何实现多可用区的高可用性。
D. Configure Amazon Route 53 rules to handle incoming requests and create a Multi-AZ Application Load Balancer
部分正确：创建Multi-AZ Application Load Balancer确实可以提高高可用性。
错误：仅配置Route 53规则不足以解决单可用区的问题，缺少EC2 Auto Scaling的支持。

A
