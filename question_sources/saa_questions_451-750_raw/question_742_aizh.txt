题目分析与解答
1. 考察的知识点
本题考察的是AWS服务的集成使用，尤其是如何管理应用程序配置和安全存储敏感信息（如数据库凭证）。
涉及的服务包括：AWS AppConfig、AWS Secrets Manager、AWS Systems Manager Parameter Store、Amazon S3、Amazon
RDS。
2. 选项分析
A. Use AWS AppConfig to store and manage the application configuration. Use AWS Secrets Manager to store and
retrieve the credentials.
正确：
AWS AppConfig是专门设计用于管理和分发应用程序配置的服务，能够动态更新配置，减少停机时间。
AWS Secrets Manager是专门用于安全存储和管理敏感信息（如数据库凭证）的服务，支持自动轮换凭证，减少管理开销。
此选项结合了两个服务的最佳实践，满足了题目要求，并且具有最低的管理开销。
B. Use AWS Lambda to store and manage the application configuration. Use AWS Systems Manager Parameter Store to
store and retrieve the credentials.
错误：
AWS Lambda不是专门设计用于存储和管理应用程序配置的服务，虽然可以通过代码实现，但会增加开发和管理复杂性。
AWS Systems Manager Parameter Store可以存储凭证，但与Secrets Manager相比，缺乏自动轮换功能，管理开销更高。
C. Use an encrypted application configuration file. Store the file in Amazon S3 for the application configuration. Create
another S3 file to store and retrieve the credentials.
错误：
虽然Amazon S3可以存储文件，但它不是专门设计用于动态管理应用程序配置的服务，缺乏版本控制和动态更新功能。
存储敏感信息（如凭证）在S3中需要额外的加密和访问控制，管理开销较高。
D. Use AWS AppConfig to store and manage the application configuration. Use Amazon RDS to store and retrieve the
credentials.
错误：
AWS AppConfig适合管理应用程序配置，但Amazon RDS不是专门设计用于存储和管理敏感信息（如凭证）的服务。
将凭证存储在RDS中会增加开发复杂性，并且不符合最佳实践。
