Question #481
A company hosts a three-tier web application in the AWS Cloud. A Multi-AZAmazon RDS for MySQL server forms the database
layer Amazon ElastiCache forms the cache layer. The company wants a caching strategy that adds or updates data in the cache
when a customer adds an item to the database. The data in the cache must always match the data in the database.
Which solution will meet these requirements?

题目分析与解答
1. 考察的知识点
Correct Answer: B
正确答案：B
Community vote distribution
社区投票分布
B (100%)
本题考察的是缓存策略的选择与实现，特别是如何确保缓存与数据库数据的一致性。
涉及的AWS服务包括Amazon RDS和Amazon ElastiCache。
2. 选项分析
A. Implement the lazy loading caching strategy
Lazy loading是一种缓存策略，只有在数据被请求时才会从数据库加载到缓存中。
这种策略无法确保缓存与数据库数据的一致性，因为数据更新后不会自动更新到缓存中。
因此此选项错误。
B. Implement the write-through caching strategy
Write-through是一种缓存策略，在数据写入数据库时同时写入缓存。
这种策略可以确保缓存与数据库数据的一致性，因为每次更新都会同步到缓存。
此选项正确。
C. Implement the adding TTL caching strategy
Adding TTL是一种设置缓存过期时间的策略，数据在缓存中存储一段时间后会自动过期。
虽然可以减少缓存中的过期数据，但无法确保缓存与数据库数据的一致性。
因此此选项错误。
D. Implement the AWS AppConfig caching strategy
AWS AppConfig主要用于管理应用程序配置，而不是专门用于缓存数据。
此选项与题目要求无关，因此错误。
3. 我选的答案
B. Implement the write-through caching strategy
4. 官方答案
B. Implement the write-through caching strategy
