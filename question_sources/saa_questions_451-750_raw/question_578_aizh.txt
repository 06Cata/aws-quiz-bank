Question #578
A company deployed a serverless application that uses Amazon DynamoDB as a database layer. The application has experienced a
large increase in users. The company wants to improve database response time from milliseconds to microseconds and to cache
requests to the database.
Which solution will meet these requirements with the LEAST operational overhead?

2. 选项分析
A. Use DynamoDB Accelerator (DAX)
正确：DAX是专门为DynamoDB设计的内存缓存服务，可以将响应时间从毫秒级降低到微秒级。
它与DynamoDB无缝集成，且操作开销非常低，因为它是托管服务。
B. Migrate the database to Amazon Redshift
错误：Amazon Redshift是数据仓库服务，适用于分析型工作负载，而不是高频事务型工作负载。
迁移到Redshift不仅不适合场景需求，还会增加操作开销。
C. Migrate the database to Amazon RDS
错误：Amazon RDS是关系型数据库服务，虽然可以提供低延迟，但它不具备DynamoDB的无服务器特性。
迁移到RDS会增加管理开销，并且不一定能达到微秒级响应时间。
D. Use Amazon ElastiCache for Redis
错误：虽然ElastiCache for Redis可以提供低延迟缓存，但它需要额外的开发工作来集成到现有的DynamoDB架构中。
相比DAX，操作开销更高，且不如DAX与DynamoDB集成紧密。
3. 我的答案
A. Use DynamoDB Accelerator (DAX)
4. 官方答案
A. Use DynamoDB Accelerator (DAX)
