Question #637
A solutions architect is designing a new service behind Amazon API Gateway. The request patterns for the service will be
unpredictable and can change suddenly from 0 requests to over 500 per second. The total size of the data that needs to be
persisted in a backend database is currently less than 1 GB with unpredictable future growth. Data can be queried using simple
key-value requests.
Which combination ofAWS services would meet these requirements? (Choose two.)
解决方案架构师正在设计一个新的服务，该服务位于 Amazon API Gateway 后面。该服务的请求模式将不可预测，并可能突然从 0
请求增加到每秒超过 500 次请求。需要在后端数据库中持久化的数据总量目前少于 1 GB，并且未来增长不可预测。数据可以使用
简单的键值请求进行查询。

题目分析与解答
1. 考察的知识点
Amazon API Gateway的集成服务选项。
无服务器架构（Serverless）的设计，包括AWS Lambda和Amazon DynamoDB的使用场景。
数据库选择：基于数据大小、查询模式（简单的key-value请求）以及未来的扩展性。
2. 选项分析
A. AWS Fargate
错误。AWS Fargate是用于运行容器化应用的无服务器计算引擎，但题目中没有提到需要运行容器化服务，也没有复杂的应用逻辑
需要容器支持。因此不适合此场景。
B. AWS Lambda
正确。AWS Lambda是无服务器计算服务，能够处理突发的请求模式（从0到500每秒），并且与Amazon API Gateway集成良好，适
合处理动态请求。
C. Amazon DynamoDB
正确。Amazon DynamoDB是一个无服务器的NoSQL数据库，支持简单的key-value查询模式，能够处理小于1GB的数据并具有良好
的扩展性，适合未来的增长需求。
D. Amazon EC2 Auto Scaling
错误。虽然Amazon EC2 Auto Scaling可以处理突发流量，但它需要管理实例和操作系统，复杂度较高，不符合题目中无服务器架构
的要求。
E. MySQL-compatible Amazon Aurora
错误。Amazon Aurora是关系型数据库，适合复杂查询和事务处理，但题目中仅需要简单的key-value查询，且数据量较小（小于
1GB），Aurora显得过于复杂和昂贵。
3. 我选的答案
B. AWS Lambda
C. Amazon DynamoDB
4. 官方答案
B. AWS Lambda
C. Amazon DynamoDB
