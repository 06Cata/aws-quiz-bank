Question #537
A company runs a three-tier web application in the AWS Cloud that operates across three Availability Zones. The application
architecture has an Application Load Balancer, an Amazon EC2 web server that hosts user session states, and a MySQL database
that runs on an EC2 instance. The company expects sudden increases in application traffic. The company wants to be able to scale
to meet future application capacity demands and to ensure high availability across all three Availability Zones.
Which solution will meet these requirements?

题目分析与解答
1. 考察的知识点
高可用性架构设计：确保应用程序在多个可用区中运行以实现容错和高可用性。
数据库迁移与优化：选择适合的数据库服务（如Amazon RDS）以支持高可用性和扩展性。
会话数据存储与缓存：使用适当的缓存服务（如Amazon ElastiCache）来优化性能。
自动扩展：使用Auto Scaling组来动态调整计算资源以应对流量变化。
2. 选项分析
A. Migrate the MySQL database to Amazon RDS for MySQL with a Multi-AZ DB cluster deployment. Use Amazon
ElastiCache for Redis with high availability to store session data and to cache reads. Migrate the web server to an Auto
Scaling group that is in three Availability Zones.
正确：Amazon RDS for MySQL的Multi-AZ部署提供高可用性和故障转移支持。
正确：Amazon ElastiCache for Redis支持高可用性，并且适合存储会话数据和缓存读取。
正确：将Web服务器迁移到跨三个可用区的Auto Scaling组可以确保高可用性和动态扩展能力。
B. Migrate the MySQL database to Amazon RDS for MySQL with a Multi-AZ DB cluster deployment. Use Amazon
ElastiCache for Memcached with high availability to store session data and to cache reads. Migrate the web server to an
Auto Scaling group that is in three Availability Zones.
正确：Amazon RDS for MySQL的Multi-AZ部署提供高可用性和故障转移支持。
错误：Amazon ElastiCache for Memcached不支持本地高可用性（不像Redis支持主从复制和故障转移）。
正确：将Web服务器迁移到跨三个可用区的Auto Scaling组可以确保高可用性和动态扩展能力。
C. Migrate the MySQL database to Amazon DynamoDB Use DynamoDB Accelerator (DAX) to cache reads. Store the
session data in DynamoDB. Migrate the web server to an Auto Scaling group that is in three Availability Zones.
错误：DynamoDB是NoSQL数据库，与MySQL（关系型数据库）不兼容，迁移会导致架构和数据模型的重大变化。
正确：DynamoDB Accelerator (DAX)可以加速读取，但不适合存储会话数据。
正确：将Web服务器迁移到跨三个可用区的Auto Scaling组可以确保高可用性和动态扩展能力。
D. Migrate the MySQL database to Amazon RDS for MySQL in a single Availability Zone. Use Amazon ElastiCache for Redis
with high availability to store session data and to cache reads. Migrate the web server to an Auto Scaling group that is in
three Availability Zones.
错误：单一可用区的Amazon RDS部署无法提供高可用性，违背了题目要求。
正确：Amazon ElastiCache for Redis支持高可用性，并且适合存储会话数据和缓存读取。
正确：将Web服务器迁移到跨三个可用区的Auto Scaling组可以确保高可用性和动态扩展能力。
