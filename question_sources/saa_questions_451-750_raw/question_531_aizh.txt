A company needs to integrate with a third-party data feed. The data feed sends a webhook to notify an external service when new
data is ready for consumption. A developer wrote an AWS Lambda function to retrieve data when the company receives a
webhook callback. The developer must make the Lambda function available for the third party to call.
Which solution will meet these requirements with the MOST operational efficiency?

题目分析与解答
1. 考察的知识点
本题考察的是AWS Lambda的集成方式，尤其是如何通过最简便和高效的方式使Lambda函数能够被第三方调用。
涉及的服务包括Lambda Function URL、Application Load Balancer (ALB)、Amazon SNS、Amazon SQS。
2. 选项分析
A. Create a function URL for the Lambda function. Provide the Lambda function URL to the third party for the webhook.
正确：Lambda Function URL是AWS提供的一种直接访问Lambda函数的方式，无需额外的配置或资源。它支持HTTP调用，适合第
三方集成，且操作效率高。
此选项最符合题目要求，提供了最简单和高效的解决方案。
B. Deploy an Application Load Balancer (ALB) in front of the Lambda function. Provide the ALB URL to the third party for
the webhook.
错误：虽然ALB可以与Lambda集成，但这种方式需要额外的配置和资源（例如，设置目标组、监听器等），增加了复杂性和成本。
题目要求“最具操作效率”的解决方案，使用ALB显然不符合这一要求。
C. Create an Amazon Simple Notification Service (Amazon SNS) topic. Attach the topic to the Lambda function. Provide
the public hostname of the SNS topic to the third party for the webhook.
错误：SNS主题通常用于发布/订阅模式，而不是直接处理HTTP webhook回调。
此外，SNS主题的公共主机名无法直接用于第三方HTTP调用。
D. Create an Amazon Simple Queue Service (Amazon SQS) queue. Attach the queue to the Lambda function. Provide the
public hostname of the SQS queue to the third party for the webhook.
错误：SQS队列用于消息队列处理，而不是直接处理HTTP webhook回调。
类似SNS，SQS的公共主机名无法直接用于第三方HTTP调用。

理由：Lambda Function URL是AWS提供的最直接、最高效的方式来使Lambda函数可被第三方调用，符合题目要求。
