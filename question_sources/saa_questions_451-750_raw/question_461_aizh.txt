Question #461
A company is developing a mobile gaming app in a single AWS Region. The app runs on multiple Amazon EC2 instances in an
Auto Scaling group. The company stores the app data in Amazon DynamoDB. The app communicates by using TCP traffic and
UDP traffic between the users and the servers. The application will be used globally. The company wants to ensure the lowest
possible latency for all users.
Which solution will meet these requirements?

题目分析与解答
1. 考察的知识点
全球用户低延迟访问的解决方案。
AWS Global Accelerator的使用场景。
负载均衡器的选择（ALB vs NLB）。
TCP和UDP流量支持的架构设计。
2. 选项分析
A. 使用AWS Global Accelerator和ALB：
ALB主要用于处理HTTP/HTTPS流量，不支持UDP流量。
题目明确要求支持TCP和UDP流量，因此ALB不符合要求。
此选项错误。
B. 使用AWS Global Accelerator和NLB：
NLB支持TCP和UDP流量，符合题目要求。
AWS Global Accelerator可以优化全球用户的访问路径，降低延迟。
此选项正确。
C. 使用CloudFront和NLB：
CloudFront主要用于内容分发（CDN），不适合处理实时的TCP和UDP流量。
NLB支持TCP和UDP流量，但CloudFront无法直接优化实时通信的延迟。
此选项错误。
D. 使用CloudFront和ALB：
ALB不支持UDP流量，无法满足题目要求。
CloudFront不适合处理实时的TCP和UDP流量。
此选项错误。
