Question #484
A company wants to move from many standalone AWS accounts to a consolidated, multi-account architecture. The company
plans to create many new AWS accounts for different business units. The company needs to authenticate access to these AWS
accounts by using a centralized corporate directory service.
Which combination of actions should a solutions architect recommend to meet these requirements? (Choose two.)
一家公司希望从多个独立的 AWS 账户迁移到一个整合的多账户架构。该公司计划为不同的业务部门创建许多新的 AWS 账户。公司
需要通过一个集中式企业目录服务对这些 AWS 账户进行访问认证。
解决方案架构师应该推荐哪些操作组合来满足这些需求？（选择两个）
A. Create a new organization in AWS Organizations with all features turned on. Create the new AWS accounts in the
organization. Most Voted
在 AWS Organizations 中创建一个新组织，并启用所有功能。在组织中创建新的 AWS 账户。
B. Set up an Amazon Cognito identity pool. Configure AWS IAM Identity Center (AWS Single Sign-On) to accept Amazon
Cognito authentication.
设置一个 Amazon Cognito 身份池。配置 AWS IAM Identity Center (AWS Single Sign-On) 接受 Amazon Cognito 身份认证。
C. Configure a service control policy (SCP) to manage the AWS accounts. Add AWS IAM Identity Center (AWS Single Sign-On)
to AWS Directory Service.
配置服务控制策略 (SCP) 来管理 AWS 账户。将 AWS IAM Identity Center (AWS Single Sign-On) 添加到 AWS Directory Service。
D. Create a new organization in AWS Organizations. Configure the organization's authentication mechanism to use AWS
Directory Service directly.
在 AWS Organizations 中创建一个新组织。直接配置组织的身份认证机制以使用 AWS Directory Service。
E. Set up AWS IAM Identity Center (AWS Single Sign-On) in the organization. Configure IAM Identity Center, and integrate it
with the company's corporate directory service. Most Voted
在组织中设置 AWS IAM Identity Center (AWS Single Sign-On)。配置 IAM Identity Center，并将其与公司的企业目录服务集
成。
Correct Answer: AE
正确答案：AE
题目分析与解答
1. 考察的知识点
AWS Organizations的功能与使用，包括创建多账户架构。
AWS IAM Identity Center (AWS Single Sign-On) 的配置与企业目录服务的集成。
如何实现集中式身份认证和访问管理。
2. 选项分析
A. Create a new organization in AWS Organizations with all features turned on. Create the new AWS accounts in the
organization. 分析：正确。AWS Organizations允许集中管理多个AWS账户，并启用所有功能后可以使用服务控制策略（SCP）等
高级功能。创建新账户在组织中是实现多账户架构的关键步骤。
B. Set up an Amazon Cognito identity pool. Configure AWS IAM Identity Center (AWS Single Sign-On) to accept Amazon
Cognito authentication. 分析：错误。Amazon Cognito主要用于管理应用程序用户身份，而不是用于集中管理AWS账户的访问权
限。AWS IAM Identity Center不直接依赖Amazon Cognito进行认证。
C. Configure a service control policy (SCP) to manage the AWS accounts. Add AWS IAM Identity Center (AWS Single Sign-
On) to AWS Directory Service. 分析：错误。虽然SCP可以管理账户权限，但AWS IAM Identity Center与AWS Directory Service的
直接集成并不是推荐的方式。IAM Identity Center通常与企业目录服务（如Active Directory）集成。
D. Create a new organization in AWS Organizations. Configure the organization's authentication mechanism to use AWS
Directory Service directly. 分析：错误。AWS Organizations本身不支持直接使用AWS Directory Service作为认证机制。认证通常
通过IAM Identity Center与企业目录服务集成来实现。
E. Set up AWS IAM Identity Center (AWS Single Sign-On) in the organization. Configure IAM Identity Center, and
integrate it with the company's corporate directory service. 分析：正确。AWS IAM Identity Center (AWS Single Sign-On) 是集
中管理用户访问多个AWS账户的推荐解决方案。它可以与企业目录服务（如Active Directory）集成，实现统一身份认证。
