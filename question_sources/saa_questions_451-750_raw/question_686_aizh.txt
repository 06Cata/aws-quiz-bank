The company needs to connect the on-premises locations to VPCs in an AWS Region in the AWS Cloud. The number of accounts
and VPCs will increase during the next year. The network architecture must simplify the administration of new connections and
must provide the ability to scale.
Which solution will meet these requirements with the LEAST administrative overhead?

题目分析与解答
1. 考察的知识点
本题考察的是AWS网络架构设计，包括VPC连接、VPN连接、Transit Gateway的使用，以及如何简化管理和扩展网络架构。
2. 选项分析
A. Create a peering connection between the VPCs. Create a VPN connection between the VPCs and the on-premises
locations.
对等连接（VPC Peering）适用于两个VPC之间的直接连接，但随着VPC数量增加，管理多个对等连接会变得复杂。
VPN连接可以连接本地位置和VPC，但无法简化多个VPC之间的连接管理。
此选项无法满足“简化管理”和“可扩展性”的要求，因此错误。
B. Launch an Amazon EC2 instance. On the instance, include VPN software that uses a VPN connection to connect all VPCs
and on-premises locations.
使用EC2实例运行VPN软件是一种手动管理的解决方案，增加了运维复杂性。
此方法不具备自动扩展能力，且需要额外的实例管理工作。
此选项无法满足“最少管理开销”的要求，因此错误。
C. Create a transit gateway. Create VPC attachments for the VPC connections. Create VPN attachments for the on-
premises connections.
Transit Gateway是一种集中式网络连接解决方案，可以简化多个VPC和本地位置之间的连接管理。
通过VPC附件和VPN附件，可以轻松扩展网络架构，满足未来的扩展需求。
此选项符合“简化管理”和“可扩展性”的要求，因此正确。
D. Create an AWS Direct Connect connection between the on-premises locations and a central VPC. Connect the central
VPC to other VPCs by using peering connections.
Direct Connect是一种高性能的连接方式，但仅适用于本地位置和一个中央VPC之间的连接。
使用对等连接连接其他VPC会增加管理复杂性，尤其是随着VPC数量增加。
此选项无法满足“简化管理”的要求，因此错误。
