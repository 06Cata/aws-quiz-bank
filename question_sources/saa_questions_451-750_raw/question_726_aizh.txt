Question #726
A company has deployed a multiplayer game for mobile devices. The game requires live location tracking of players based on
latitude and longitude. The data store for the game must support rapid updates and retrieval of locations.
The game uses an Amazon RDS for PostgreSQL DB instance with read replicas to store the location data. During peak usage
periods, the database is unable to maintain the performance that is needed for reading and writing updates. The game's user base
is increasing rapidly.
What should a solutions architect do to improve the performance of the data tier?
一家公司已部署了一款针对移动设备的多人游戏。该游戏需要基于经纬度对玩家进行实时位置跟踪。游戏的数据存储必须支持快速
更新和位置检索。
该游戏使用一个带有只读副本的 Amazon RDS for PostgreSQL 数据库实例来存储位置数据。在高峰使用期间，数据库无法维持所需
的读写更新性能。游戏的用户群正在迅速增长。
解决方案架构师应该怎么做以改善数据层的性能？
A. Take a snapshot of the existing DB instance. Restore the snapshot with Multi-AZ enabled.
对现有数据库实例进行快照。使用启用 Multi-AZ 的快照进行恢复。
B. Migrate from Amazon RDS to Amazon OpenSearch Service with OpenSearch Dashboards.
从 Amazon RDS 迁移到带有 OpenSearch Dashboards 的 Amazon OpenSearch Service。
C. Deploy Amazon DynamoDB Accelerator (DAX) in front of the existing DB instance. Modify the game to use DAX.
在现有数据库实例前部署 Amazon DynamoDB Accelerator (DAX)。修改游戏以使用 DAX。
D. Deploy an Amazon ElastiCache for Redis cluster in front of the existing DB instance. Modify the game to use Redis.
Most Voted
在现有数据库实例前部署一个 Amazon ElastiCache for Redis 集群。修改游戏以使用 Redis。
Correct Answer: D

题目分析与解答
1. 考察的知识点
本题考察的是如何优化数据库性能，特别是在高并发读写场景下的解决方案。
涉及的AWS服务包括Amazon RDS、Amazon ElastiCache、Amazon DynamoDB Accelerator (DAX)、以及Amazon OpenSearch
Service。
2. 选项分析
A. Take a snapshot of the existing DB instance. Restore the snapshot with Multi-AZ enabled.
Multi-AZ部署主要用于提高数据库的可用性和容错能力，而不是性能优化。
启用Multi-AZ不会解决读写性能瓶颈问题，因此此选项错误。
B. Migrate from Amazon RDS to Amazon OpenSearch Service with OpenSearch Dashboards.
Amazon OpenSearch Service主要用于搜索和分析场景，而不是实时读写高频更新的数据存储。
此选项不适合解决游戏实时位置更新的性能问题，因此错误。
C. Deploy Amazon DynamoDB Accelerator (DAX) in front of the existing DB instance. Modify the game to use DAX.
DynamoDB Accelerator (DAX)是用于加速Amazon DynamoDB查询的缓存服务。
本题中使用的是Amazon RDS，而不是DynamoDB，因此DAX无法直接应用于现有架构。
此选项错误。
D. Deploy an Amazon ElastiCache for Redis cluster in front of the existing DB instance. Modify the game to use Redis.
Amazon ElastiCache for Redis是一种高性能缓存解决方案，适合处理高频读写场景。
通过在数据库前部署Redis，可以显著减少数据库的读写压力，提高性能。
此选项正确。
3. 我的答案
我的选择是：D
4. 官方答案
官方答案是：D
5. 与官方答案比较
我的答案与官方答案一致。
原因分析：Amazon ElastiCache for Redis能够有效缓解数据库的读写压力，特别是在高并发场景下，适合解决本题中的性能问题。
D (74%)
