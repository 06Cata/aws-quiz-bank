wants to offload reporting from its primary database system.

题目分析与解答
1. 考察的知识点
Amazon RDS的部署选项，包括Single-AZ和Multi-AZ。
Amazon Aurora的特性，包括高可用性和读写分离。
如何使用AWS服务实现数据库迁移和优化性能。
读副本的用途及其在报告功能中的应用。
2. 选项分析
A. Use AWS Database Migration Service (AWS DMS) to create an Amazon RDS DB instance in multiple AWS Regions. Point
the reporting functions toward a separate DB instance from the primary DB instance.
正确性分析：AWS DMS可以用于迁移数据库，但题目没有提到需要跨区域部署，也没有明确要求多个区域的数据库实例。此选项增
加了复杂性，且没有提到高可用性和性能优化的具体实现。
结论：错误。
B. Use Amazon RDS in a Single-AZ deployment to create an Oracle database. Create a read replica in the same zone as the
primary DB instance. Direct the reporting functions to the read replica.
正确性分析：Single-AZ部署不提供高可用性，且读副本通常用于跨区域或跨可用区的读写分离，而不是在同一可用区内部署。
结论：错误。
C. Use Amazon RDS deployed in a Multi-AZ cluster deployment to create an Oracle database. Direct the reporting
functions to use the reader instance in the cluster deployment.
正确性分析：Multi-AZ部署提供高可用性，但Oracle数据库的Multi-AZ部署主要用于故障转移，而不是读写分离。此选项没有提到
性能优化的具体实现。
结论：错误。
D. Use Amazon RDS deployed in a Multi-AZ instance deployment to create an Amazon Aurora database. Direct the
reporting functions to the reader instances.
正确性分析：Amazon Aurora支持高可用性和读写分离，读实例可以用于处理报告功能，从而减轻主实例的负载。此选项满足题目
要求的高可用性和性能优化。
结论：正确。
