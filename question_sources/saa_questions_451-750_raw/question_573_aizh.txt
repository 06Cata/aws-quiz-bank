Question #573
A company wants to use an event-driven programming model with AWS Lambda. The company wants to reduce startup latency
for Lambda functions that run on Java 11. The company does not have strict latency requirements for the applications. The
company wants to reduce cold starts and outlier latencies when a function scales up.
Which solution will meet these requirements MOST cost-effectively?
一家公司希望使用事件驱动的编程模型与AWS Lambda。该公司希望减少运行在Java 11上的Lambda函数的启动延迟。该公司对应
用程序没有严格的延迟要求。该公司希望减少冷启动和函数扩展时的异常延迟。
哪种解决方案将最具成本效益地满足这些要求？
A. Configure Lambda provisioned concurrency.
配置Lambda预置并发。
B. Increase the timeout of the Lambda functions.
增加Lambda函数的超时时间。
C. Increase the memory of the Lambda functions.
增加Lambda函数的内存。
D. Configure Lambda SnapStart. Most Voted
配置Lambda SnapStart。
题目分析与解答
1. 考察的知识点
本题考察的是AWS Lambda的性能优化，特别是如何减少冷启动（Cold Start）延迟。
Correct Answer: D
正确答案：D
Community vote distribution
社区投票分布
D (100%)
涉及的技术包括Lambda SnapStart、预置并发（Provisioned Concurrency）、内存配置和超时设置。
2. 选项分析
A. 配置Lambda预置并发
分析：预置并发可以减少冷启动延迟，因为它会预先启动一定数量的Lambda实例。然而，这种方法会增加成本，因为需要为预置
的并发实例付费。题目要求“最具成本效益”，因此此选项不符合要求。
B. 增加Lambda函数的超时时间
分析：增加超时时间不会直接减少冷启动延迟。超时时间仅影响函数运行的最大时间限制，与冷启动无关。因此此选项错误。
C. 增加Lambda函数的内存
分析：增加内存可以提高函数的执行速度，但对冷启动的影响有限。冷启动主要与初始化时间相关，而不是运行时性能。因此此选
项不符合题目要求。
D. 配置Lambda SnapStart
分析：Lambda SnapStart是专门设计用于减少冷启动延迟的功能，特别适用于Java 11等语言。它通过在函数初始化后创建快照并复
用快照来减少启动时间。此选项符合题目要求，并且是最具成本效益的解决方案。
3. 我的答案
D. 配置Lambda SnapStart
4. 官方答案
D. 配置Lambda SnapStart
