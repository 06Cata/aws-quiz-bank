Question #721
A company wants to rearchitect a large-scale web application to a serverless microservices architecture. The application uses
Amazon EC2 instances and is written in Python.
The company selected one component of the web application to test as a microservice. The component supports hundreds of
requests each second. The company wants to create and test the microservice on an AWS solution that supports Python. The
solution must also scale automatically and require minimal infrastructure and minimal operational support.
Which solution will meet these requirements?
一家公司希望将一个大型的网页应用程序重新架构为无服务器的微服务架构。该应用程序使用Amazon EC2实例，并且是用Python
编写的。
该公司选择了网页应用程序中的一个组件作为微服务进行测试。该组件每秒支持数百个请求。公司希望在支持Python的AWS解决方

哪种解决方案能够满足这些需求？

题目分析与解答
1. 考察的知识点
本题考察的是AWS服务的选择与架构设计，特别是如何构建一个支持高并发、自动扩展、最小化基础设施和运维支持的无服务器架
构。
涉及的服务包括AWS Lambda、Amazon EC2、AWS Elastic Beanstalk和Amazon EKS。
2. 选项分析
A. 使用运行最新Amazon Linux操作系统的EC2实例的Spot Fleet，并启用自动扩展。
错误：虽然Spot Fleet可以提供成本优化的计算资源，并支持自动扩展，但它仍然需要管理EC2实例的操作系统、应用部署和扩展策
略，无法满足“最小化基础设施和运维支持”的要求。
B. 使用配置了高可用性的AWS Elastic Beanstalk网页服务器环境。
错误：Elastic Beanstalk虽然简化了应用部署和管理，但它仍然基于EC2实例运行，涉及一定的基础设施管理。相比AWS Lambda，
它的无服务器特性不够突出，无法完全满足“最小化基础设施和运维支持”的要求。
C. 使用Amazon Elastic Kubernetes Service (Amazon EKS)。启动自管理EC2实例的Auto Scaling组。
错误：Amazon EKS是一个强大的容器编排服务，但它需要管理Kubernetes集群和EC2实例，运维复杂度较高，不符合“最小化基础
设施和运维支持”的要求。
D. 使用运行自定义开发代码的AWS Lambda函数。
正确：AWS Lambda是一个完全无服务器的计算服务，支持Python代码运行，能够自动扩展以处理高并发请求。它无需管理底层基
础设施，完全符合“最小化基础设施和运维支持”的要求。
3. 我选的答案
D. 使用运行自定义开发代码的AWS Lambda函数。
4. 官方答案
D. 使用运行自定义开发代码的AWS Lambda函数。
5. 与官方答案的比较
我的答案与官方答案一致。
原因：AWS Lambda完全符合题目要求，能够支持Python代码运行，自动扩展以处理高并发请求，并且无需管理底层基础设施，最
小化了运维支持。
