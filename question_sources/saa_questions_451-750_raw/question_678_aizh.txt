to fully control the ability of users to create, rotate, and disable encryption keys with minimal effort for any data that must be
encrypted.
Which solution will meet these requirements?

题目分析与解答
1. 考察的知识点
Amazon S3 数据加密选项
AWS Key Management Service (KMS) 的使用
客户管理密钥 (Customer Managed Key) 的控制能力
服务器端加密 (Server-Side Encryption) 的实现方式
2. 选项分析
A. 使用默认的服务器端加密和 Amazon S3 管理的加密密钥 (SSE-S3)
分析：此选项使用 Amazon S3 管理的加密密钥 (SSE-S3)，但公司要求完全控制密钥的创建、轮换和禁用能力。SSE-S3 不允许用户
完全控制密钥，因此此选项不符合要求。
结论：错误。
B. 使用 AWS Key Management Service (AWS KMS) 创建一个客户管理密钥，并使用新密钥通过服务器端加密和 AWS KMS 密钥
(SSE-KMS) 加密 S3 对象
分析：此选项使用 AWS KMS 创建客户管理密钥 (Customer Managed Key)，并通过 SSE-KMS 加密 S3 对象。客户管理密钥允许用户
完全控制密钥的创建、轮换和禁用能力，符合公司的要求。
结论：正确。
C. 使用 AWS Key Management Service (AWS KMS) 创建一个 AWS 管理密钥，并使用新密钥通过服务器端加密和 AWS KMS 密
钥 (SSE-KMS) 加密 S3 对象
分析：此选项使用 AWS 管理密钥 (AWS Managed Key)，而不是客户管理密钥。AWS 管理密钥由 AWS 管理，用户无法完全控制密
钥的创建、轮换和禁用能力，因此不符合公司的要求。
结论：错误。
D. 将 S3 对象下载到 Amazon EC2 实例，使用客户管理密钥加密对象，然后将加密后的对象重新上传到 Amazon S3
分析：此选项涉及手动加密过程，需要下载和重新上传数据，增加了复杂性和成本。公司要求的是最小化努力的解决方案，因此此
选项不符合要求。
结论：错误。
