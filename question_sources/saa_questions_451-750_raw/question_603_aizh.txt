Question #603
A company recently migrated to the AWS Cloud. The company wants a serverless solution for large-scale parallel on-demand
processing of a semistructured dataset. The data consists of logs, media files, sales transactions, and IoT sensor data that is stored
in Amazon S3. The company wants the solution to process thousands of items in the dataset in parallel.
Which solution will meet these requirements with the MOST operational efficiency?
一家公司最近迁移到了AWS云。该公司希望为半结构化数据集的大规模并行按需处理提供一个无服务器解决方案。数据包括日志、
媒体文件、销售交易和存储在Amazon S3中的IoT传感器数据。该公司希望解决方案能够并行处理数据集中数千项数据。

本题考察的是AWS服务的选择与架构设计，特别是针对大规模并行处理的最佳实践。
涉及的服务包括AWS Step Functions、AWS Glue、AWS Lambda，以及如何高效处理存储在Amazon S3中的数据。
2. 选项分析
A. Use the AWS Step Functions Map state in Inline mode to process the data in parallel.
Step Functions的Map状态的Inline模式适用于较小规模的并行处理，因为它的并行度受限于Step Functions的执行限制。
对于大规模数据集（如题目中提到的“处理数千项数据”），Inline模式可能无法满足需求。
因此此选项不符合题目要求。
B. Use the AWS Step Functions Map state in Distributed mode to process the data in parallel.
Step Functions的Map状态的Distributed模式专为大规模并行处理设计，能够高效地处理数千项数据。
Distributed模式可以利用AWS Lambda函数并行处理数据，同时具有高扩展性和低运维负担。
此选项符合题目要求，是最佳选择。
C. Use AWS Glue to process the data in parallel.
AWS Glue是一种ETL服务，适用于数据转换和分析，但它的主要用途是处理结构化或半结构化数据，而不是直接处理大规模并行任
务。
虽然AWS Glue可以处理数据，但它的设计目标与题目要求的“高效并行处理”不完全匹配。
此选项不是最佳选择。
D. Use several AWS Lambda functions to process the data in parallel.
AWS Lambda可以实现并行处理，但需要额外的逻辑来管理任务分配和协调。
相比Step Functions的Distributed模式，直接使用Lambda函数会增加开发和运维复杂性。
此选项虽然可行，但不是最优解。
3. 我选的答案
B. Use the AWS Step Functions Map state in Distributed mode to process the data in parallel.
4. 官方答案
B. Use the AWS Step Functions Map state in Distributed mode to process the data in parallel.

我的答案与官方答案一致。

具有较高的操作效率。
