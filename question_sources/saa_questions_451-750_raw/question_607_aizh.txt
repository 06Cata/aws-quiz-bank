Question #607
A company has migrated a two-tier application from its on-premises data center to the AWS Cloud. The data tier is a Multi-AZ
deployment of Amazon RDS for Oracle with 12 TB of General Purpose SSD Amazon Elastic Block Store (Amazon EBS) storage. The
application is designed to process and store documents in the database as binary large objects (blobs) with an average document
size of 6 MB.
The database size has grown over time, reducing the performance and increasing the cost of storage. The company must improve
the database performance and needs a solution that is highly available and resilient.
Which solution will meet these requirements MOST cost-effectively?
一家公司已将一个两层应用程序从其本地数据中心迁移到 AWS 云。数据层是一个 Amazon RDS for Oracle 的 Multi-AZ 部署，使用
了 12 TB 的通用型 SSD Amazon Elastic Block Store (Amazon EBS) 存储。该应用程序被设计为在数据库中以二进制大对象（blobs）
的形式处理和存储文档，平均文档大小为 6 MB。
随着时间的推移，数据库的大小不断增长，导致性能下降和存储成本增加。公司必须提高数据库性能，并需要一个高度可用且具有
弹性的解决方案。
哪种解决方案能够最具成本效益地满足这些需求？
A. Reduce the RDS DB instance size. Increase the storage capacity to 24 TiB. Change the storage type to Magnetic.

题目分析与解答
1. 考察的知识点
Amazon RDS性能优化与存储类型选择
Amazon S3的使用场景与成本优化
数据库设计与架构优化
高可用性与弹性架构设计
2. 选项分析
A. 减少 RDS 数据库实例的大小。将存储容量增加到 24 TiB。将存储类型更改为磁性存储。
错误原因：磁性存储（Magnetic）是较旧的存储类型，性能较低，不适合处理高性能需求的数据库工作负载。
减少实例大小会进一步降低性能，与题目要求的性能提升相悖。
B. 增加 RDS 数据库实例的大小。将存储容量增加到 24 TiB。将存储类型更改为预配置 IOPS。
错误原因：虽然预配置 IOPS可以提升性能，但成本较高，不符合题目要求的“最具成本效益”的解决方案。
增加实例大小和存储容量会进一步增加成本。
C. 创建一个 Amazon S3 存储桶。更新应用程序以将文档存储在 S3 存储桶中。在现有数据库中存储对象元数据。
正确原因：将文档存储在Amazon S3中可以显著降低存储成本，同时减轻数据库的负担，从而提升性能。
Amazon S3具有高可用性和弹性，适合存储大规模的二进制对象（如文档）。
将元数据保留在数据库中可以保持应用程序的功能完整性。
D. 创建一个 Amazon DynamoDB 表。更新应用程序以使用 DynamoDB。使用 AWS Database Migration Service (AWS DMS)
将数据从 Oracle 数据库迁移到 DynamoDB。
错误原因：DynamoDB是一个NoSQL数据库，适合处理键值对数据，而不是存储大规模的二进制对象（如文档）。
迁移到DynamoDB需要对应用程序进行重大改动，成本和复杂性较高。
