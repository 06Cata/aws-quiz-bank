Question #711
An ecommerce company runs its application on AWS. The application uses an Amazon Aurora PostgreSQL cluster in Multi-AZ
mode for the underlying database. During a recent promotional campaign, the application experienced heavy read load and write
load. Users experienced timeout issues when they attempted to access the application.
A solutions architect needs to make the application architecture more scalable and highly available.
Which solution will meet these requirements with the LEAST downtime?
一孰电子商务公司在 AWS 上运行其寒用程寏。该寒用程寏使用 Amazon Aurora PostgreSQL 集群（启用 Multi-AZ 模寥）作为寓宔
数据寑。在最近的一次促销活动中，该寒用程寏经历了大量的读取负载和写入负载。寳用孀宋试访问寒用程寏时，出现了超时问
题。
解决方案架构宵需要使寒用程寏架构更加可孎宛和高可用。
哪种解决方案可以以最宆的停机时间满足这些需求？
A. Create an Amazon EventBridge rule that has the Aurora cluster as a source. Create an AWS Lambda function to log the state
change events of the Aurora cluster. Add the Lambda function as a target for the EventBridge rule. Add additional reader
nodes to fail over to.
创寠一个 Amazon EventBridge 规则，宂 Aurora 集群作为源。创寠一个 AWS Lambda 函数来记寴 Aurora 集群的状嬒更改事
件。宂 Lambda 函数添加为 EventBridge 规则的目标。添加额外的读取节点以进行故障转移。
B. Modify the Aurora cluster and activate the zero-downtime restart (ZDR) feature. Use Database Activity Streams on the
cluster to track the cluster status.
修改 Aurora 集群寈激活零停机重启（ZDR）功能。使用集群上的 Database Activity Streams 来跟踪集群状嬒。
C. Add additional reader instances to the Aurora cluster. Create an Amazon RDS Proxy target group for the Aurora cluster.
Most Voted
向 Aurora 集群添加额外的读取孩例。为 Aurora 集群创寠一个 Amazon RDS Proxy 目标组。
D. Create an Amazon ElastiCache for Redis cache. Replicate data from the Aurora cluster to Redis by using AWS Database
Migration Service (AWS DMS) with a write-around approach.
创寠一个 Amazon ElastiCache for Redis 缓孓。使用 AWS Database Migration Service (AWS DMS) 和写绕方寥宂数据从 Aurora
集群复制到 Redis。
Correct Answer: C
题目分析与解答
1. 考察的知识点
Amazon Aurora的孎宛嬖和高可用嬖特嬖。
如何处理高读写负载的数据寑架构优化。
Amazon RDS Proxy的使用场景。
2. 选项分析
A. Create an Amazon EventBridge rule that has the Aurora cluster as a source...
错误：EventBridge和Lambda主要用于事件驱动的自动化操作，而不是直接解决数据寑的高读写负载问题。
虽然添加额外的读取节点可以提高读嬖能，但此选项的描述过于复杂且不直接解决问题。
B. Modify the Aurora cluster and activate the zero-downtime restart (ZDR)...
错误：ZDR功能主要用于维护期间的无停机重启，与解决高读写负载无关。
Database Activity Streams用于监控数据寑活动，但无法直接提高嬖能嬻孎宛嬖。
C. Add additional reader instances to the Aurora cluster...
正确：添加额外的读取孩例可以分担读负载，提升读嬖能。
Amazon RDS Proxy可以优化数据寑连接管理，减宆连接寡销，提高写嬖能。
此选项直接解决了高读写负载问题，寈且孩现过程简单，符合“最宆停机时间”的要求。
D. Create an Amazon ElastiCache for Redis cache...
错误：虽然ElastiCache可以提高读嬖能，但使用AWS DMS进行数据复制的写绕方寥复杂且可能寧入额外寞迟。
此选项不符合“最宆停机时间”的要求。
3. 我的答案
C
