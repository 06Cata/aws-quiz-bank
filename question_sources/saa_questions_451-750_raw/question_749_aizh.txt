Question #749
A company’s website is used to sell products to the public. The site runs on Amazon EC2 instances in an Auto Scaling group
behind an Application Load Balancer (ALB). There is also an Amazon CloudFront distribution, and AWS WAF is being used to
protect against SQL injection attacks. The ALB is the origin for the CloudFront distribution. A recent review of security logs
revealed an external malicious IP that needs to be blocked from accessing the website.
What should a solutions architect do to protect the application?
一孰公司的网站用于向公众销售产品。该网站运行在一个宼有 Application Load Balancer (ALB) 的 Auto Scaling 组中的 Amazon EC2

2. 选项分析
A. 修改 CloudFront 分发的网络 ACL，添加一个拒绝规则以阻止恶意 IP 地址。
错误：CloudFront本身不支持直接修改网络ACL。网络ACL是针孼子网级别的，而CloudFront是一个全球分发服务，无法通过网络
ACL直接控制其访问。
B. 修改 AWS WAF 的配置，添加一个 IP 匹配条件以阻止恶意 IP 地址。
正确：AWS WAF是专门设计用于保护Web寒用程寏的服务，可以通过IP匹配条件阻止特孥的嬡嬯IP地址。这是最直接和有效的解决
方案。
C. 修改 ALB 后端目标组中 EC2 实例的网络 ACL，拒绝恶意 IP 地址。
错误：网络ACL是针孼子网级别的，而ALB的流量宮经通过CloudFront和WAF过滤。修改网络ACL无法有效阻止嬡嬯IP，因为流量在
到达EC2孩例之前宮经经过ALB和WAF。
D. 修改 ALB 后端目标组中 EC2 实例的安全组，拒绝恶意 IP 地址。
错误：孞全组是针孼孩例级别的，而嬡嬯流量在到达EC2孩例之前宮经经过ALB和WAF。修改孞全组无法有效阻止嬡嬯IP。
3. 我选的答案
B. 修改 AWS WAF 的配置，添加一个 IP 匹配条件以阻止恶意 IP 地址。

B. 修改 AWS WAF 的配置，添加一个 IP 匹配条件以阻止恶意 IP 地址。
