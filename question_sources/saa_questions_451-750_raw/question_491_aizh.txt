题目分析与解答
1. 考察的知识点
Amazon SQS队列类型（标准队列 vs FIFO队列）的特性。
AWS Lambda事件源映射的工作原理。
加密机制（SSE-KMS vs SSE-SQS）的使用和权限配置。
如何设计异步应用以确保消息至少处理一次。
2. 选项分析
A. Use AWS Lambda event source mapping. Set Amazon Simple Queue Service (Amazon SQS) standard queues as the
event source. Use AWS Key Management Service (SSE-KMS) for encryption. Add the kms:Decrypt permission for the
Lambda execution role.
正确：SQS标准队列支持“至少一次”消息处理，符合题目要求。
正确：使用SSE-KMS加密可以确保数据安全，且需要为Lambda执行角色添加kms:Decrypt权限。
成本较低：标准队列比FIFO队列成本更低，适合成本敏感的场景。
B. Use AWS Lambda event source mapping. Use Amazon Simple Queue Service (Amazon SQS) FIFO queues as the event
source. Use SQS managed encryption keys (SSE-SQS) for encryption. Add the encryption key invocation permission for
the Lambda function.
错误：FIFO队列支持“恰好一次”消息处理，但题目要求“至少一次”，因此不符合要求。
正确：SSE-SQS是SQS托管的加密机制，适合加密需求。
成本较高：FIFO队列的成本高于标准队列，不是最具成本效益的选择。
C. Use the AWS Lambda event source mapping. Set Amazon Simple Queue Service (Amazon SQS) FIFO queues as the
event source. Use AWS KMS keys (SSE-KMS). Add the kms:Decrypt permission for the Lambda execution role.
错误：FIFO队列支持“恰好一次”消息处理，但题目要求“至少一次”，因此不符合要求。
正确：使用SSE-KMS加密可以确保数据安全，且需要为Lambda执行角色添加kms:Decrypt权限。
成本较高：FIFO队列的成本高于标准队列，不是最具成本效益的选择。
D. Use the AWS Lambda event source mapping. Set Amazon Simple Queue Service (Amazon SQS) standard queues as the
event source. Use AWS KMS keys (SSE-KMS) for encryption. Add the encryption key invocation permission for the
Lambda function.
正确：SQS标准队列支持“至少一次”消息处理，符合题目要求。
错误：加密权限描述不准确，应该是为Lambda执行角色添加kms:Decrypt权限，而不是“加密密钥调用权限”。
