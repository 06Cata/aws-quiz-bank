Question #569
An Amazon EventBridge rule targets a third-party API. The third-party API has not received any incoming traffic. A solutions
architect needs to determine whether the rule conditions are being met and if the rule's target is being invoked.
Which solution will meet these requirements?
一个 Amazon EventBridge 规则的目标是一个第三方 API。该第三方 API 尚未接收到任何传入流量。解决方案架构师需要确定规则条
件是否被满足，以及规则的目标是否被调用。
哪种解决方案可以满足这些要求？
A. Check for metrics in Amazon CloudWatch in the namespace for AWS/Events. Most Voted
在 Amazon CloudWatch 中检查 AWS/Events 命名空间中的指标。
B. Review events in the Amazon Simple Queue Service (Amazon SQS) dead-letter queue.
查看 Amazon Simple Queue Service (Amazon SQS) 死信队列中的事件。
C. Check for the events in Amazon CloudWatch Logs.
在 Amazon CloudWatch Logs 中检查事件。
D. Check the trails in AWS CloudTrail for the EventBridge events.
在 AWS CloudTrail 中检查 EventBridge 事件的踪迹。

Amazon EventBridge规则的监控与调试。

2. 选项分析
A. Check for metrics in Amazon CloudWatch in the namespace for AWS/Events.
正确：Amazon EventBridge会将相关的指标发布到Amazon CloudWatch的AWS/Events命名空间中。这些指标可以帮助确认规则是
否触发以及目标是否被调用。
B. Review events in the Amazon Simple Queue Service (Amazon SQS) dead-letter queue.
错误：死信队列用于存储无法成功处理的消息，但题目中并未提到使用SQS作为目标，也没有提到消息处理失败的情况。
C. Check for the events in Amazon CloudWatch Logs.
错误：虽然CloudWatch Logs可以记录日志，但题目中并未提到将EventBridge事件发送到CloudWatch Logs进行记录。
D. Check the trails in AWS CloudTrail for the EventBridge events.
错误：AWS CloudTrail记录API调用和事件历史，但它主要用于审计和合规性，而不是实时监控EventBridge规则是否触发。
3. 我的答案
A. Check for metrics in Amazon CloudWatch in the namespace for AWS/Events.
4. 官方答案
A. Check for metrics in Amazon CloudWatch in the namespace for AWS/Events.
5. 答案比较与分析
我的答案与官方答案一致。
原因：Amazon CloudWatch的AWS/Events命名空间提供了EventBridge规则的相关指标，是验证规则是否触发以及目标是否被调用
的最佳工具。
