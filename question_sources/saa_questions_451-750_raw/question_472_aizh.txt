题目分析与解答
1. 考察的知识点
本题考察的是如何优化 Amazon DynamoDB 的读取性能，特别是针对低延迟读取的解决方案。
涉及的服务包括 Amazon DynamoDB、DynamoDB Accelerator (DAX)、DynamoDB 读副本、ElastiCache for Redis。
2. 选项分析
A. Configure Amazon DynamoDB Accelerator (DAX) for the new messages table. Update the code to use the DAX
endpoint.
正确：Amazon DynamoDB Accelerator (DAX) 是一种内存中缓存解决方案，专门用于减少 DynamoDB 的读取延迟。通过使用
DAX，应用程序可以显著降低读取延迟，而无需对 DynamoDB 表进行结构性更改。
需要更新代码以使用 DAX 端点，但这属于最小的应用程序更改范围。
B. Add DynamoDB read replicas to handle the increased read load. Update the application to point to the read endpoint
for the read replicas.
错误：DynamoDB 读副本主要用于扩展读取吞吐量，而不是优化低延迟读取。虽然可以增加读取容量，但它无法显著减少读取延
迟。
此外，使用读副本需要更新应用程序以指向新的读取端点，这可能涉及更多的代码更改。
C. Double the number of read capacity units for the new messages table in DynamoDB. Continue to use the existing
DynamoDB endpoint.
错误：增加读取容量单位 (RCU) 可以提高吞吐量，但不会减少读取延迟。用户的需求是低延迟读取，而不是处理更高的读取负载。
此选项无法满足题目要求的“低延迟读取”。
D. Add an Amazon ElastiCache for Redis cache to the application stack. Update the application to point to the Redis cache
endpoint instead of DynamoDB.
错误：虽然 ElastiCache for Redis 是一种强大的缓存解决方案，但它需要显著的应用程序更改，包括管理缓存一致性和更新逻辑。
题目要求“最小的应用程序更改”，而使用 Redis 需要重新设计应用程序的读取逻辑。
3. 我的答案
A
