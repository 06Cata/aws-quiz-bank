Question #700
A gaming company wants to launch a new internet-facing application in multiple AWS Regions. The application will use the TCP
and UDP protocols for communication. The company needs to provide high availability and minimum latency for global users.
Which combination of actions should a solutions architect take to meet these requirements? (Choose two.)
一家游戏公司希望在多个 AWS 区域推出一个新的面向互联网的应用程序。该应用程序将使用 TCP 和 UDP 协议进行通信。公司需要
为全球用户提供高可用性和最低延迟。
解决方案架构师应采取哪些组合操作来满足这些需求？（选择两个。）
A. Create internal Network Load Balancers in front of the application in each Region. Most Voted
在每个区域的应用程序前创建内部 Network Load Balancers。
B. Create external Application Load Balancers in front of the application in each Region.
在每个区域的应用程序前创建外部 Application Load Balancers。
C. Create an AWS Global Accelerator accelerator to route traffic to the load balancers in each Region. Most Voted
创建一个 AWS Global Accelerator 加速器，将流量路由到每个区域的负载均衡器。
D. Configure Amazon Route 53 to use a geolocation routing policy to distribute the traffic.
配置 Amazon Route 53 使用地理位置路由策略分配流量。
E. Configure Amazon CloudFront to handle the traffic and route requests to the application in each Region
配置 Amazon CloudFront 处理流量并将请求路由到每个区域的应用程序。

2. 选项分析
A. Create internal Network Load Balancers in front of the application in each Region.
错误：内部 Network Load Balancer 是为内部流量设计的，无法处理互联网流量。
题目要求“互联网面对的应用”，因此此选项不符合要求。
B. Create external Application Load Balancers in front of the application in each Region.
错误：虽然 Application Load Balancer 可以处理互联网流量，但它主要用于 HTTP/HTTPS 协议，而题目明确提到需要支持 TCP 和
UDP 协议。
此选项不符合题目要求。
C. Create an AWS Global Accelerator accelerator to route traffic to the load balancers in each Region.
正确：AWS Global Accelerator 提供全球流量优化，支持 TCP 和 UDP 协议，并能将流量路由到各区域的负载均衡器，满足高可用性
和低延迟需求。
D. Configure Amazon Route 53 to use a geolocation routing policy to distribute the traffic.
错误：虽然 Route 53 的地理位置路由可以分配流量，但它无法优化 TCP 和 UDP 协议的性能，也无法提供类似 Global Accelerator
的低延迟和高可用性功能。
E. Configure Amazon CloudFront to handle the traffic and route requests to the application in each Region.
错误：CloudFront 是内容分发网络（CDN），主要用于缓存和分发静态内容。它不适合直接处理 TCP 和 UDP 流量。
