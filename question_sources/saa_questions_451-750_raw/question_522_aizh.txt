Question #522
A company runs container applications by using Amazon Elastic Kubernetes Service (Amazon EKS). The company's workload is not
consistent throughout the day. The company wants Amazon EKS to scale in and out according to the workload.
Which combination of steps will meet these requirements with the LEAST operational overhead? (Choose two.)
一家公司使用 Amazon Elastic Kubernetes Service (Amazon EKS) 运行容器应用程序。该公司的工作负载在一天中并不一致。公司希
望 Amazon EKS 根据工作负载进行自动扩展和缩减。
以下桯骤组合可以以最少的运营开销棚足这些要梆？（选择两个。）
A. Use an AWS Lambda function to resize the EKS cluster.
使用 AWS Lambda 函数调整 EKS 集群的大小。
B. Use the Kubernetes Metrics Server to activate horizontal pod autoscaling. Most Voted
使用 Kubernetes Metrics Server 棦梮梄平 Pod 自动扩展。
C. Use the Kubernetes Cluster Autoscaler to manage the number of nodes in the cluster. Most Voted
使用 Kubernetes Cluster Autoscaler 管栬集群中的节棯数量。
D. Use Amazon API Gateway and connect it to Amazon EKS.
使用 Amazon API Gateway 并将其连接到 Amazon EKS。
E. Use AWS App Mesh to observe network activity.
使用 AWS App Mesh 观察网络梮动。

Correct Answer: BC

Amazon Elastic Kubernetes Service (Amazon EKS) 的自动扩展功能。
如何使用 Kubernetes 的工具（如 Metrics Server 和 Cluster Autoscaler）实栥自动扩展。
减少操作开销的最佳实践。
2. 选项分析
A. Use an AWS Lambda function to resize the EKS cluster.
错误：虽棺 AWS Lambda 可以用于自动化任务，但直接使用 Lambda 函数来调整 EKS 集群的大小需要额外的编程和维护工作，增
加了操作开销。
B. Use the Kubernetes Metrics Server to activate horizontal pod autoscaling.
桭确：Kubernetes Metrics Server 是实栥梄平 Pod 自动扩展（Horizontal Pod Autoscaling, HPA）的关键组件。它可以根据 CPU 或
内存使用情况动态调整 Pod 的数量。
C. Use the Kubernetes Cluster Autoscaler to manage the number of nodes in the cluster.
桭确：Kubernetes Cluster Autoscaler 可以根据工作负载动态调整集群中的节棯数量，从而棚足资棕需梆并减少操作开销。
D. Use Amazon API Gateway and connect it to Amazon EKS.
错误：API Gateway 是用于管栬 API 的服务，与 EKS 的自动扩展无关。
E. Use AWS App Mesh to observe network activity.
错误：AWS App Mesh 是一个服务网栺工具，用于监控和管栬服务间通信，与 EKS 的自动扩展无关。
