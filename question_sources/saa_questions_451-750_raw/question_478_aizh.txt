Modifications or deletions of the files by anyone before a designated future date are prohibited.
Which solution will meet these requirements in the MOST secure way?

题目分析与解答
1. 考察的知识点
Amazon S3静态网站托管
S3 Object Lock功能及其保留期设置
S3 Versioning功能
IAM权限管理
S3存储桶策略
2. 选项分析
A. Upload all files to an Amazon S3 bucket that is configured for static website hosting. Grant read-only IAM permissions
to any AWS principals that access the S3 bucket until the designated date.
分析：
虽然配置了静态网站托管和只读IAM权限，但没有使用S3 Object Lock功能，因此无法防止文件被修改或删除。
此选项无法满足“禁止修改或删除”的要求。
结论：错误。
B. Create a new Amazon S3 bucket with S3 Versioning enabled. Use S3 Object Lock with a retention period in accordance
with the designated date. Configure the S3 bucket for static website hosting. Set an S3 bucket policy to allow read-only
access to the objects.
分析：
启用了S3 Versioning，可以保留对象的多个版本。
使用了S3 Object Lock并设置了保留期，可以防止文件在指定日期之前被修改或删除。
配置了静态网站托管和存储桶策略，允许只读访问。
此选项全面满足题目要求。
结论：正确。
C. Create a new Amazon S3 bucket with S3 Versioning enabled. Configure an event trigger to run an AWS Lambda
function in case of object modification or deletion. Configure the Lambda function to replace the objects with the original
versions from a private S3 bucket.
分析：
启用了S3 Versioning，可以保留对象的多个版本。
使用Lambda函数来恢复被修改或删除的对象，但这种方法复杂且无法完全防止文件被修改或删除。
此选项不如使用S3 Object Lock直接防止修改或删除有效。
结论：错误。
D. Upload all files to an Amazon S3 bucket that is configured for static website hosting. Select the folder that contains the
files. Use S3 Object Lock with a retention period in accordance with the designated date. Grant read-only IAM permissions
to any AWS principals that access the S3 bucket.
分析：
虽然使用了S3 Object Lock并设置了保留期，但“选择文件夹”这一操作在S3中并不是直接支持的，因为S3是基于对象存储的，而不是
基于文件夹的。
此选项的描述存在技术上的不准确性。
结论：错误。
