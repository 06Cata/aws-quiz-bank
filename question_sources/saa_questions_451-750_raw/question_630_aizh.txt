本题考察的是如何设计一个成本优化的解决方案，同时确保任务的可靠性和可恢复性。
涉及的AWS服务包括：Amazon EC2、AWS Lambda、Amazon ECS、Fargate、Amazon EventBridge。
2. 选项分析
A. Create a script that runs locally on an Amazon EC2 Reserved Instance that is triggered by a cron job.
错误原因：虽然EC2 Reserved Instance可以降低成本，但它仍然需要预付费用，并且需要持续运行实例，即使任务只运行2小时，实
例的成本仍然较高。
此外，如果任务中断，EC2实例无法自动恢复任务，需要额外的逻辑来处理失败的任务。
B. Create an AWS Lambda function triggered by an Amazon EventBridge scheduled event.
错误原因：AWS Lambda适合短时间运行的任务（最大15分钟），而本题任务需要运行长达2小时，超出了Lambda的执行时间限
制。
因此，Lambda无法满足任务的运行时需求。
C. Use an Amazon Elastic Container Service (Amazon ECS) Fargate task triggered by an Amazon EventBridge scheduled
event.
正确原因：Amazon ECS Fargate是无服务器的容器运行服务，按需计费，适合运行长时间任务。
通过EventBridge定时触发，可以确保任务每天运行一次。
如果任务中断，Fargate可以重新启动任务，确保任务从头开始运行。
这是最具成本效益且可靠的解决方案。
D. Use an Amazon Elastic Container Service (Amazon ECS) task running on Amazon EC2 triggered by an Amazon
EventBridge scheduled event.
错误原因：虽然ECS任务可以运行长时间任务，但运行在EC2实例上需要持续支付EC2实例的费用，即使任务只运行2小时，实例的
成本仍然较高。
相比Fargate，EC2的成本优化较差，且需要额外的管理工作。
