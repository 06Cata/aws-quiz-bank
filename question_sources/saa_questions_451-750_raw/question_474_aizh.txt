Question #474
A company has multiple VPCs across AWS Regions to support and run workloads that are isolated from workloads in other
Regions. Because of a recent application launch requirement, the company’s VPCs must communicate with all other VPCs across
all Regions.
Which solution will meet these requirements with the LEAST amount of administrative effort?

需求，该公司的 VPC 必须与所有区域中的其他 VPC 进行通信。
哪种解决方案能够以最少的管理工作量满足这些需求？
A. Use VPC peering to manage VPC communication in a single Region. Use VPC peering across Regions to manage VPC

C (100%)
本题考察的是跨区域VPC通信的解决方案，以及如何以最少的管理工作量实现高效的VPC互联。
涉及的服务包括：VPC Peering、AWS Direct Connect Gateway、AWS Transit Gateway、AWS PrivateLink。
2. 选项分析
A. 使用 VPC peering 在单一区域内管理 VPC 通信。使用跨区域的 VPC peering 管理 VPC 通信。
VPC Peering可以实现VPC之间的通信，但需要手动配置每个VPC之间的连接，尤其是跨区域时，管理工作量会显著增加。
随着VPC数量的增加，配置和维护的复杂性会呈指数级增长，因此不符合“最少的管理工作量”的要求。
此选项错误。
B. 使用 AWS Direct Connect gateways 跨所有区域连接 VPC 并管理 VPC 通信。
AWS Direct Connect主要用于建立本地数据中心与AWS之间的专线连接，而不是用于VPC之间的通信。
虽然Direct Connect Gateway可以支持跨区域通信，但它并不是设计用于管理多个VPC之间的通信，且配置复杂度较高。
此选项错误。
C. 使用 AWS Transit Gateway 在单一区域内管理 VPC 通信，并通过 Transit Gateway peering 跨区域管理 VPC 通信。
AWS Transit Gateway是专门设计用于管理多个VPC之间通信的服务，可以显著简化网络架构。
Transit Gateway支持跨区域的peering连接，能够高效地实现跨区域VPC通信，同时减少管理工作量。
此选项正确。
D. 使用 AWS PrivateLink 跨所有区域连接 VPC 并管理 VPC 通信。
AWS PrivateLink主要用于将服务暴露给其他VPC或账户，而不是用于VPC之间的全面通信。
PrivateLink需要为每个服务单独配置，无法满足本题中“所有VPC之间通信”的需求。
此选项错误。
