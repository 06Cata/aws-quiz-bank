Question #475
A company is designing a containerized application that will use Amazon Elastic Container Service (Amazon ECS). The application
needs to access a shared file system that is highly durable and can recover data to another AWS Region with a recovery point
objective (RPO) of 8 hours. The file system needs to provide a mount target m each Availability Zone within a Region.
A solutions architect wants to use AWS Backup to manage the replication to another Region.
Which solution will meet these requirements?
一家公司正在设计一个容器化应用程序，该应用程序将使用 Amazon Elastic Container Service (Amazon ECS)。该应用程序需要访问

一个解决方案架构师希望使用 AWS Backup 来管理到另一个区域的复制。

题目分析与解答
1. 考察的知识点
Amazon ECS与文件系统的集成。
文件系统的高可用性和跨区域数据恢复能力。
AWS Backup的使用和支持的文件系统类型。
2. 选项分析
A. Amazon FSx for Windows File Server with a Multi-AZ deployment
Amazon FSx for Windows File Server支持多可用区部署，提供高可用性。
但它主要用于Windows环境，不适合容器化应用场景。
此外，AWS Backup支持FSx for Windows File Server的备份，但跨区域恢复能力不如Amazon EFS。
错误原因：不符合容器化应用需求，且跨区域恢复能力不完全满足题目要求。
B. Amazon FSx for NetApp ONTAP with a Multi-AZ deployment
Amazon FSx for NetApp ONTAP支持多可用区部署，提供高可用性。
它支持高级存储功能，但主要用于NetApp环境，复杂性较高。
虽然AWS Backup支持FSx for NetApp ONTAP，但跨区域恢复能力不如Amazon EFS。
错误原因：复杂性较高，不完全符合题目需求。
C. Amazon Elastic File System (Amazon EFS) with the Standard storage class
Amazon EFS是一个完全托管的文件系统，支持容器化应用场景。
它提供多可用区的挂载目标，满足高可用性需求。
通过AWS Backup，Amazon EFS可以实现跨区域数据恢复，满足RPO为8小时的要求。
正确原因：完全符合题目需求，包括容器化支持、高可用性和跨区域恢复能力。
D. Amazon FSx for OpenZFS
Amazon FSx for OpenZFS支持高性能文件存储，但主要用于OpenZFS环境。
它不支持多可用区挂载目标，且AWS Backup的跨区域恢复能力不如Amazon EFS。
错误原因：不支持多可用区挂载目标，且跨区域恢复能力不完全满足题目要求。
3. 我选的答案
C. Amazon Elastic File System (Amazon EFS) with the Standard storage class
4. 官方答案
C. Amazon Elastic File System (Amazon EFS) with the Standard storage class

我的答案与官方答案一致。
原因分析完全符合题目需求，Amazon EFS是最佳选择。
