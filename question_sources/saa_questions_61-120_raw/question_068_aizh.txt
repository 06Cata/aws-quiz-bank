题目分析与解答
1. 考察的知识点
混合架构设计：如何将本地基础设施扩展到AWS。
高可用性和低延迟连接：AWS Direct Connect和VPN的使用场景。
成本优化：选择合适的备份连接方案。
2. 选项分析
A. Provision an AWS Direct Connect connection to a Region. Provision a VPN connection as a backup if the primary Direct
Connect connection fails.
正确：AWS Direct Connect提供高性能、低延迟的专线连接，非常适合需要稳定连接的场景。
正确：VPN作为备份连接成本较低，虽然性能不如Direct Connect，但可以满足故障切换的需求。
符合题目要求：高可用性、低延迟、成本优化。
B. Provision a VPN tunnel connection to a Region for private connectivity. Provision a second VPN tunnel for private
connectivity and as a backup if the primary VPN connection fails.
错误：VPN连接的性能和延迟不如Direct Connect，无法满足题目要求的“低延迟”需求。
错误：虽然成本较低，但两个VPN隧道无法提供Direct Connect的高性能连接。
C. Provision an AWS Direct Connect connection to a Region. Provision a second Direct Connect connection to the same
Region as a backup if the primary Direct Connect connection fails.
错误：虽然两个Direct Connect连接可以提供高可用性，但成本较高，不符合题目要求的“成本优化”。
错误：题目明确表示公司愿意接受较慢的备份连接，因此不需要两个Direct Connect连接。
D. Provision an AWS Direct Connect connection to a Region. Use the Direct Connect failover attribute from the AWS CLI to
automatically create a backup connection if the primary Direct Connect connection fails.
错误：Direct Connect failover属性并不能自动创建备份连接，它仅用于配置现有连接的故障切换。
错误：题目要求明确提到备份连接需要成本优化，而此选项未提及VPN作为备份。

A
