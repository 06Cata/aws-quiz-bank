A company has an AWS Glue extract, transform, and load (ETL) job that runs every day at the same time. The job processes XML
data that is in an Amazon S3 bucket. New data is added to the S3 bucket every day. A solutions architect notices that AWS Glue is
processing all the data during each run.
What should the solutions architect do to prevent AWS Glue from reprocessing old data?
一家公司有一个 AWS Glue 提取、转换和加载 (ETL) 作业，每天在同一时间运行。该作业处理存储在 Amazon S3 存储桶中的 XML
数据。每天都会有新数据添加到 S3 存储桶中。一位解决方案架构师注意到 AWS Glue 在每次运行时都会处理所有数据。
解决方案架构师应该怎么做以防止 AWS Glue 重新处理旧数据？
A. Edit the job to use job bookmarks. Most Voted
编辑作业以使用作业书签。
B. Edit the job to delete data after the data is processed.
编辑作业以在数据处理后删除数据。
C. Edit the job by setting the NumberOfWorkers field to 1.
编辑作业，通过设置 NumberOfWorkers 字段为 1。
D. Use a FindMatches machine learning (ML) transform.
使用 FindMatches 机器学习 (ML) 转换。
题目分析与解答
1. 考察的知识点
本题考察的是AWS Glue的作业书签（Job Bookmarks）功能，该功能用于跟踪ETL作业的处理状态，避免重复处理已处理过的数
据。
2. 选项分析
Correct Answer: A
正确答案：A
Community vote distribution
社区投票分布
A (100%)
A. Edit the job to use job bookmarks.
正确：AWS Glue的作业书签功能可以跟踪已处理的数据，防止重复处理旧数据。这正是解决问题的最佳方法。
B. Edit the job to delete data after the data is processed.
错误：删除数据可能会导致数据丢失，且不符合实际业务需求。题目并未要求删除数据，而是要求避免重复处理。
C. Edit the job by setting the NumberOfWorkers field to 1.
错误：NumberOfWorkers字段用于设置作业的并行度，与避免重复处理数据无关。
D. Use a FindMatches machine learning (ML) transform.
错误：FindMatches是用于数据去重的机器学习转换，与避免重复处理数据无关。
3. 我的答案
A. Edit the job to use job bookmarks.
