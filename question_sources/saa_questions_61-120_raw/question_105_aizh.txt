本题考察的是 AWS Lambda 的权限配置，尤其是基于资源的策略（Resource-based Policy）和执行角色（Execution Role）的使
用。
同时涉及到 AWS 服务之间的调用权限配置，例如 Amazon EventBridge 调用 AWS Lambda 的权限。
重点是遵循最小权限原则（Principle of Least Privilege）。
2. 选项分析
A. Add an execution role to the function with lambda:InvokeFunction as the action and * as the principal.
错误：执行角色是赋予 Lambda 函数权限去调用其他服务，而不是赋予其他服务权限调用 Lambda 函数。
主体（Principal）为 * 是不符合最小权限原则的，因为它允许任何主体调用 Lambda 函数。
B. Add an execution role to the function with lambda:InvokeFunction as the action and Service: lambda.amazonaws.com
as the principal.
错误：执行角色的主体（Principal）应该是 Lambda 函数本身，而不是其他服务（如 lambda.amazonaws.com）。
此外，lambda:InvokeFunction 是用于调用 Lambda 函数的权限，而不是赋予 Lambda 函数执行权限的正确操作。
C. Add a resource-based policy to the function with lambda:* as the action and Service: events.amazonaws.com as the
principal.
错误：lambda:* 的权限范围过大，不符合最小权限原则。
主体（Principal）Service: events.amazonaws.com 是正确的，因为 EventBridge 需要调用 Lambda 函数。
D. Add a resource-based policy to the function with lambda:InvokeFunction as the action and Service:
events.amazonaws.com as the principal.
正确：lambda:InvokeFunction 是最小权限的正确操作，允许 EventBridge 调用 Lambda 函数。
主体（Principal）Service: events.amazonaws.com 是正确的，因为 EventBridge 是调用 Lambda 的服务。
