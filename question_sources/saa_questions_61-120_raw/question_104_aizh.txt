2. 选项分析
A. Use AWS Shield Advanced to stop the DDoS attack.
正确： AWS Shield Advanced是专门设计用于保护应用程序免受复杂和大规模DDoS攻击的服务。它提供了高级保护功能，并与其他
AWS服务集成，可以有效缓解攻击。
B. Configure Amazon GuardDuty to automatically block the attackers.
错误： Amazon GuardDuty是一个威胁检测服务，用于识别潜在的恶意活动，但它不会自动阻止攻击者。它的主要功能是检测和报
告，而不是直接防御。
C. Configure the website to use Amazon CloudFront for both static and dynamic content.
正确： Amazon CloudFront是一个内容分发网络（CDN），可以帮助缓解DDoS攻击。它通过分布式边缘位置提供内容，减少对源
服务器的直接访问，并提供内置的DDoS保护。
D. Use an AWS Lambda function to automatically add attacker IP addresses to VPC network ACLs.
错误： 虽然可以使用AWS Lambda自动更新VPC网络ACL，但这种方法不适合应对大规模DDoS攻击，因为攻击可能来自成千上万的
IP地址，手动管理ACL效率低且不够全面。
E. Use EC2 Spot Instances in an Auto Scaling group with a target tracking scaling policy that is set to 80% CPU utilization.
错误： 使用Spot实例和Auto Scaling可以提高成本效益，但它无法直接缓解DDoS攻击。DDoS攻击的目标是耗尽资源，而不是优化
资源使用。
