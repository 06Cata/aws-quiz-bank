A company hosts an application on AWS Lambda functions that are invoked by an Amazon API Gateway API. The Lambda
functions save customer data to an Amazon Aurora MySQL database. Whenever the company upgrades the database, the Lambda
functions fail to establish database connections until the upgrade is complete. The result is that customer data is not recorded for
some of the event.
A solutions architect needs to design a solution that stores customer data that is created during database upgrades.
Which solution will meet these requirements?

解决方案架构师需要设计一个解决方案来存储在数据库升级期间创建的客户数据。
哪种解决方案可以满足这些需求？

Correct Answer: D
正确答案：D
题目分析与解答
1. 考察的知识点
Amazon SQS FIFO队列的使用场景和优势。
AWS Lambda与Amazon Aurora MySQL数据库的集成。
如何设计高可用性和容错的架构以应对数据库升级或不可用的情况。
2. 选项分析
A. Provision an Amazon RDS proxy to sit between the Lambda functions and the database. Configure the Lambda
functions to connect to the RDS proxy.
正确性分析：Amazon RDS Proxy可以帮助管理数据库连接池，减少连接管理的开销，但它无法解决数据库升级期间的不可用问题。
结论：此选项无法满足题目要求，因为它不能存储客户数据以防止数据丢失。
B. Increase the run time of the Lambda functions to the maximum. Create a retry mechanism in the code that stores the
customer data in the database.
正确性分析：增加Lambda运行时间和重试机制可以减少失败的可能性，但如果数据库升级时间较长，Lambda函数仍可能超时或失
败。
结论：此选项无法完全解决问题，且不符合最佳实践。
C. Persist the customer data to Lambda local storage. Configure new Lambda functions to scan the local storage to save
the customer data to the database.
正确性分析：Lambda的本地存储是临时的，无法保证数据持久性。此外，扫描本地存储的方式复杂且不可靠。
结论：此选项不符合题目要求，也不符合最佳实践。
D. Store the customer data in an Amazon Simple Queue Service (Amazon SQS) FIFO queue. Create a new Lambda function
that polls the queue and stores the customer data in the database.
正确性分析：Amazon SQS FIFO队列可以确保消息的顺序和持久性。即使数据库升级期间不可用，消息仍然可以存储在队列中，升
级完成后可以通过轮询将数据写入数据库。
结论：此选项符合题目要求，是最佳解决方案。
