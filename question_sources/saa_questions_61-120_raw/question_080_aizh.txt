题目分析与解答
1. 考察的知识点
Amazon Machine Image (AMI)共享机制
Amazon Elastic Block Store (EBS)加密与AWS Key Management Service (KMS)的使用
安全性最佳实践
2. 选项分析
A. Make the encrypted AMI and snapshots publicly available. Modify the key policy to allow the MSP Partner's AWS
account to use the key.
错误：将加密的AMI和快照公开可用会导致安全风险，因为任何人都可以访问这些资源。
错误：虽然修改密钥策略可以允许MSP合作伙伴使用密钥，但公开共享AMI违背了安全性最佳实践。
B. Modify the launchPermission property of the AMI. Share the AMI with the MSP Partner's AWS account only. Modify the
key policy to allow the MSP Partner's AWS account to use the key.
正确：通过修改launchPermission属性，可以限制AMI的共享范围，仅与指定的AWS账户共享。
正确：修改密钥策略以允许MSP合作伙伴的AWS账户使用加密密钥，确保EBS卷的加密仍然有效。
此选项符合安全性最佳实践，同时满足共享需求。
C. Modify the launchPermission property of the AMI. Share the AMI with the MSP Partner's AWS account only. Modify the
key policy to trust a new KMS key that is owned by the MSP Partner for encryption.
错误：虽然修改launchPermission属性是正确的，但修改密钥策略以信任MSP合作伙伴的新KMS密钥会增加复杂性。
错误：此选项要求额外的密钥管理步骤，而这些步骤并非必要。
D. Export the AMI from the source account to an Amazon S3 bucket in the MSP Partner's AWS account, Encrypt the S3
bucket with a new KMS key that is owned by the MSP Partner. Copy and launch the AMI in the MSP Partner's AWS
account.
错误：导出AMI到S3并重新加密增加了不必要的复杂性。
错误：此方法不直接利用AWS提供的AMI共享功能，效率较低。
