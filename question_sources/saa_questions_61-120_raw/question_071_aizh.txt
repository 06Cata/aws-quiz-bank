A company runs a shopping application that uses Amazon DynamoDB to store customer information. In case of data corruption, a
solutions architect needs to design a solution that meets a recovery point objective (RPO) of 15 minutes and a recovery time
objective (RTO) of 1 hour.
What should the solutions architect recommend to meet these requirements?
一家公司运行一个购物应用程序，该应用程序使用 Amazon DynamoDB 存储客户信息。为防止数据损坏，解决方案架构师需要设计
一个解决方案，以满足 15 分钟的恢复点目标 (RPO) 和 1 小时的恢复时间目标 (RTO)。
解决方案架构师应该推荐什么来满足这些要求？
A. Configure DynamoDB global tables. For RPO recovery, point the application to a different AWS Region.
配置 DynamoDB 全球表。对于 RPO 恢复，将应用程序指向另一个 AWS 区域。

恢复点目标（RPO）和恢复时间目标（RTO）的概念及其在灾难恢复中的应用。
2. 选项分析
A. Configure DynamoDB global tables. For RPO recovery, point the application to a different AWS Region.
错误原因：虽然DynamoDB全球表可以实现跨区域复制，但它主要用于高可用性和低延迟访问，而不是备份和恢复。全球表无法满
足数据恢复到特定时间点的需求，因此无法满足RPO为15分钟的要求。
B. Configure DynamoDB point-in-time recovery. For RPO recovery, restore to the desired point in time.
正确原因：点时间恢复（Point-in-Time Recovery）是DynamoDB的内置功能，允许用户将表恢复到过去35天内的任意时间点。这完
全满足RPO为15分钟的要求，同时恢复时间目标（RTO）也可以在1小时内完成。
C. Export the DynamoDB data to Amazon S3 Glacier on a daily basis. For RPO recovery, import the data from S3 Glacier to
DynamoDB.
错误原因：将数据导出到S3 Glacier是一个冷存储解决方案，数据恢复时间较长，无法满足RTO为1小时的要求。此外，数据导出是
每日进行的，无法满足RPO为15分钟的要求。
D. Schedule Amazon Elastic Block Store (Amazon EBS) snapshots for the DynamoDB table every 15 minutes. For RPO
recovery, restore the DynamoDB table by using the EBS snapshot.
错误原因：Amazon DynamoDB不使用EBS存储，因此无法对DynamoDB表进行EBS快照。这种方法在技术上不可行。
3. 我选的答案
B. Configure DynamoDB point-in-time recovery. For RPO recovery, restore to the desired point in time.

B. Configure DynamoDB point-in-time recovery. For RPO recovery, restore to the desired point in time.

我的答案与官方答案一致。
原因：点时间恢复（Point-in-Time Recovery）是DynamoDB的最佳实践，能够满足题目中提出的RPO和RTO要求。
