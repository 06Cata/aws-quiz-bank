Which design principles are included in the reliability pillar of the AWS Well-Architected Framework? (Choose two.)
哪些设计原则包含在AWS Well-Architected Framework的可靠性支柱中？（选择两个）
A. Automatically recover from failure.
自动从故障中恢复。
B. Grant everyone access to increase AWS service quotas.
授予所有人访问权限以增加AWS服务配额。
C. Stop guessing capacity.
停止猜测容量。
D. Design applications to run in a single Availability Zone.
设计应用程序在单个可用区运行。
E. Plan to increase AWS service quotas first in a secondary AWS Region.
计划首先在次级AWS区域增加AWS服务配额。
题目分析与解答
1. 考察的知识点
本题考察的是AWS Well-Architected Framework中的可靠性（Reliability）支柱的设计原则。
可靠性支柱关注系统在故障情况下的恢复能力、容量规划以及架构设计的弹性。
2. 选项分析
A. Automatically recover from failure. 自动从故障中恢复。
正确。可靠性支柱强调系统能够自动从故障中恢复，例如通过自动化监控和故障处理机制。
B. Grant everyone access to increase AWS service quotas. 授予所有人访问权限以增加AWS服务配额。
Correct Answer: AC
正确答案：AC

AC (100%)
错误。授予所有人访问权限与安全性相关，而不是可靠性支柱的设计原则。
C. Stop guessing capacity. 停止猜测容量。
正确。可靠性支柱强调使用自动扩展和弹性设计来避免容量不足或过度分配。
D. Design applications to run in a single Availability Zone. 设计应用程序在单个可用区运行。
错误。可靠性支柱建议跨多个可用区设计应用程序以提高容错能力，而不是限制在单个可用区运行。
E. Plan to increase AWS service quotas first in a secondary AWS Region. 计划首先在次级AWS区域增加AWS服务配额。
错误。可靠性支柱并未建议在次级区域增加服务配额，这与容量规划的核心原则不符。
