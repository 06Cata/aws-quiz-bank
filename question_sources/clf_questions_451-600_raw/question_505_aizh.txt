Which AWS Cloud design principle is a company using when the company implements AWS CloudTrail?
公司在实施 AWS CloudTrail 时使用了哪项 AWS 云设计原则？
A. Activate traceability.
激活可追溯性。
B. Use serverless compute architectures.
使用无服务器计算架构。
C. Perform operations as code.
将操作作为代码执行。
D. Go global in minutes.
几分钟内实现全球化。
题目分析与解答
1. 考察的知识点
本题考察的是AWS设计原则中的“激活可追溯性（Activate traceability）”，以及AWS CloudTrail的功能和用途。
2. 选项分析
A. Activate traceability. 激活可追溯性。 正确。AWS CloudTrail的主要功能是记录用户和服务的操作日志，提供审计和可追溯性功
能。这与“激活可追溯性”设计原则完全吻合。
B. Use serverless compute architectures. 使用无服务器计算架构。 错误。无服务器计算架构（如AWS Lambda）是关于如何设计
应用程序的计算资源，而CloudTrail是一个日志记录服务，与无服务器架构无直接关联。
C. Perform operations as code. 将操作作为代码执行。 错误。“将操作作为代码执行”是指使用代码来自动化操作和管理资源（如
通过AWS CloudFormation），而CloudTrail是用于记录操作日志，不涉及代码执行。
Correct Answer: A

A (100%)
D. Go global in minutes. 几分钟内实现全球化。 错误。“几分钟内实现全球化”是指AWS的全球基础设施支持快速扩展到多个区
域，而CloudTrail的功能与全球化无直接关系。
3. 我的答案
A. Activate traceability. 激活可追溯性。
4. 官方答案
A. Activate traceability. 激活可追溯性。
