A company needs an event history of which AWS resources the company has created.
Which AWS service will provide this information?
一家公司需要了解其创建的AWS资源的事件历史。
哪个AWS服务可以提供此信息？
A. Amazon CloudWatch
Amazon CloudWatch

AWS CloudTrail

Amazon Aurora
D. Amazon EventBridge
Amazon EventBridge
题目分析与解答
1. 考察的知识点
本题考察的是AWS服务的功能，特别是与事件历史记录和资源创建相关的服务。
2. 选项分析
A. Amazon CloudWatch: 错误。Amazon CloudWatch主要用于监控和管理AWS资源的性能指标和日志数据，但它不提供资源创建
的事件历史。
Correct Answer: B
正确答案：B

社区投票分布

B. AWS CloudTrail: 正确。AWS CloudTrail记录与AWS账户相关的所有API调用和事件历史，包括资源创建的详细信息。
C. Amazon Aurora: 错误。Amazon Aurora是一个关系数据库服务，与事件历史记录无关。
D. Amazon EventBridge: 错误。Amazon EventBridge是一个事件总线服务，用于连接应用程序和触发事件，但它不记录资源创建
的历史。
3. 我的答案
B. AWS CloudTrail
4. 官方答案
B. AWS CloudTrail
