A company uses a third-party identity provider (IdP). The company wants to provide its employees with access to AWS accounts
and services without requiring another set of login credentials.
Which AWS service will meet this requirement?
A. AWS Directory Service
B. Amazon Cognito
C. AWS IAM Identity Center Most Voted
D. AWS Resource Access Manager (AWS RAM)

dragossky Most Recent 5 months ago
Selected Answer: C
Correct answer: C. AWS IAM Identity Center
Explanation:
AWS IAM Identity Center (formerly AWS Single Sign-On) enables federated access to AWS accounts and applications using an existing third-party IdP
via SAML or OIDC, so employees can sign in with their existing corporate credentials without creating separate AWS logins.
AWS Directory Service (A) integrates with Microsoft Active Directory for directory needs but doesn’t itself provide cross-account SSO with third-party
IdPs as seamlessly as IAM Identity Center.
Amazon Cognito (B) is primarily for end-user authentication to your applications, not for workforce access to AWS accounts and services.

Selected Answer: C
AWS IAM Identity Center (formerly AWS Single Sign-On) enables single sign-on (SSO) for workforce users by integrating with a third-party identity
provider (IdP).
It allows employees to:
Use their existing corporate credentials
Access multiple AWS accounts and AWS services
Avoid managing separate AWS usernames and passwords
Why the other options are incorrect
A. AWS Directory Service – Provides managed directories, but does not directly provide SSO across AWS accounts by itself.
B. Amazon Cognito – Designed for customer-facing application users, not workforce access to AWS accounts.
D. AWS Resource Access Manager (AWS RAM) – Shares AWS resources across accounts, not identities.
✅ Final Answer: C. AWS IAM Identity Center
