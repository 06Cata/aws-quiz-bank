Selected Answer: C
Correct answer: C. AWS Direct Connect
Explanation:
AWS Direct Connect provides a dedicated, private network link between your on-premises environment and AWS. Implementing it typically involves
working with an ISP or network provider and using a colocation facility or a Direct Connect partner location to establish the physical connection.
AWS VPN (A) uses the public internet to create an encrypted tunnel; it does not require a colocation facility.
Amazon Connect (B) is a cloud-based contact center service; no ISP/colo setup is required beyond normal internet access.
Internet gateway (D) is a VPC component that enables internet access for resources in the VPC; it does not require ISP coordination or colocation.

Selected Answer: C
AWS Direct Connect requires working with:
An internet service provider (ISP) or AWS Direct Connect Partner
An AWS Direct Connect location (typically a colocation facility)
AWS Direct Connect provides a dedicated, private network connection from on-premises infrastructure to AWS, which improves bandwidth
consistency and reduces latency compared to internet-based connections.
Why the other options are incorrect
A. AWS VPN – Uses encrypted tunnels over the public internet; no colocation facility required.
B. Amazon Connect – A cloud contact center service.
D. Internet gateway – Enables internet access for a VPC; does not require a colocation facility.
✅ Final Answer: C. AWS Direct Connect
