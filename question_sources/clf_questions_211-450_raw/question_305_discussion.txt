Provisioning complete environments
While you can create complex test environments containing multiple instances using the AWS APIs, command line tools, or the AWS Management
Console, AWS CloudFormation makes it even easier to create a collection of related AWS resources and provision them in an orderly and predictable
fashion.
AWS CloudFormation uses templates to create and delete a collection of resources together as a single unit (a stack). A complete test environment
running on AWS can be described in a template, which is a text file in JSON or YAML format.
