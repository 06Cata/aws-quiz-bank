Question #322
A user needs a relational database but does not have the resources to manage the hardware, resiliency, and replication.
Which AWS service option meets the user's requirements?
用户需要一个关系型数据库，但没有资源来管理硬件、弹性和复制。
哪个AWS服务选项满足用户的需求？
A. Run MySQL on Amazon Elastic Container Service (Amazon ECS).
在Amazon Elastic Container Service (Amazon ECS)上运行MySQL。
B. Run MySQL on Amazon EC2.
在Amazon EC2上运行MySQL。
C. Choose Amazon RDS for MySQL. Most Voted
选择Amazon RDS for MySQL。
D. Choose Amazon ElastiCache for Redis.
选择Amazon ElastiCache for Redis。
题目分析与解答
1. 考察的知识点
本题考察的是AWS托管服务的选择，尤其是针对关系型数据库的管理需求。
重点在于理解AWS服务的功能和适用场景，例如Amazon RDS、Amazon EC2、Amazon ECS和Amazon ElastiCache。
2. 选项分析
A. Run MySQL on Amazon Elastic Container Service (Amazon ECS)
Correct Answer: C

不符合题目要求的“无需管理硬件、弹性和复制”的条件。
C. Choose Amazon RDS for MySQL
正确：Amazon RDS是一个托管的关系型数据库服务，支持MySQL。它自动处理硬件管理、备份、弹性扩展和复制。
完全符合题目要求的“无需管理硬件、弹性和复制”的条件。
D. Choose Amazon ElastiCache for Redis
错误：Amazon ElastiCache是一个托管的内存缓存服务，支持Redis和Memcached，但它不是关系型数据库。
题目明确要求关系型数据库，因此此选项不符合要求。
